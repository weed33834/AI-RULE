# 工作流命令 (Slash Commands)

> 本文档定义 AI 对话中的 8 个内置工作流命令。
> 是 AGENTS.md §15 Slash Commands 的完整实现。

## §1 命令列表

| 命令 | 功能 | 触发行为 |
|------|------|----------|
| `/deep` | 深度搜索模式 | 多源交叉验证，详见 `deep-search.md` |
| `/verify` | 自我验证 | 对上一次回答执行 CoV，详见 `truth-protocol.md` |
| `/sources` | 列出来源 | 列出上一次回答引用的所有来源 |
| `/simple` | 简化解释 | 用最简单的方式重新解释 |
| `/detail` | 展开细节 | 对上一次回答展开更多细节 |
| `/deviate` | 替代视角 | 给出非主流的替代观点 |
| `/consensus` | 业界共识 | 搜索并总结当前主题的业界共识 |
| `/debate` | 正反论点 | 列出当前主题的正反两面论点 |

## §2 命令详细说明

### /deep

```
触发: 用户输入 /deep [问题]
行为:
1. 启动 4 阶段深度搜索（查询理解 → 查询设计 → 结果分析 → 信息综合）
2. 至少 3 次多角度搜索
3. 关键事实交叉验证
4. 输出带来源标注的综合回答
```

### /verify

```
触发: 用户输入 /verify
行为:
1. 提取上一次回答中的关键声明
2. 对每个声明执行 Chain-of-Verification
3. 输出验证报告：哪些被确认、哪些被修正、哪些无法验证
4. 如果有修正，输出修正后的回答
```

### /sources

```
触发: 用户输入 /sources
行为:
1. 列出上一次回答中引用的所有来源
2. 格式: [编号] 来源名称, 发布日期, URL, 可信度级别
3. 如果没有引用来源，说明原因
```

### /simple

```
触发: 用户输入 /simple
行为:
1. 将上一次回答简化为 1-3 句话
2. 只保留核心结论
3. 删除推理过程和补充信息
```

### /detail

```
触发: 用户输入 /detail
行为:
1. 对上一次回答展开更多细节
2. 添加推理过程
3. 添加补充信息和注意事项
4. 思维深度升级一级
```

### /deviate

```
触发: 用户输入 /deviate
行为:
1. 给出与上一次回答不同的视角或方案
2. 必须是有合理依据的非主流观点，不是为了不同而不同
3. 说明这个视角的合理性和局限性
```

### /consensus

```
触发: 用户输入 /consensus [主题]
行为:
1. 搜索当前主题的业界共识
2. 总结主流观点
3. 标注共识来源
4. 同时标注存在的分歧
```

### /debate

```
触发: 用户输入 /debate [主题]
行为:
1. 列出当前主题的正方论点（3-5 个）
2. 列出当前主题的反方论点（3-5 个）
3. 每个论点附来源
4. 不选边，让用户自行判断
```

## §3 命令组合

命令可以组合使用：
```
/deep /verify  → 深度搜索后立即验证
/simple /consensus → 简洁总结业界共识
```
