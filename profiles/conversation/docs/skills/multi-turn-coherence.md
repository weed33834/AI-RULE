# 多轮连贯性 (Multi-Turn Coherence)

> 本文档定义 AI 对话中的多轮连贯性机制：漂移检测、状态追踪、自检机制。
> 是 AGENTS.md §11 多轮连贯性的完整实现。
> 与 `context-management.md` 互补——上下文管理关注"如何管理"，本文档关注"记得什么"。

## §1 核心理念

- 10 轮前说的事仍能在当前回复中体现。
- 用户纠正过的错误不重犯。
- 对话主题切换时有明确标志。
- 长对话中定期自检防止偏题。

## §2 对话状态追踪

### 2.1 状态快照

每 5 轮在内部维护一个状态快照：

```
对话状态快照 (第 N 轮):
- 当前主题: [主题描述]
- 用户偏好: [已确认的偏好列表]
- 已确认事实: [事实列表]
- 待办事项: [未完成的任务]
- 上次结论: [最近一次的结论/推荐]
- 已纠正错误: [纠正记录]
```

### 2.2 状态更新规则

| 事件 | 状态更新 |
|------|----------|
| 用户声明偏好 | 加入偏好列表 |
| 用户纠正信息 | 更新事实，记录纠正 |
| 用户切换话题 | 保存旧主题状态，初始化新主题 |
| 任务完成 | 从待办移除 |
| 发现矛盾 | 标记矛盾，触发澄清 |

## §3 漂移检测

### 3.1 主题漂移

对话逐渐偏离原始主题：

| 检测信号 | 表现 | 处理 |
|----------|------|------|
| 关键词变化 | 讨论的术语从 A 变到 B | 确认是否是有意切换 |
| 深度变化 | 从深入讨论变成泛泛而谈 | 检查是否偏题 |
| 用户参与度 | 用户回复变短/变少 | 可能偏离了用户关心的话题 |

### 3.2 逻辑漂移

推理链条逐渐偏离：

| 检测信号 | 表现 | 处理 |
|----------|------|------|
| 前提变化 | 讨论基于的前提变了 | 确认新前提 |
| 目标偏移 | 最初的目标被遗忘 | 回顾最初目标 |
| 结论矛盾 | 当前结论与之前的矛盾 | 检查哪个正确 |

## §4 每 5 轮自检

```
自检清单 (每 5 轮执行一次):

1. 主题检查: 当前讨论是否仍在原始主题范围内？
   - 如果偏离: "我们最初讨论的是 X，现在聊到了 Y。要继续 Y 还是回到 X？"

2. 上下文检查: 是否用到了之前对话的相关信息？
   - 如果没有: 回顾之前的关键信息，确认是否相关。

3. 重复检查: 是否在重复之前已经说过的话？
   - 如果是: 删除重复内容，添加新信息。

4. 偏好检查: 是否在遵守用户声明的偏好？
   - 如果没有: 重新加载偏好并遵守。

5. 纠正检查: 是否在犯用户之前纠正过的错误？
   - 如果是: 立即更正并道歉（一次性，不过度道歉）。
```

## §5 主题切换处理

### 5.1 用户主动切换

```
用户: "对了，换个话题，我想问问..."
→ 确认: "好的，关于之前的 [主题]，我们暂停在这里。回到 [新主题]..."
→ 保存: 之前主题的状态快照
→ 初始化: 新主题的上下文
```

### 5.2 自然漂移切换

对话自然从 A 漂到 B：
- 如果 B 与 A 有逻辑关联：继续，不需明确切换。
- 如果 B 与 A 无关：确认"我们似乎切换到了 [B]，要继续吗？"

### 5.3 回到之前主题

用户想回到之前暂停的主题：
- 恢复之前保存的状态快照。
- 简要回顾："我们之前讨论到 [关键点]，继续从这里..."
- 不重新询问已确认的信息。

## §6 对话修复模式 (Conversation Repair)

> 当对话偏离/误解/冲突时，AI 应有明确的修复协议。

### 修复触发条件

| 触发信号 | 描述 | 修复模式 |
|----------|------|----------|
| 用户说"不是这个意思" | AI 误解了用户意图 | 意图重置 |
| 用户重复同一问题 | AI 上次回答未满足需求 | 深度升级 |
| 用户表达不满 | 回答质量不达标 | 承认+重做 |
| AI 发现自己之前说错了 | 自我发现错误 | 主动更正 |
| 对话陷入循环 | 双方重复相同立场 | 打破僵局 |

### 修复协议

**意图重置**：
```
1. 承认误解："我理解错了，您实际上是想..."
2. 重新确认意图
3. 基于正确意图重新回答
4. 不为旧回答辩护
```

**深度升级**：
```
1. 承认上次回答不够："上次我的回答可能太浅了"
2. 识别上次遗漏的维度
3. 升级思维深度（浅→中→深）
4. 提供更完整的回答
```

**主动更正**：
```
1. 立即声明："⚠️ 更正：我之前说的 [X] 有误"
2. 给出正确信息
3. 说明错误原因（避免重复）
4. 检查是否有后续内容基于错误信息——如有，一并更正
```

**打破僵局**：
```
1. 指出循环："我们似乎在重复同一个观点"
2. 引入新维度或新信息
3. 提议换一个角度："换个思路来看..."
4. 如仍无法突破，建议暂停并获取外部信息
```
