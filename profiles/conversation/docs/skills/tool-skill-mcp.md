# 工具/技能/MCP 策略 (Tool/Skill/MCP)

> 本文档定义工具、技能、MCP 三者的关系和使用策略。
> 是 AGENTS.md §14 工具与技能策略的完整实现。

## §1 三者关系

| 概念 | 比喻 | 说明 | 控制权 |
|------|------|------|--------|
| Tool | 手和脚 | 内置工具，开箱即用 | AI 可自主使用 |
| Skill | 菜谱 | `docs/skills/` 下的文档，教怎么做复杂事 | AI 按需读取 |
| MCP | 输血管 | 连接外部系统的后台服务 | 用户手动配置 |

## §2 工具使用原则

- 优先使用专用工具而非通用 shell 命令。
- 文件操作前先读取文件内容。
- 不创建不必要的文件。
- 优先编辑现有文件而非创建新文件。
- 高风险操作（删除、push、写入数据库）前确认。

## §3 技能使用原则

- 遇到复杂任务时，先检查 `docs/skills/` 是否有匹配的技能文档。
- 技能文档是"菜谱"——按需读取，不是每次都全量加载。
- 技能文档中的规则与 AGENTS.md 冲突时，以 AGENTS.md 为准。
- 执行完复杂任务后，考虑提取为新的技能文档。

## §4 MCP 策略

- MCP 是连接外部系统的后台服务，需要环境变量、端口、权限。
- AI 不得自行下载、安装、启动 MCP 服务。
- MCP 配置权在用户手里。
- AI 可以输出安装命令和配置 JSON 供用户参考。
- 已批准的 MCP 服务记录在配置文件中，仅供人工参考。

## §5 工具选择优先级

| 需求 | 优先级 |
|------|--------|
| 文件读取 | 专用读取工具 > shell cat |
| 文件搜索 | Grep/Glob > shell find/grep |
| 文件编辑 | 编辑工具 > shell sed/awk |
| 文件写入 | 写入工具 > shell echo/cat |
| 代码执行 | 终端工具 |
| 版本控制 | 终端 git |

## §6 与其他文档的关系

- 所有技能文档都是 "Skill" 层的一部分。
- `evolution-policy.md` 管理技能的生命周期。
- `security-checklist.md` 包含工具使用的安全检查。
