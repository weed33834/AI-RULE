# 信息源可信度评估 (Source Credibility)

> 本文档定义信息源可信度评估框架：5 级分类、冲突处理、时效性检查、偏见识别。
> 是 AGENTS.md §9 信息源质量评估的完整实现。
> 与 `truth-protocol.md` 互补——真实性协议定义验证流程，本文档定义来源评估标准。
> 与 `deep-search.md` 互补——搜索找到来源，本文档评估来源质量。
> **DAR 集成**：来源分级统一使用 DAR T1-T4 四档（本文档 A-E 映射为 T1-T4）。详见 `core/dar-spec.md` §3。

## §1 来源分级（DAR T1-T4 统一分级）

> **统一标准**：所有 Profile 使用 DAR 定义的 T1-T4 四档分级，替代此前 A-E 五档。映射关系：A→T1, B→T2, C→T3, D/E→T4。

| Tier | 来源类型 | 可信度 | 使用方式 | 典型示例 |
|------|----------|--------|----------|----------|
| **T1** (原A) | 官方文档、同行评审论文、标准制定机构 | 最高（×1.0） | 可直接引用 | Python docs、Nature、RFC、WHO |
| **T2** (原B) | 权威媒体、知名技术出版物、行业报告 | 高（×0.8） | 可引用，标注来源 | BBC、TechCrunch、Statista |
| **T3** (原C) | 技术博客、SO 高票回答、百科条目 | 中（×0.5） | 可引用，建议交叉验证 | Medium、Dev.to、Wikipedia |
| **T4** (原D/E) | 社区讨论、个人观点、社交媒体、匿名来源 | 低（×0.2） | 需交叉验证后才可引用 | Reddit、Twitter、匿名论坛 |

## §2 评估维度

### 2.1 作者资质

| 维度 | 高分 | 低分 |
|------|------|------|
| 专业背景 | 领域专家、知名机构 | 匿名、无背景 |
| 历史记录 | 有其他高质量内容 | 首次发布或历史质量低 |
| 利益冲突 | 无利益关联 | 有商业利益关联 |

### 2.2 内容质量

| 维度 | 高分 | 低分 |
|------|------|------|
| 数据支撑 | 有数据、引用、实验 | 纯观点、无数据 |
| 深度 | 详细分析、多维度 | 浅层概述、列表文章 |
| 平衡性 | 呈现正反两面 | 单方面、有偏见 |
| 更新频率 | 定期更新维护 | 一次性、长期不更新 |

### 2.3 时效性

| 信息类型 | 有效期 | 过期处理 |
|----------|--------|----------|
| 技术框架版本 | 1-2 年 | 标注"可能已过时" |
| 市场数据 | 6-12 个月 | 标注日期，建议查最新 |
| 学术研究 | 5-10 年 | 检查是否有新研究推翻 |
| 定义/概念 | 长期有效 | 通常无需更新 |
| 安全漏洞 | 3-6 个月 | 必须查最新状态 |

## §3 来源冲突处理

### 3.1 冲突类型

| 冲突类型 | 示例 | 处理方式 |
|----------|------|----------|
| 数据冲突 | A 说市场规模 5B，B 说 6B | 标注分歧，呈现范围 |
| 结论冲突 | A 说好，B 说不好 | 呈现双方论点，分析差异原因 |
| 时效冲突 | A（2023）说 X，B（2024）说 Y | 优先采用最新来源 |
| 定义冲突 | A 和 B 对同一术语定义不同 | 呈现两种定义，说明使用场景 |

### 3.2 冲突解决优先级

1. 同级别来源冲突 → 呈现各方观点，不强行选边。
2. 不同级别来源冲突 → 优先高级别来源，但标注低级别来源的不同意见。
3. 时效冲突 → 优先最新来源（除非旧来源是经典/公理）。
4. 方法论差异导致冲突 → 说明差异原因（如市场定义不同）。

## §4 偏见识别

### 4.1 常见偏见类型

| 偏见类型 | 表现 | 规避方法 |
|----------|------|----------|
| 来源偏见 | 过度依赖某一种类型来源 | 确保来源类型多样化 |
| 时效偏见 | 只看最新不看经典 | 对稳定知识不追新 |
| 可获得性偏见 | 只引用容易找到的信息 | 主动搜索不易找到的视角 |
| 确认偏见 | 只找支持自己观点的来源 | 主动搜索反面观点 |
| 语言偏见 | 只搜索中文/英文 | 关键信息双语搜索 |

### 4.2 偏见自检

引用信息前问自己：
- 我引用的来源是否都是同一类型？
- 我是否主动搜索过反面观点？
- 我是否因为"容易找到"而非"质量高"选择了某个来源？
- 我是否因为语言偏好遗漏了重要信息？

## §5 引用规范

### 5.1 引用格式

```
[信息内容] [来源标注]

来源标注格式：
- 在线文章: [来源名称, 发布日期, URL]
- 论文: [作者, 标题, 期刊/会议, 年份]
- 文档: [文档名称, 版本号, URL]
- 书籍: [作者, 书名, 出版社, 年份, 页码]
```

### 5.2 引用密度标准

| 内容类型 | 引用要求 |
|----------|----------|
| 事实声明 | 必须引用 |
| 数据/统计 | 必须引用 |
| 观点/推荐 | 标注"基于..." |
| 定义/概念 | 权威来源引用 |
| 推测 | 标注"推测："，无需引用 |
