# 演进策略 (Evolution Policy)

> 本文档定义规则的自我演进机制：技能生命周期、策展器、轨迹洞察。
> 是 AGENTS.md §17-18 的完整实现。

## §1 规则自进化

### 1.1 错两次加规则

在同一类问题上犯两次相同错误时：
1. 识别错误模式。
2. 向用户建议新增规则。
3. 用户确认后写入 AGENTS.md。
4. 运行 sync_rules.py 同步。

### 1.2 规则提案格式

```
[建议新增规则]
位置: §X（建议放在哪一节）
内容: [规则文本]
原因: 第 N 次犯同样的错误（附上具体案例）
预期效果: 添加此规则后能避免什么问题
```

### 1.3 规则衰减

| 遵守情况 | 措辞变化 |
|----------|----------|
| 连续 10 次正确遵守 | "必须" → "优先" |
| 连续 20 次正确遵守 | "优先" → "建议" |
| 出现 1 次违反 | 回升一级 |
| 出现 2 次违反 | 回到 "必须" |

## §2 技能 5 阶段生命周期

| 阶段 | 触发条件 | 动作 | 示例 |
|------|----------|------|------|
| 创建 | 完成复杂任务后 | 提取为可复用技能文档 | 完成了一次复杂的市场分析→创建分析模板 |
| 使用 | 遇到相似任务 | 自动加载匹配技能 | 下次市场分析时加载之前的模板 |
| 评估 | 技能使用后 | 收集反馈，评估效果 | 模板是否帮助提高了效率？ |
| 改进 | 评估发现缺陷 | 优化技能内容 | 添加遗漏的维度，删除冗余步骤 |
| 淘汰 | 过时或被替代 | 归档旧技能 | 工具已废弃，对应技能归档 |

## §3 自主技能策展器

### 3.1 定期扫描

每月扫描技能库：
- 检测过时技能（引用的工具/API 已废弃）。
- 检测重复技能（多个技能覆盖同一主题）。
- 检测低效技能（使用后效果不佳）。

### 3.2 策展报告

```
策展报告模板:
- 评审周期: 2025-01
- 评审技能数: 15
- 建议合并: 2 组（A+B → C）
- 建议淘汰: 1 个（X 工具已废弃）
- 建议改进: 3 个（缺少 Y 维度）
- 新增建议: 2 个（从最近任务中提取）
```

### 3.3 安全约束

- 策展器只建议，不执行。
- 合并/淘汰须用户确认。
- 改进可以自动执行但需记录变更。

## §4 轨迹洞察

记录 AI 在对话中的行为轨迹，用于发现改进机会：

| 洞察类型 | 记录内容 | 用途 |
|----------|----------|------|
| 高频降智模式 | 哪种降智模式出现最多 | 针对性加强对应规则 |
| 高频澄清 | 哪类信息最常需要澄清 | 考虑是否可以自动推断 |
| 用户满意度信号 | 用户追问/纠正/满意的模式 | 识别回答质量的改进方向 |
| 搜索效率 | 搜索次数与结果质量的关联 | 优化搜索策略 |

## §5 与其他文档的关系

- 所有技能文档都可能被策展器评估和改进。
- `truth-protocol.md` 的 CoV 流程效果通过轨迹洞察评估。
- `anti-dumb-ai.md` 的降智模式通过轨迹洞察发现高频问题。
