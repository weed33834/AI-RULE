# 类型惯例技能 (Genre Conventions)

> 每种小说类型都有读者期待的惯例。了解惯例不是为了套公式，而是知道什么时候遵守、什么时候打破。

## §1 主要类型惯例速查

### 科幻（Sci-Fi）
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 科技推演 | 基于科学原理的合理推演 | 软科幻可放宽 |
| "如果...会怎样" | 核心假设驱动故事 | 不可打破 |
| 科技对社会的影响 | 技术改变人类生活方式 | 不可打破 |
| 新奇感 | 展示读者未见过的世界/技术 | 不可打破 |

### 奇幻（Fantasy）
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 魔法体系 | 有规则的魔法系统 | 可打破（软奇幻） |
| 世界观深度 | 历史、地理、文化丰富 | 不可打破 |
| 善恶对抗 | 核心道德冲突 | 可打破（暗黑奇幻） |
| 旅途结构 | 角色踏上冒险之旅 | 可打破（都市奇幻） |

### 悬疑/推理（Mystery）
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 公平线索 | 读者有机会在揭晓前猜到答案 | 不可打破 |
| 红鲱鱼 | 误导性线索 | 可用但不过度 |
| 揭晓时刻 | 侦探解释推理过程 | 不可打破 |
| 动机合理 | 犯罪动机需令人信服 | 不可打破 |

### 言情（Romance）
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 浪漫张力 | 两人之间的吸引力贯穿始终 | 不可打破 |
| Happily Ever After | 主角最终在一起 | 可打破（悲剧言情） |
| 外部障碍 | 阻碍两人在一起的客观因素 | 不可打破 |
| 内部障碍 | 角色自身的心理/性格障碍 | 不可打破 |

### 恐怖（Horror）
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 恐惧递进 | 威胁从暗示→显现→逼近 | 不可打破 |
| 孤立感 | 角色无法获得帮助 | 不可打破 |
| 安全空间被入侵 | 最后的避风港也被攻破 | 不可打破 |
| 未知恐惧 | 看不见的比看得见的更可怕 | 部分可打破 |

### 历史（Historical）
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 史实准确 | 大事件、人物、时代背景忠于史实 | 不可打破 |
| 时代细节 | 服饰、饮食、交通、语言符合时代 | 不可打破 |
| 虚构合理 | 虚构角色/事件需与史实不矛盾 | 不可打破 |
| 现代视角平衡 | 不用现代价值观过度批判历史人物 | 可打破 |

### 武侠/仙侠
| 惯例 | 说明 | 可打破？ |
|------|------|----------|
| 功夫/修炼体系 | 有层级的武学/修仙体系 | 不可打破 |
| 江湖/修仙界 | 独立于世俗社会的世界 | 可打破（都市武侠） |
| 侠义精神 | 核心道德准则 | 可打破（暗黑武侠） |
| 师承门派 | 武学传承关系 | 可打破 |

## §2 类型融合

### 常见融合模式
| 融合 | 示例 |
|------|------|
| 科幻+悬疑 | 赛博朋克侦探 |
| 奇幻+言情 | 魔法学院恋爱 |
| 恐怖+科幻 | 太空隔离恐怖 |
| 历史+奇幻 | 架空历史魔法 |
| 武侠+悬疑 | 江湖推理 |

### 融合原则
- 选择 1 个主类型 + 1 个副类型——不要三四个类型混在一起。
- 主类型的"不可打破"惯例优先。
- 副类型的惯例选择性融入。

## §3 反套路

### 何时反套路
- 当套路已成为陈词滥调、读者已审美疲劳时。
- 当反套路能带来更好的故事效果时。
- 当角色的发展方向自然偏离套路时。

### 反套路的风险
- 反套路不等于随机——打破惯例需要用更好的东西替代。
- 读者期待的"不可能打破"的惯例不要打破——除非你有绝对的把握写得更好。
- 反套路需要提前铺垫——不能在结尾突然"颠覆"。

## §4 类型与内容分级

| 类型 | 默认建议分级 | 注意事项 |
|------|-------------|----------|
| 全年龄向 | 全年龄 | 避免恐怖/暴力元素 |
| YA（青少年） | PG-13 | 可有冲突但不露骨 |
| 成人向 | R-15 / R-18 | 需用户明确确认分级 |

## 核心原则
- 惯例是工具不是枷锁——了解它才能用好它或打破它。
- 读者选择类型时代入了期待——完全无视惯例会让读者失望。
- 最好的创新是在惯例框架内找到新的表达方式。
