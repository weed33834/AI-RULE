# 生成后分析管线 (Post-Generation Pipeline)

> 本文档定义 AI 小说创作中生成文本的质量分析流水线。
> 研究表明：约 70% 的 AI 写作质量问题无法靠提示词解决，需要生成后的系统化分析。
> 与 `anti-ai-patterns.md` 互补——后者定义"什么是 AI 味"，本文档定义"如何检测和修复"。

## §1 核心理念

提示工程只能修复约 30% 的质量问题。剩余 70% 需要在生成后通过结构化分析管线处理：

```
AI 生成初稿 → 事实核查 → 语气一致性检测 → 语义重复检测 → GPT-ism 扫描 → 情感深度评分 → 修复建议
```

每一步独立检测、独立评分、独立输出修复建议，最后汇总为质量报告。

## §2 五步分析管线

### 2.1 事实核查 (Fact Check)

检测 AI 生成的文本中是否存在与已建立的世界观、角色设定、前文内容矛盾的事实。

| 检查维度 | 检查内容 | 数据源 |
|----------|----------|--------|
| 世界观一致 | 魔法体系/物理规则/社会制度是否与设定矛盾 | 世界观文档 |
| 角色一致 | 角色行为/能力/外貌是否与事实表矛盾 | `character-consistency-system.md` |
| 时间线一致 | 事件顺序/时间流逝是否与时间线矛盾 | 时间线文档 |
| 信息流一致 | 角色是否使用了尚未获得的信息 | 信息流追踪表 |
| 前文一致 | 是否与已写章节的内容矛盾 | 已完成章节摘要 |

**检测方法**：
```
输入: 生成的文本 + 已建立的事实库
输出: {
  "conflicts": [
    {
      "type": "character_ability",
      "text": "他用魔法治愈了伤口",
      "conflict": "角色设定中明确标注无魔法能力",
      "severity": "critical",
      "suggestion": "改为使用草药或他人帮助"
    }
  ]
}
```

### 2.2 语气一致性检测 (Tone Consistency)

检测文本中语气是否在段落间发生不合理的漂移。

| 检查维度 | 检测方法 | 问题信号 |
|----------|----------|----------|
| 整体语气 | 对比目标语气（如"庄重史诗"）与实际语气 | 从庄重滑向口语化 |
| 段落间语气 | 相邻段落的语气差异是否过大 | 动作段突然变成哲学反思段 |
| 角色语气 | 对话中角色语气是否与角色语言规则一致 | 谨慎角色突然用粗俗语言 |
| 叙述者语气 | 叙述者语气是否在全文保持一致 | 第三人称全知突然变成第二人称 |

**漂移检测阈值**：
- 段落间语气差异 > 0.7（0-1 标度）→ 标记为"语气断裂"
- 整体语气与目标语气偏差 > 0.5 → 标记为"语气偏移"

### 2.3 语义重复检测 (Semantic Repetition)

检测同一观点被换三种说法重复、用冗长冒充深度的模式。

| 检测类型 | 描述 | 示例 |
|----------|------|------|
| 观点循环 | 同一观点在3段内以不同措辞重复 | "他感到孤独"→"孤独如影随形"→"他是如此孤单" |
| 情感循环 | 同一情感标签反复出现而无递进 | "恐惧"在第2/4/6段各出现一次，无深化 |
| 描写循环 | 同一场景元素反复描写 | "月光"在一页内被提及5次 |
| 结构循环 | 句式或段落结构重复 | 连续3段以"他看着"开头 |

**检测方法**：
```
对每3个连续段落：
1. 提取核心语义（主旨/情感/描写对象）
2. 计算语义相似度
3. 相似度 > 0.8 且无递进 → 标记为"语义重复"
4. 建议：保留表达最强的一次，删除其余，或改为递进深化
```

### 2.4 GPT-ism 扫描 (AI Fingerprint Scan)

检测高频 AI 指纹词汇和句式结构。

**禁用词表**（高频 AI 指纹词）：

| 类别 | 禁用词 |
|------|--------|
| 动词 | delve, navigate, explore, unravel, demystify, elucidate, traverse |
| 名词 | landscape, tapestry, realm, frontier, paradigm, nexus, beacon |
| 形容词 | nuanced, intricate, multifaceted, profound, paramount, quintessential |
| 短语 | in the realm of, it's worth noting that, it's important to recognize |
| 中文等价 | 深入探讨、织就、画卷、领域、细微的、多层次的 |

**禁用句式**（文学性伪装指纹）：

| 指纹 | 描述 | 示例 | 修复方向 |
|------|------|------|----------|
| 纠正式重构 | "不是因为X，而是因为Y"的虚假二元 | "不是出于善意，而是出于解脱" | 直接陈述真实动机 |
| 碎片化反转 | 冒号+断句制造伪深刻感 | "她笑了笑：不是善意。是解脱。" | 合并为自然句 |
| 箴言式总结 | 段末独立成句的"智慧总结" | "有些伤口永远不会愈合" | 删除或融入叙事 |
| 过度变化对话标签 | 不用"说"，用 murmur/breathe/exhale | "他murmured""她breathed" | 默认用"说"，有理由时才变 |
| 感官清单化 | 嗅/听/视/触依次快速击中 | "空气中弥漫着花香，远处传来钟声，阳光刺眼，石墙冰冷" | 只保留最相关的1-2个感官 |
| 反思性动名词开头 | "看着...她意识到" | "看着他离开，她意识到自己错了" | 变换句式结构 |

### 2.5 情感深度评分 (Emotional Depth Score)

检测文本是"传达感受"还是"标签感受"。

| 评分维度 | 1分（标签化） | 3分（展示化） | 5分（沉浸化） |
|----------|---------------|---------------|---------------|
| 情感表达 | "一阵悲伤涌上心头" | "她的手停在半空，茶杯倾斜，茶水滴在膝上" | "茶杯倾倒的瞬间她没有去接——膝盖上的烫痕像是从很远的地方传来" |
| 感官细节 | 无或清单化 | 1-2个恰当的感官细节 | 感官细节与情感状态深度交织 |
| 行为反应 | 直接陈述情绪 | 通过行为间接展示 | 行为本身成为情感的隐喻 |
| 读者连接 | 提供信息 | 建立画面 | 建立共鸣 |

**评分阈值**：
- 平均分 < 2.0 → 标记为"情感扁平"，需重写关键段落
- 2.0-3.5 → 可接受，但有提升空间
- > 3.5 → 质量良好

## §3 质量报告模板

```markdown
## 生成质量报告 - 第 N 章

### 总评分: X.X / 5.0

### 事实核查
- 冲突数: X 处 (critical: X, major: X, minor: X)
- 详情: [列出每处冲突]

### 语气一致性
- 整体偏差: X.XX (目标: 庄重史诗, 实际: 接近但偏口语化)
- 断裂点: 第X段→第X段

### 语义重复
- 重复段落组: X 组
- 详情: [列出每组的重复内容]

### GPT-ism 扫描
- 禁用词命中: X 次 [列出]
- 禁用句式命中: X 次 [列出]

### 情感深度
- 平均分: X.X / 5.0
- 最弱段落: 第X段 (X.X分) - 原因: [分析]
- 最强段落: 第X段 (X.X分) - 原因: [分析]

### 修复建议（按优先级）
1. [critical] 修复事实冲突: ...
2. [major] 重写情感扁平段落: ...
3. [minor] 替换 GPT-ism 词汇: ...
```

## §4 修复流程

```
1. 修复 critical 级事实冲突（必须）
2. 修复 major 级语气断裂和情感扁平段落
3. 消除语义重复
4. 替换 GPT-ism 词汇和句式
5. 重新生成修复后的段落
6. 二次分析（只跑受影响的步骤）
7. 确认无 critical/major 问题后定稿
```

## §5 与其他文档的关系

- **`anti-ai-patterns.md`**: 定义 AI 写作的反模式清单，本文档是这些反模式的**检测和量化工具**。
- **`character-consistency-system.md`**: 事实核查步骤使用角色事实表作为数据源。
- **`creative-evaluation.md`**: 创意评估关注"好不好"，本文档关注"有没有 AI 味和一致性错误"。
- **`writing-techniques.md`**: 修复建议应参考写作技巧文档中的正确写法。
- **`dialogue-crafting.md`**: 对话中的 GPT-ism 扫描参考对话技巧文档中的正确对话写法。
