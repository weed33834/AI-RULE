# 网文创作指南 (Web Novel Guide)

> 网络小说与传统文学有显著差异——节奏更快、读者互动更强、连载特性决定了不同的创作策略。
> 来源：网文创作底层逻辑 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)、爆款网络小说写作方法论 [$TRAE_REF](https://m.book118.com/html/2026/0707/5314021011013241.shtm)、小说创作方法论 [$TRAE_REF](https://blog.csdn.net/longzekai/article/details/162316029)、番茄小说创作方法 [$TRAE_REF](https://fanqienovel.com/reader/7473424408346903065)。

## 核心定位

本文件专注于网络小说特有的创作技巧。如果用户的创作种子是"网文/连载/平台投稿"，则必须参考本文件。传统出版向的长篇/短篇可参考 `writing-techniques.md` 中的通用技法。

## 一、网文 vs 传统文学的关键差异

| 维度 | 传统文学 | 网络小说 |
|------|----------|----------|
| 节奏 | 张弛有度，可慢热 | 前 3 章必须抓住读者 |
| 章节长度 | 自由 | 2000-4000 字/章（平台标准） |
| 悬念 | 全书性 | 每章必须有钩子（章末悬念） |
| 角色 | 可慢热展开 | 第 1 章主角必须出场且有记忆点 |
| 主题 | 可含蓄 | 需通过情节展示，不可说教 |
| 互动 | 无 | 读者评论直接影响后续创作 |
| 更新 | 完稿出版 | 日更/周更，边写边发 |
| 篇幅 | 5-20 万字 | 50-300 万字 |

## 二、平台特性与节奏密度

> 不同平台有不同的读者群体和节奏期待。来源 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)。

| 平台 | 节奏要求 | 内容偏好 | 注意事项 |
|------|----------|----------|----------|
| 番茄小说 | 前期冲突极强 | 快节奏爽文、题材新颖 | 放大前期冲突强度 |
| 起点中文网 | 中等节奏 | 设定流、系统流、大世界观 | 男频为主，注重升级体系 |
| 晋江文学城 | 可慢但前期不能注水 | 人设细腻、情感描写 | 女频为主，加强人设细腻度 |
| 七猫/纵横 | 适度可慢 | 通俗易懂、情节驱动 | 前期必须快，后期适度可注水 |
| 知乎盐选 | 快准狠 | 短篇、高概念、反转 | 精准节奏，不拖泥带水 |
| 刺猬猫 | 中快 | 二次元、同人、脑洞 | ACG 文化元素 |

- 创作种子收集时，如果用户指定了目标平台，按平台特性调整节奏密度。

## 三、网文节奏控制体系

### 3.1 过山车定律
> 来源 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)。

- **三章一小高潮**：每 3 章左右有一个情绪爆点或情节转折。
- **十章一大转折**：每 10 章左右有一个重大剧情转折或阶段高潮。
- 情绪曲线不是直线上升——要有"虐后更甜""爽后更虐"的反差。

### 3.2 情绪价值曲线设计
```
第1章: 期待感（引出剧情，拉动读者情绪）
→ 第3章: 爽感（当众打脸/逆转/爆发）
→ 第7章: 虐感（发现真相/背叛/失去）
→ 第12章: 苏感（携手共度/黑暗中握手）
→ 终章: 圆满感（收束+伏笔回收）
```

### 3.3 每 3000 字法则
- **每 3000 字必有一个情绪爆点**——这是网文的黄金法则。
- 情绪爆点不一定是大事件——可以是一个眼神、一句话、一个动作。
- 爆点类型：爽感 / 虐感 / 苏感 / 惊讶感 / 期待感。

### 3.4 三阶段式大纲（长篇连载专用）
> 来源 [$TRAE_REF](https://m.book118.com/html/2026/0707/5314021011013241.shtm)。

| 阶段 | 字数 | 核心目标 | 节奏 |
|------|------|----------|------|
| **阶段一** | 0-10 万字 | 立人设、定节奏、铺垫世界观 | 偏快，让读者记住每个角色 |
| **阶段二** | 10-50 万字 | 核心冲突展开、支线交错、势力博弈 | 中等，张弛有度 |
| **阶段三** | 50 万+ | 终局推进、伏笔回收、收束 | 偏快，推向高潮 |

- 三幕剧撑不住 200 万字——三阶段式更适合长线连载。

## 四、网文人设特化技巧

### 4.1 矛盾特质动态标签
- 人设记忆点 = 矛盾特质（专业技能 vs 私人恐惧/弱点）。
- 示例：外科圣手却怕打雷 / 杀手之王却养流浪猫 / 商业天才却路痴。

### 4.2 活人感五要素
> 来源 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)。

| 要素 | 说明 | 示例 |
|------|------|------|
| 生理反应 | 紧张时的无意识小动作 | 摩挲袖口暗纹 |
| 私人习惯 | 独处时的固定行为 | 每月十五祭拜生母 |
| 语言指纹 | 场合切换时说话方式变化 | 长辈前用小名，外人前自称"本姑娘" |
| 认知局限 | 角色的认知盲区 | 始终认错童年恩人 |
| 物质依赖 | 随身物品有隐藏用途 | 药瓶实为解百毒剂 |

### 4.3 致命缺陷设计
- 人设必须存在"可被利用的致命缺陷"——这是后续情节的驱动力。
- 缺陷类型：过度信任 / 执念 / 恐惧 / 虚荣 / 软肋（某人）。
- 缺陷被利用时产生最大戏剧张力。

## 五、网文情节特化技巧

### 5.1 意外事件四幕剧
> 来源 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)。

1. **起**（铺垫）：利用场景共识制造反差（大婚夜发现合卺酒被下毒）
2. **承**（转折）：引爆人设矛盾（毒药出自女主独家配方）
3. **转**（升级）：情感痛点打击（下毒者身上有女主生母遗物）
4. **合**（余波）：关系重新洗牌（男主暗中替换毒酒产生认知冲击）

### 5.2 紧迫感营造三技巧
1. **时间沙漏**：倒计时压力（"七日内找到解药"）
2. **道德困境**：选择有代价（"救 A 会暴露自己 B 身份"）
3. **信息差**：读者知道角色不知道的事（上帝视角焦虑）

### 5.3 转折同时推动双线
- 好的转折同时推动感情线和事业线——不只推动一条。
- 检验：这个转折是否让角色的感情关系和事业/目标都发生了变化？

### 5.4 10% 未解释伏笔原则
- 保留 10% 的伏笔不回收——为续作/番外留空间。
- 但这 10% 必须是"有趣的未解之谜"，不是"忘了回收的线索"。

## 六、网文标签组合策略

> 来源 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)。

### 基础热门标签
- **女频**：先婚后爱 / 追妻火葬场 / 重生之XX王妃 / 白切黑男主 / 扮猪吃老虎 / 团宠 / 逆袭
- **男频**：系统流 / 无限流 / 赘婿逆袭 / 重生修仙 / 都市异能 / 基地建设 / 种田流

### 新兴复合标签
- 玄学直播 + 豪门恩怨 + 真假千金
- 机甲女将军 + 星际联姻 + 古地球
- 破产千金 + 元宇宙创业 + 无 CP
- 丧尸围城 + 产科医生 + 人类命运共同体

### 标签组合原则
- 2-3 个基础标签 + 1 个差异化标签。
- 差异化标签是"为什么读者要看你的而不是别人的"。
- 创作种子收集时，如果用户指定了标签组合，按标签特性调整情节侧重。

## 七、网文检验清单

- [ ] 每 3000 字必有一个情绪爆点
- [ ] 人设存在可被利用的致命缺陷
- [ ] 意外转折同时推动感情线和事业线
- [ ] 保留 10% 未解释伏笔（为续作留空间）
- [ ] 前 3 章主角已出场且有记忆点
- [ ] 每章末尾有钩子
- [ ] 三章一小高潮，十章一大转折
- [ ] 节奏密度匹配目标平台

## 八、AI 辅助网文创作的特殊注意事项

### 8.1 日更场景下的上下文管理
- 日更模式下，每次会话开始时注入：前 3 章摘要 + 当前角色状态 + 未回收伏笔列表。
- 本仓库实现：`.ai-memory/session-summary.md` + `.ai-memory/plot-threads.md`。

### 8.2 防止套路化
- 网文类型套路比传统文学更强——AI 更容易掉入套路。
- 参照 `anti-ai-patterns.md` 的 35 项检查清单，重点检查第 1 类（叙事结构）和第 7 类（平台检测）。

### 8.3 读者反馈整合
- 网文连载中读者评论可能影响后续创作方向。
- 收到读者反馈后，更新 `.ai-memory/author-profile.md` 中记录读者偏好。
- 但核心情节走向不应被读者评论绑架——作者保持创作主导权。

### 8.4 防"注水"
- 网文平台 AI 检测会标记"无意义灌水"段落。
- 参照 `anti-ai-patterns.md` 第 32 项（无意义灌水）的规避策略。
- 每个场景必须有叙事目的——删掉不影响故事的场景。

## 核心原则
- 网文的本质是"情绪价值"——读者来网文寻找的是情感体验，不是文学分析。
- 节奏是网文的命脉——前 3 章定生死，每章末尾定留存。
- 人设是网文的核心资产——读者追的不是情节，是角色。
- 平台特性决定节奏密度——同一个故事在不同平台需要不同的节奏处理。
