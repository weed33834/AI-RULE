# 修订策略 (Revision Strategy)

> 本文档提供系统化的小说修订方法论：三层修订体系、修订优先级、常见修订模式。
> 初稿是"把故事写出来"，修订是"把故事写好"。
> 与 `anti-ai-patterns.md` 互补——反面清单识别问题，本文档提供修复方法。
> 与 `creative-evaluation.md` 互补——评估发现缺陷，修订解决缺陷。

## §1 修订原则

- **先大后小**：先修结构再修句子。在烂结构上修句子是浪费。
- **一次一维**：每次修订只关注一个维度，不混着改。
- **冷眼重读**：修订前至少隔一天再读，带着"读者视角"而非"作者视角"。
- **不改就删**：犹豫要不要改的地方，大概率该删。
- **保留原稿**：修订前备份原稿，便于对比和回退。

## §2 三层修订体系

```
┌─────────────────────────┐
│ 第一层: 结构修订 (Structural)  │  故事骨架是否成立
├─────────────────────────┤
│ 第二层: 句段修订 (Line Edit)   │  表达是否精准、有力
├─────────────────────────┤
│ 第三层: 校对润色 (Proofread)   │  错别字、标点、格式
└─────────────────────────┘
```

### 2.1 第一层：结构修订

在修改任何句子之前，先确认故事结构是否成立。

| 检查维度 | 检查内容 | 常见问题 |
|----------|----------|----------|
| 情节逻辑 | 因果链是否成立？每个事件是否有合理的前因？ | 无因之果、强行转折 |
| 角色弧光 | 主角是否有变化？变化是否有足够的事件驱动？ | 角色静止、突变无铺垫 |
| 节奏控制 | 高潮低谷是否交替？是否有冗长的平缓段？ | 一路平铺、高潮堆叠 |
| 伏笔回收 | 所有伏笔是否回收？回收是否自然？ | 遗忘伏笔、强行回收 |
| 主题表达 | 故事是否传达了主题？是否说教？ | 主题模糊、强行点题 |
| 视角一致 | 叙事视角是否统一？切换是否有理由？ | 视角混乱、随意切换 |
| 信息密度 | 关键信息是否前置？非关键信息是否冗余？ | 重要信息后置、废话前置 |

**结构修订的操作方式**：
1. 画出全篇情节曲线（章节 × 紧张度）。
2. 标注每个伏笔的埋设点和回收点。
3. 列出每个主要角色的状态变化时间线。
4. 检查是否有"删了也不影响故事"的章节/段落。

### 2.2 第二层：句段修订

结构确认后，逐段逐句打磨表达。

| 检查维度 | 检查内容 | 常见问题 |
|----------|----------|----------|
| AI 味检测 | 是否有 35 项 AI 写作毛病中的表现？ | 参见 `anti-ai-patterns.md` |
| 展示vs讲述 | 是否用行为/感官展示而非直接告知？ | "他很伤心" → 展示行为 |
| 对话个性 | 不同角色的说话方式是否有区分度？ | 所有角色一个腔调 |
| 感官沉浸 | 是否调动了视觉/听觉/嗅觉/触觉？ | 只有视觉描写 |
| 节奏变化 | 句子长短是否有变化？段落密度是否有起伏？ | 全篇句式统一 |
| 词语精准 | 动词是否精准？形容词是否必要？ | 弱动词+堆砌形容词 |
| 情感共鸣 | 读者是否能感受到角色的情感？ | 情感描述浮于表面 |

**句段修订的操作方式**：
1. 朗读——耳朵比眼睛更敏感，别扭的地方读出来会卡。
2. 逐段检查：这段的目的是什么？是否达到了？有没有更短的方式？
3. 标记所有"感觉不对但说不出哪里不对"的段落，重点突破。

### 2.3 第三层：校对润色

最后一遍，修文字层面的硬伤。

| 检查项 | 检查方法 | 工具辅助 |
|--------|----------|----------|
| 错别字 | 逐字阅读 | 拼写检查 |
| 标点 | 中文标点/英文标点混用、省略号格式 | 搜索替换 |
| 人名地名一致性 | 全文搜索所有出现位置 | Ctrl+F 逐个确认 |
| 时间线一致性 | 列出所有时间点对比 | `plot-threads.md` |
| 数字格式 | "三天" vs "3天" 是否统一 | 统一为中文数字或阿拉伯数字 |
| 格式排版 | 对话格式、分段长度、空行 | 统一样式 |

## §3 修订优先级

当时间有限时，按以下优先级修订：

| 优先级 | 修订内容 | 投入占比 | 理由 |
|--------|----------|----------|------|
| P0 | 情节逻辑错误 | 40% | 逻辑不通，其他再好也白搭 |
| P1 | 角色行为不一致 | 25% | 读者最容易被角色违和感出戏 |
| P2 | AI 味和模板化表达 | 20% | 直接影响阅读体验 |
| P3 | 节奏问题（拖沓/仓促） | 10% | 影响整体观感但非致命 |
| P4 | 文字层面的瑕疵 | 5% | 最后统一处理 |

## §4 常见修订模式

### 4.1 "讲述转展示" 修订

```
修订前（讲述）: 李明很紧张，他害怕被发现。

修订后（展示）: 李明的手不自觉地攥紧了衣角，指节发白。他的目光在门口和窗户之间来回游移，像一只被逼到角落的猫。
```

**修订步骤**：
1. 找到所有"感到/觉得/很/非常 + 情绪词"的句子。
2. 问自己：这个情绪通过什么行为可以表现出来？
3. 用具体的动作、表情、生理反应替代直接陈述。

### 4.2 "对话去统一化" 修订

```
修订前（统一化）:
  张三说："我们需要尽快出发。"
  李四说："是的，时间不多了。"
  王五说："我同意，我们走吧。"

修订后（个性化）:
  张三敲了敲桌子："走。现在。"（简洁直接，军人风格）
  李四推了推眼镜："还有二十分钟，来得及...大概。"（犹豫，学者风格）
  王五已经在门口穿鞋了。（行动派，不说话直接做）
```

**修订步骤**：
1. 列出每个角色的说话特征（语速、用词、句式、口头禅）。
2. 逐段检查对话是否符合角色特征。
3. 能用行动代替的对话，改为行动。

### 4.3 "删减冗余" 修订

| 冗余类型 | 表现 | 修订方法 |
|----------|------|----------|
| 重复信息 | 同一件事用不同方式说了 3 遍 | 保留最有力的一次 |
| 无效过渡 | "第二天早上，阳光照进窗户..." | 直接切到有内容的场景 |
| 过度描写 | 用 200 字描写一个不重要的路人的穿着 | 压缩到 1 句或删除 |
| 解释过度 | 把伏笔的含义解释给读者听 | 让伏笔自己说话 |
| 情绪叠加 | "他既悲伤又愤怒，同时感到绝望和无力" | 选一个主情绪，用行为展示 |

### 4.4 "节奏调整" 修订

```
节奏诊断：
章节1 ████████░░ 8/10 紧张度（开头过高，后面无空间上升）
章节2 █████░░░░░ 5/10（下降太快）
章节3 ██████░░░░ 6/10
章节4 ██████████ 10/10（高潮）
章节5 █████░░░░░ 5/10（高潮后急降，缺少余韵）

修订方向：
章节1 → 降到 6/10（留出上升空间）
章节2 → 升到 4/10（制造低谷反差）
章节3 → 升到 7/10（稳步上升）
章节4 → 保持 10/10（高潮）
章节5 → 升到 7/10（余韵渐降，不要急降）
```

## §5 章末自检清单

每章修订完成后，对照以下清单确认：

| # | 检查项 | ✅/❌ |
|---|--------|------|
| 1 | 本章是否推进了主线/支线？ | |
| 2 | 角色行为是否符合其性格和当前状态？ | |
| 3 | 是否有未回收的伏笔需要提醒？ | |
| 4 | 是否有 AI 写作毛病的典型表现？ | |
| 5 | 对话是否有个性区分度？ | |
| 6 | 结尾是否有钩子让读者想看下一章？ | |
| 7 | 字数是否在目标范围内？ | |

## §6 修订工作流

```
1. 完成初稿 → 冷却 24 小时
2. 第一遍通读（只读不改）→ 标记"感觉不对"的地方
3. 结构修订 → 修情节逻辑、角色弧光、节奏
4. 句段修订 → 逐段打磨表达
5. 校对润色 → 修文字硬伤
6. 章末自检 → 对照清单确认
7. 提交用户审阅
```
