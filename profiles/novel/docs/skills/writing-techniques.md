# 写作技法库 (Writing Techniques Library)

> 整合了 AI 写作提示词工程、专业编剧理论和网文创作实践中的最佳写作技法。
> 来源：50 个 AI 小说写作模板 [$TRAE_REF](https://www.fictionai.pro/blog/ai-writing-prompts-for-fiction-50-templates-that-work-1)、瀑布法长篇工程指南 [$TRAE_REF](https://www.glbgpt.com/hub/how-to-use-chatgpt-to-write-a-full-novel-the-2026-engineering-guide/)、网文创作底层逻辑 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)、爆款网络小说写作方法论 [$TRAE_REF](https://m.book118.com/html/2026/0707/5314021011013241.shtm)、小说创作方法论 [$TRAE_REF](https://blog.csdn.net/longzekai/article/details/162316029)。

## 一、瀑布法长篇工程（Waterfall Method）

> 来源：GlobalGPT 2026 工程指南 [$TRAE_REF](https://www.glbgpt.com/hub/how-to-use-chatgpt-to-write-a-full-novel-the-2026-engineering-guide/)。
> 解决"AI 长篇创作中逻辑断裂、文风突变、记忆丢失"的核心问题。

### 四步瀑布流程

| 步骤 | 产出 | 不可跳过 |
|------|------|----------|
| 1. 高概念提案 | 2-3 句话概括：主角是谁、想要什么、什么阻止他 | ✅ |
| 2. 完整大纲 | 2 页故事摘要，含开头-中间-结局 | ✅ |
| 3. 章节节拍 | 每章 3-5 个 beat（具体发生什么） | ✅ |
| 4. 正文生成 | 按 beat 逐章写正文 | ✅ |

- **铁律**：步骤 4 之前不写任何正文。先有路线图，再有旅程。
- **导演心态**：作者是导演，AI 是编剧助理——你提供愿景，AI 负责展开。

### 逆向写作法（Reverse Writing）
- 先写封底简介（假设书已出版），设计 killer ending 和 hook。
- 再倒推：角色是怎么走到这个结局的？
- 优势：确保故事"去到某个令人兴奋的地方"，而非漫无目的。

### 系列圣经（Series Bible）
> 与本仓库的 `.ai-memory/` 系统对应。

- 主文档：角色名/年龄/外貌/说话风格、世界规则、每章摘要。
- 递归摘要技巧：写新章前先注入"前 3 章摘要 + 系列圣经"，让 AI 刷新记忆。
- 本仓库实现：`.ai-memory/creative-blueprint.md` + `.ai-memory/characters/` + `.ai-memory/worldbuilding.md`。

### 多标签并行写作（Multi-Tab Strategy）
- 不同章节/场景在不同会话中并行写作，每个会话只聚焦一个场景。
- 使用"滚动摘要"（Rolling Summary）保持连贯：写完一个场景，让 AI 生成 3 句摘要，粘贴到下一个会话。
- 本仓库实现：每个场景完成后更新 `.ai-memory/session-summary.md`。

## 二、角色深度开发模板（6 项）

> 来源：50 个 AI 写作模板 [$TRAE_REF](https://www.fictionai.pro/blog/ai-writing-prompts-for-fiction-50-templates-that-work-1)。

### 2.1 深度角色档案
- 物理外观 + 一个标志性特征
- 核心性格 + 致命缺陷
- 童年创伤（塑造其行为的根源）
- 秘密欲望（对他人隐瞒的）
- 语言指纹（口头禅/句式/紧张时的变化）
- 压力下的变化模式

### 2.2 角色矛盾构建器
- 好角色包含矛盾——既 A 又（A 的对立面）。
- 矛盾共存的方式、童年根源、日常表现、触发冲突的具体场景。
- 示例：外科圣手却怕打雷（专业技能 vs 私人恐惧）。

### 2.3 角色弧光规划器（5 关键时刻）
1. 开场状态：错误信念在行动中展示。
2. 激励事件：挑战错误信念。
3. 中点：瞥见真相但拒绝。
4. 至暗时刻：错误信念彻底失败。
5. 高潮：拥抱真相并转变。

### 2.4 对话声音区分
- 4 个角色同一场戏，去掉对话标签后读者应能猜出说话者。
- 差异维度：词汇量、句式长度、口头禅、打断方式。

### 2.5 角色压力测试
- 设定高压场景，写角色的内心独白（200 字）+ 外部行动 + 做出的决定。
- 决定应"揭示真实性格"——压力下的选择才是角色。

### 2.6 角色采访法
- 以角色身份接受采访，完全保持角色——包括回避、谎言、真话。
- 用于深度理解角色的内在逻辑。

## 三、情节与结构模板（8 项）

### 3.1 三幕结构构建器
- 每幕：核心故事问题 + 3-4 个主要情节节点 + 主角情感旅程 + 赌注升级。
- 高潮回答核心问题——意外但必然。

### 3.2 反转生成器（5 个选项）
- 被已有故事元素伏笔
- 重新语境化读者以为已知的一切
- 意外但必然
- 大幅提升赌注
- 在遵守类型惯例的同时颠覆期待

### 3.3 场景价值分析器
- 每个场景必须通过 4 项中至少 2 项：推进情节 / 揭示角色 / 增加张力 / 提供必要信息。
- 如果全不通过——删掉这个场景。

### 3.4 支线整合器
- 支线须：映射或对比主题 / 与主线交叉至少 3 次 / 有自己的迷你弧光 / 结局影响主线高潮。

### 3.5 张力升级序列（5 重递进）
1. 障碍比上一个更严重
2. 角色必须牺牲某物
3. 消除一个简单的解决方案
4. 揭示新信息
5. 推向不可能的选择——最终障碍看起来无法存活

### 3.6 开篇钩子生成器
- 5 种开篇方式：动作切入 / 悬念切入 / 声音切入 / 场景切入 / 中间切入（in medias res）。
- 标准：立即制造好奇 / 建立声音和基调 / 暗示核心冲突 / 让读者迫切想知道后续。

### 3.7 中点镜像
- 中点场景与结局镜像但结果相反——结局是胜利，中点是虚假胜利转为失败。
- 主题关联：中点设置结局。

### 3.8 赌注三层定义
| 层次 | 内容 | 示例 |
|------|------|------|
| 个人 | 角色情感上失去什么 | 失去爱人信任 |
| 公共 | 他人失去什么 | 城市被毁 |
| 哲学 | 什么真理死去 | 正义不存在 |

## 四、世界观构建模板（6 项）

### 4.1 场景即角色
- 把场景当作角色写：性格（如何对待居民）、历史、秘密、日夜/季节变化、不同角色的不同体验。

### 4.2 感官调色板
- 为每个地点建立感官清单：5 种独有声响 / 5 种气味（好闻和难闻）/ 5 种触感 / 5 个视觉细节 / 3 种味觉。
- 禁止泛泛描写——每个细节必须具体且引发联想。

### 4.3 魔法/科技体系构建器
- 来源/燃料、明确限制和代价、社会如何适应、谁有权限谁没有、一种尚未被发现的用法。
- **限制比能力更有趣**。

### 4.4 文化生成器
- 核心价值观与禁忌、日常仪式与习俗、对外人态度、艺术与叙事传统、一个看似奇怪但合乎历史逻辑的习俗。

### 4.5 经济世界观
- 什么资源有价值及为什么、财富分配、普通人职业、贸易运作、经济矛盾可创造的冲突。

### 4.6 冰山法则
- 知道的比写出来的多十倍。用这些模板发展深度理解，但只展示服务于故事的部分。

## 五、网文创作底层逻辑（9 项）

> 来源：网文创作底层逻辑 [$TRAE_REF](https://www.bilibili.com/opus/1100801427974914069)。

### 5.1 爆款三角模型
| 维度 | 内容 |
|------|------|
| 情感共鸣核 | 精准打击读者深层欲望（安全感/征服欲/救赎欲） |
| 节奏控制轴 | 三章一小高潮 + 十章一大转折的"过山车定律" |
| 人设记忆点 | 具有矛盾特质的"动态标签"（如：外科圣手却怕打雷） |

### 5.2 人设设置模板
- **基础身份**：具有阶层辨识度（如"落魄贵族庶女"）
- **核心矛盾**：制造内在张力（如"精通毒术却需伪装小白兔"）
- **记忆点**：可视化特征（如"手腕朱砂胎记"）
- **缺陷**：为后续转折埋伏笔（如"过度信任童年恩人"）
- **动态成长性**：确保人设可进化

### 5.3 意外事件设计四幕剧
1. **起**（铺垫）：利用场景共识制造反差
2. **承**（转折）：引爆人设矛盾
3. **转**（升级）：情感痛点打击
4. **合**（余波）：关系重新洗牌

### 5.4 痛点与爆点设计
- 阶层痛点：身份导致的"被系统性羞辱"
- 情感爆点：角色在关键时刻的逆转行为
- 意外机制：当读者期待 A 时，发生 B（制造错愕感）

### 5.5 紧迫感营造三技巧
1. **时间沙漏**：倒计时压力（"必须在七日内找到解药"）
2. **道德困境**：选择有代价（"救 A 会暴露自己 B 身份"）
3. **信息差**：读者知道角色不知道的事（上帝视角焦虑）

### 5.6 活人感塑造五要素
| 要素 | 说明 | 示例 |
|------|------|------|
| 生理反应 | 紧张时的无意识小动作 | 摩挲袖口暗纹 |
| 私人习惯 | 独处时的固定行为 | 每月十五祭拜生母木牌 |
| 语言指纹 | 不同场合的说话方式差异 | 长辈前用小名，外人前用"本姑娘" |
| 认知局限 | 角色的认知盲区 | 始终认错童年恩人（关键伏笔） |
| 物质依赖 | 随身携带的物件 | 药瓶实为解百毒剂 |

### 5.7 情绪价值曲线
```
第1章: 期待感 → 第3章: 爽感 → 第7章: 虐感 → 第12章: 苏感 → 终章: 圆满感
```
- 每 3000 字必有一个情绪爆点。
- 情绪曲线不是线性的——要有起伏，要有"虐后更甜"的反差。

### 5.8 三阶段式大纲骨架（长篇连载专用）
> 来源：爆款网络小说写作方法论 [$TRAE_REF](https://m.book118.com/html/2026/0707/5314021011013241.shtm)。

| 阶段 | 字数 | 目标 | 节奏 |
|------|------|------|------|
| 阶段一 | 0-10 万字 | 立人设、定节奏、铺垫世界观 | 偏快 |
| 阶段二 | 10-50 万字 | 核心冲突展开、支线交错 | 中等 |
| 阶段三 | 50 万+ | 终局推进、伏笔回收、收束 | 偏快 |

- 三幕剧撑不住 200 万字的长线连载——三阶段式更适合。

### 5.9 网文检验清单
- [ ] 每 3000 字必有一个情绪爆点
- [ ] 人设存在可被利用的致命缺陷
- [ ] 意外转折同时推动感情线和事业线
- [ ] 保留 10% 未解释伏笔（为续作留空间）

## 六、AI 协作高级技巧（6 项）

> 来源：GlobalGPT 工程指南 [$TRAE_REF](https://www.glbgpt.com/hub/how-to-use-chatgpt-to-write-a-full-novel-the-2026-engineering-guide/)。

### 6.1 Few-Shot 风格注入
- 写正文前，先给 AI 提供 300 字你自己的写作样本（或你喜欢的作者风格）。
- 提示词："分析这段文字的句式长度、词汇选择和语调。然后用完全相同的风格写下一个场景。"
- 本仓库实现：通过 `.ai-memory/author-profile.md` 持久化风格参数。

### 6.2 逻辑压力测试（Roast My Plot）
- 写正文前，先把大纲交给逻辑模型检查："扮演严格的逻辑教授，阅读这个大纲，找出每一个逻辑错误、时间线错误和角色做的愚蠢决定。"
- 本仓库实现：通过 reviewer 子智能体的"内部一致性检查"实现。

### 6.3 慢动作描写法
- AI 倾向于快进到结局。强制它写"极慢动作"："聚焦感官细节——雨的气味、墙的粗糙质感、脚步声。不要总结动作。逐拍（beat-by-beat）写。"
- 与 `anti-ai-patterns.md` 第 8 项（情感反应过快）配合使用。

### 6.4 Save the Cat 节拍表
- 15 个标准节拍：开场画面 / 主题陈述 / 催化剂 / 争辩 / 进入第二幕 / B 故事 / 趣味与游戏 / 中点 / 坏人逼近 / 一切尽失 / 灵魂黑夜 / 进入第三幕 / 高潮 / 终场画面。
- 用节拍表确保"激动人心的部分出现在正确的位置"。

### 6.5 滚动摘要技术
- 每写完一个场景，让 AI 用 3 句话总结。
- 把摘要粘贴到下一个会话——故事无缝衔接。
- 本仓库实现：`.ai-memory/session-summary.md`。

### 6.6 模型分工策略
| 任务 | 推荐模型类型 | 原因 |
|------|-------------|------|
| 大纲/逻辑检查 | 推理型模型（DeepSeek R1 / o1） | 逻辑强 |
| 正文生成 | 创意型模型（Claude Sonnet） | 文学性强 |
| 全文一致性检查 | 大上下文模型（Gemini Pro） | 记忆容量大 |

- 本仓库不绑定特定模型——但在 `author-profile.md` 中记录模型偏好。

## 核心原则
- 技法服务于故事——不是炫技，而是让故事更有力。
- 不同类型需要不同技法——网文节奏和文学小说节奏完全不同。
- AI 协作的关键是"导演心态"——你提供愿景，AI 展开细节。
- 先有路线图再有旅程——瀑布法的核心是"不跳步"。
