# 创作瓶颈突破 (Creative Block Breaker)

> 本文档提供系统化的创作瓶颈突破方法：6 种瓶颈类型诊断、15+ 突破技术、紧急救场方案。
> 创作瓶颈不是"没有灵感"，而是"卡在了某个具体的点上"。
> 与 `revision-strategy.md` 互补——修订解决"写得不好"，本文档解决"写不下去"。

## §1 核心理念

- **瓶颈是具体的**：不是"我写不出来"，而是"我不知道这个角色在这个场景该怎么反应"。
- **先诊断再开方**：不同类型的瓶颈需要不同的突破方法。
- **允许写烂**：先写出来再改。烂稿 > 空稿。
- **切换而非硬磕**：卡住时换方法，不要用同样的方式反复尝试。

## §2 六种瓶颈类型

| # | 瓶颈类型 | 表现 | 根因 | 对应突破法 |
|---|----------|------|------|-----------|
| 1 | 情节瓶颈 | "不知道接下来该发生什么" | 缺乏冲突驱动 | §3.1 目标倒推法 |
| 2 | 角色瓶颈 | "角色不活了，像个木偶" | 角色动机不清晰 | §3.2 角色访谈法 |
| 3 | 场景瓶颈 | "不知道怎么写这个场景" | 缺乏感官锚点 | §3.3 感官锚定法 |
| 4 | 对话瓶颈 | "对话写不出/写不好" | 角色声音不区分 | §3.4 声音切换法 |
| 5 | 情感瓶颈 | "写不出情感冲击力" | 情感距离太远 | §3.5 情感记忆法 |
| 6 | 开头瓶颈 | "第一章怎么都写不好" | 不确定故事基调 | §3.6 破冰启动法 |

## §3 突破技术

### 3.1 目标倒推法（情节瓶颈）

从故事终点倒推到现在：

```
故事终点: 主角击败反派，拯救世界
  ↑ 需要什么条件？
主角需要找到反派的弱点
  ↑ 怎么找到？
需要一个知道反派过去的人来告知
  ↑ 谁知道？
反派的 former 盟友，现在隐居
  ↑ 主角怎么找到这个人？
通过一条线索 → 去某个地方 → 遇到这个人
  ↑ 现在就可以写这段了
```

**操作步骤**：
1. 写下这个 arc 的终点（角色要达成什么）。
2. 问"达成这个需要什么条件？"
3. 对每个条件再问"这个条件怎么满足？"
4. 一直倒推到当前位置。
5. 从倒推路径中选择最有趣的一条。

### 3.2 角色访谈法（角色瓶颈）

假装采访你的角色，让角色"自己说话"：

```
采访问题清单：
Q: 你现在最想要什么？
Q: 你最害怕什么？
Q: 你对这个场景中的其他人有什么看法？
Q: 如果你可以选择，你会怎么做？为什么不能这么做？
Q: 你现在隐瞒了什么？
Q: 什么事会让你改变主意？
Q: 你睡前最后一个念头是什么？
```

**操作步骤**：
1. 以角色的第一人称回答所有问题（不要以作者视角替角色回答）。
2. 如果角色"不想回答"某个问题，那个问题背后就是故事的核心冲突。
3. 用角色的回答驱动他在场景中的行动。

### 3.3 感官锚定法（场景瓶颈）

从一个感官细节开始，展开整个场景：

```
第一步: 选一个感官锚点
  → "雨后泥土的气味"

第二步: 扩展到其他感官
  → 视觉: 泥泞的路面上有深浅不一的脚印
  → 听觉: 远处传来狗吠，近处有水滴从屋檐落下
  → 触觉: 潮湿的空气贴在皮肤上，衣服黏糊糊的
  → 味觉: 嘴里有铁锈的味道（紧张时咬了嘴唇）

第三步: 加入角色反应
  → 主角皱了皱鼻子——这个气味让他想起了战场

第四步: 加入行动
  → 他沿着脚印追了过去
```

**操作步骤**：
1. 闭眼想象这个场景，抓住第一个跳出来的感官细节。
2. 用"五感扩展表"补全其他感官。
3. 角色对这个环境有什么反应？
4. 角色会在这个环境中做什么？

### 3.4 声音切换法（对话瓶颈）

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1 | 列出对话参与者的说话特征 | 老兵：短句、军事术语、不解释；学者：长句、引经据典、爱解释 |
| 2 | 写一段"标准版"对话 | A说... B说... |
| 3 | 用每个角色的声音重写 | 老兵："走。" 学者："我认为我们应该先评估一下风险..." |
| 4 | 加入角色专属的口头禅/语气词 | 老兵句尾加"听到没有"；学者喜欢说"换言之" |
| 5 | 加入非语言交流 | 老兵敲桌子；学者推眼镜 |

### 3.5 情感记忆法（情感瓶颈）

用真实情感体验为角色注入情感：

```
角色情感: 失去至亲的悲痛

情感记忆挖掘:
- 你经历过类似的失去吗？（不一定是死亡——分手、搬家、宠物离世）
- 那个时刻你的身体有什么反应？（胸口发紧、手脚冰凉、耳鸣）
- 你做了什么？（发呆、反复看手机、机械地做日常的事）
- 什么细节最让你崩溃？（看到ta的杯子还在原位）

将真实体验映射到角色:
→ 不写"他很悲伤"，写他机械地把两人份的早餐做完后对着空椅子发呆
```

**操作步骤**：
1. 找到与角色情感最接近的个人记忆。
2. 回忆当时的身体感受（不是想法，是身体感受）。
3. 回忆当时最触动你的细节（越小越好）。
4. 把这些感受和细节映射到角色的情境中。
5. 只写身体感受和小细节，不直接写情绪词。

### 3.6 破冰启动法（开头瓶颈）

当第一章怎么都写不好时：

| 策略 | 操作 | 适用 |
|------|------|------|
| 从中间开始 | 跳过开头，直接写第二个场景，回头再补开头 | 开头卡住但后面有想法 |
| 写对话开头 | 用一段引人入胜的对话作为第一章第一句 | 不知如何切入场景 |
| 写感官开头 | 用一段环境描写建立氛围 | 需要先建立世界观基调 |
| 写动作开头 | 用一个动作场景直接切入 | 需要快速抓住注意力 |
| 写"那一天" | "那一天，..."作为开头，回忆体切入 | 需要建立时间距离感 |
| 跳过开头 | 先写完全文，最后回来写开头——那时你已经知道故事的全貌 | 一直卡在开头 |

## §4 紧急救场方案

当创作熔断触发（同一段重写 3 次仍不满意）时：

### 4.1 三选一法

输出 3 个完全不同的方向，让用户选择：

```
当前瓶颈: [描述卡住的地方]

方向 A: [激进方案 — 改变角色/情节走向]
  - 优势: 出人意料
  - 风险: 可能偏离大纲

方向 B: [保守方案 — 在现有框架内微调]
  - 优势: 安全可控
  - 风险: 可能不够有趣

方向 C: [折中方案 — 融合 A 和 B]
  - 优势: 兼顾创新和安全
  - 风险: 可能两头不讨好
```

### 4.2 "如果...会怎样" 法

列出 5 个"如果"，打破思维定势：

```
如果主角在这个场景做出了错误的选择会怎样？
如果这个角色突然消失了会怎样？
如果这个场景的真相和读者以为的完全不同会怎样？
如果时间倒流，角色知道了结局会怎样？
如果这个场景从另一个角色的视角写会怎样？
```

### 4.3 换人法

换一个角色来叙述这个场景：

```
原视角: 主角视角 — "我走进了酒馆..."
换视角: 酒馆老板视角 — "门被推开的时候，我正在擦杯子。进来的那个人..."
换视角: 旁观者视角 — "角落里的暗卫注意到，一个陌生人在门口犹豫了三秒..."
```

不同视角会自然带出不同的信息和节奏。

## §5 预防瓶颈的习惯

| 习惯 | 说明 |
|------|------|
| 大纲缓冲 | 永远比当前写作位置多准备 2-3 章的大纲 |
| 灵感记录 | 随时记录灵感碎片（一个画面、一句对话、一个设定），卡住时翻看 |
| 多线并行 | 主线卡住时切到支线写，保持产出节奏 |
| 角色档案维护 | 定期更新角色卡，确保角色始终"活着" |
| 伏笔台账 | 维护伏笔台账，卡住时看看有没有可以回收的伏笔 |

- **`creative-evaluation.md`**: 评估发现的问题可能导致瓶颈——比如评估发现角色弧光不清晰，修改时就会遇到角色瓶颈。
- **`web-novel-guide.md`**: 网文连载中遇到瓶颈时的特殊策略（如水一章过渡、切视角等）。
