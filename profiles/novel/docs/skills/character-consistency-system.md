# 角色一致性五文档体系 (Character Consistency System)

> 本文档定义 AI 小说创作中维护角色一致性的五文档并行维护体系。
> 与 `character-design.md` 互补——后者定义角色如何设计，本文档确保设计在长篇创作中不漂移。
> 与 `timeline-tracking.md` 互补——后者追踪故事时间线，本文档追踪角色状态线。

## §1 核心理念

AI 辅助长篇创作的最大质量瓶颈不是文采，而是**一致性**。研究数据表明：原始 AI 输出的角色一致性仅 47%，经人工编辑后可达 79%，差距最大处在于"被遗忘的细节"和"性格漂移"。

解决方案不是更好的提示词，而是**系统**——与手稿并行维护五份结构化文档，作为每次生成的上下文显式注入。

## §2 五份并行文档

### 2.1 角色事实表 (Character Fact Sheet)

为每个主要角色维护一份结构化事实表，写作前必查。

```markdown
## 角色事实表：[角色名]

### 基础信息
- 全名/别名/绰号
- 年龄/生日
- 外貌：身高、体型、发色/眼色、显著特征（疤痕/胎记/习惯动作）
- 生理限制：过敏、慢性病、旧伤、左撇子/右撇子

### 能力与局限
- 专业技能（明确等级：精通/熟练/入门）
- 语言能力
- 战斗/特殊能力（如有）
- 明确不能做的事（叙事中绝不违反）

### 习惯与行为模式
- 语言习惯：口癖、句长倾向、比喻域、回避词
- 行为习惯：紧张时的小动作、思考时的姿态
- 饮食/睡眠/穿着偏好

### 语言规则（3-5 条具体规则，非形容词）
- 示例："从不直接道歉——用幽默转移"
- 示例："用问题回答问题，不主动提供信息"
- 示例："提到家人时语气变平，语速放慢"

### 禁区
- 绝不会做的事（道德底线/恐惧/创伤）
- 绝不会说的话（回避话题/禁忌词）
```

**关键原则**：语言规则用具体行为描述而非形容词。"内向"无用，"从不主动提供信息；用问题回答问题"有用。

### 2.2 情绪状态日志 (Emotional State Log)

逐章记录主要角色的心理状态，确保情绪变化有据可循。

```markdown
## 情绪状态日志

### 第 N 章
| 角色 | 情绪状态 | 触发事件 | 持续影响 |
|------|----------|----------|----------|
| 主角 | 焦虑→决心 | 得知阴谋 | 对信任的人产生怀疑（持续2-3章）|
| 反派 | 得意→不安 | 计划被干扰 | 加强对下属的控制（持续3-5章）|

### 情绪弧线追踪
- 主角情绪曲线：[低谷]--[低谷]--[转折]--[上升]--[高峰]--[回落]--[当前:上升]
- 关键转折点：第3章（得知真相）、第7章（被背叛）、第12章（重拾信念）
```

### 2.3 关系追踪器 (Relationship Tracker)

记录每对角色关系的当前状态，注意关系是**不对称的**。

```markdown
## 关系追踪器

### 主角 → 导师
- 当前状态：尊敬但开始质疑
- 信任度：60/100（从85下降）
- 关键事件：第8章发现导师隐瞒了关键信息
- 未解决的张力：主角不知道导师的隐瞒是出于保护还是操控

### 导师 → 主角
- 当前状态：保护性的、有保留的
- 信任度：80/100
- 关键事件：第5章决定逐步透露真相
- 未解决的张力：何时告诉主角全部真相

### 关系矩阵快照（第 N 章末）
| | 主角 | 导师 | 反派 | 盟友 |
|------|------|------|------|------|
| 主角 | - | 尊敬/质疑 60 | 仇恨/恐惧 10 | 信任 75 |
| 导师 | 保护/隐瞒 80 | - | 警惕 40 | 友善 55 |
| 反派 | 利用 30 | 忌惮 50 | - | 蔑视 20 |
| 盟友 | 忠诚 85 | 礼貌 50 | 恐惧 15 | - |
```

**关键原则**：A 原谅 B 不等于 B 原谅 A。关系追踪器必须双向记录。

### 2.4 时间线 (Timeline)

维护故事内的时间线，防止"瞬移""信息悖论""未获得的技能"。

```markdown
## 时间线

### 故事时间
- 第1-3章：第1天（抵达城镇、初步调查）
- 第4-6章：第2-3天（发现线索、初次冲突）
- 第7-9章：第5天（时间跳跃2天——需交代去向）
- 第10章：第5天夜间（高潮准备）

### 角色位置追踪
| 时间 | 主角位置 | 导师位置 | 反派位置 |
|------|----------|----------|----------|
| 第1天上午 | 城镇入口 | 学院 | 未知 |
| 第1天下午 | 酒馆 | 学院 | 城外据点 |
| 第2天上午 | 市场 | 学院 | 城内（伪装）|

### 信息流追踪（谁知道什么）
| 事实 | 主角 | 导师 | 反派 | 盟友 |
|------|------|------|------|------|
| 阴谋存在 | ✅ 第3章 | ✅ 第1章 | ✅ 始终 | ❌ |
| 反派身份 | ❌ | ✅ 第5章 | ✅ | ❌ |
| 密信位置 | ✅ 第7章 | ✅ | ❌ | ✅ 第9章 |
```

**关键原则**：角色不能使用尚未获得的信息。信息流追踪防止"信息悖论"。

### 2.5 弧线大纲 (Arc Outline)

为每个主要角色预写角色弧线，每 5-10 章标注检查点。

```markdown
## 角色弧线大纲

### 主角弧线：从逃避到承担
- 起点（第1章）：逃避责任，只想过平静生活
- 里程碑1（第5章）：被动卷入，不得不行动
- 里程碑2（第10章）：主动选择承担，但仍有犹豫
- 里程碑3（第15章）：面对核心恐惧，做出关键抉择
- 终点（第20章）：完全接受责任，成长为真正的领袖
- 当前位置：里程碑1与里程碑2之间（第8章）

### 弧线检查点
| 检查点 | 预期状态 | 实际状态 | 偏差处理 |
|--------|----------|----------|----------|
| 第5章末 | 被动卷入 | ✅ 已达成 | - |
| 第10章末 | 主动承担 | 待验证 | - |
| 第15章末 | 面对恐惧 | 待验证 | - |

### 被抛弃的弧线检测
- [ ] 所有已铺垫的弧线是否有催化事件推进？
- [ ] 是否有弧线超过5章没有进展？
- [ ] 弧线终点是否仍在故事结局前可达？
```

## §3 生成时注入协议

每次生成新章节前，将五文档的相关部分注入上下文：

```
[生成上下文 - 第 N 章]
- 当前角色弧线阶段：[主角]里程碑1→里程碑2之间，当前状态：被动卷入但开始主动思考
- 角色情绪状态：[主角]焦虑→决心；[导师]保护性的、有保留
- 关系当前状态：[主角→导师]尊敬但质疑(60)；[导师→主角]保护但隐瞒(80)
- 时间线约束：第N天，主角在[位置]，[角色]不知道[事实]
- 语言规则提醒：[主角]用问题回答问题；[导师]说话总带保留
- 弧线下一里程碑：第10章——主动选择承担
```

## §4 一致性检查清单

每章生成后执行：

- [ ] 角色行为是否符合事实表中的语言规则？
- [ ] 角色是否使用了尚未获得的信息？（查信息流表）
- [ ] 情绪变化是否有触发事件？（查情绪日志）
- [ ] 关系变化是否双向记录？（查关系追踪器）
- [ ] 时间流逝是否合理？（查时间线）
- [ ] 弧线是否在预期轨道上？（查弧线大纲）
- [ ] 是否有被遗忘的早期细节需要呼应？

## §5 与其他文档的关系

- **`character-design.md`**: 角色设计定义"角色是什么"，本文档确保"角色不变样"。
- **`timeline-tracking.md`**: 时间线追踪文档与本文档的时间线部分互补——前者关注事件，后者关注角色在时间中的状态。
- **`anti-ai-patterns.md`**: AI 写作中的"性格漂移"和"选择性失忆"通过本文档的五文档体系系统性预防。
- **`plot-architecture.md`**: 弧线大纲与情节架构互补——情节关注"发生什么"，弧线关注"角色如何变化"。
- **`dialogue-techniques.md`**: 对话中角色的语言模式应遵循事实表中的语言规则。
