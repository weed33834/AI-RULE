# 情节架构技能 (Plot Architecture)

> 情节是骨架，角色是血肉。好的情节不是"发生了什么"，而是"为什么发生"和"它如何改变角色"。

## §1 经典结构模型

### 三幕结构
| 幕 | 占比 | 核心功能 | 关键节点 |
|----|------|----------|----------|
| 第一幕 | 25% | 建立世界、角色、日常 | 激励事件 → 第一转折点 |
| 第二幕 | 50% | 冲突升级、角色挣扎 | 中点 → 第二转折点（至暗时刻） |
| 第三幕 | 25% | 高潮、解决 | 高潮 → 结局 |

### 起承转合（四幕结构）
- **起**：建立日常世界和主角状态。
- **承**：激励事件打破平衡，主角被动应对。
- **转**：主角从被动转为主动，核心冲突爆发。
- **合**：高潮对决，新的平衡建立。

### 英雄之旅（Campbell 12 阶段简化版）
日常世界 → 冒险召唤 → 拒绝召唤 → 遇见导师 → 跨越门槛 → 考验/盟友/敌人 → 接近深渊 → 核心考验 → 获得奖赏 → 回归之路 → 复活 → 带着仙丹回归。

## §2 情节设计原则

### 因果链 > 事件序列
- ❌ "然后发生了 A，然后发生了 B，然后发生了 C"（时间序列）
- ✅ "因为 A 发生了，所以 B 不得不发生，而 B 导致了 C"（因果链）
- 每个情节节点都应与前一个有因果关系——"因为...所以..."。

### 冲突的层次
| 层次 | 冲突类型 | 示例 |
|------|----------|------|
| 内在冲突 | 角色与自我 | 良知 vs 生存 |
| 人际冲突 | 角色与角色 | 爱人 vs 对手 |
| 社会冲突 | 角色与群体/制度 | 个人 vs 腐败政权 |
| 命运冲突 | 角色与自然/命运 | 人类 vs 末日 |

### 节奏控制
- **收紧**：短场景、快节奏、高密度冲突——适合高潮和动作段落。
- **舒缓**：长场景、慢节奏、角色内心——适合过渡和铺垫。
- 张弛有度：持续紧张让读者疲劳，持续舒缓让读者无聊。
- 章节末尾钩子：每章结尾留一个悬念或转折，驱动翻页欲。

## §3 伏笔设计

### 伏笔三原则
1. **提前埋设**：伏笔在需要回收前至少 2-3 章埋下。
2. **自然伪装**：伏笔初次出现时不引人注目，融入场景中。
3. **回收有力**：回收时让读者恍然大悟——"原来那个细节是这个意思！"

### 伏笔台账格式
```
[伏笔内容] | [埋设章节] | [状态: 待回收/已回收] | [预计回收章节] | [回收方式]
```

### 伏笔类型
| 类型 | 说明 | 示例 |
|------|------|------|
| 物品伏笔 | 出现的物品后来起关键作用 | 第 2 章的旧地图在第 8 章指引方向 |
| 对话伏笔 | 角色说的话后来应验 | "他从不喝酒" → 后来被下毒 |
| 行为伏笔 | 角色的某个习惯后来成为线索 | 总是摸左手指 → 左手缺失的秘密 |
| 设定伏笔 | 世界观规则后来成为关键 | 魔法消耗记忆 → 主角遗忘关键信息 |

## §5 情节一致性检查清单

- [ ] 因果链是否完整？（每个事件都有前因后果）
- [ ] 伏笔是否全部回收？（检查 plot-threads.md）
- [ ] 时间线是否连贯？（事件顺序、角色年龄、季节）
- [ ] 角色动机是否驱动了情节？（而非"剧情需要"）
- [ ] 节奏是否张弛有度？（有无连续 3 章都是高潮或都是铺垫）
- [ ] 章节末尾是否有钩子？

## §6 进阶：故事知识图谱辅助情节管理

> 情节管理的进阶形态见 `docs/skills/story-graph.md` 和 AGENTS.md §6.3。
> 图谱将情节结构化为 `Event` 节点和 `causes`/`foreshadows` 关系边，支持：
> - "从事件 A 到事件 B 的因果链是什么？"路径追踪。
> - "哪些伏笔还未回收？"全局查询。
> - "删除这个事件会影响哪些后续事件？"影响范围分析。
> 详见 `docs/skills/creative-evaluation.md` 模式 11（创意 GraphRAG 全局查询）。

## 核心原则
- 情节是角色的延伸——好情节让角色做选择，选择定义角色。
- 意外不等于转折——让读者意外的转折必须事后觉得"合情合理"。
- 删比加难——砍掉不推动情节或角色发展的场景，即使写得很好。
