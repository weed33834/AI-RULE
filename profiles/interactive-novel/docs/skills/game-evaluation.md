# 游戏质量评估（Game Evaluation）

> 本文档定义互动小说游戏引擎的 6 维评估框架、具体指标与测量方式、玩家体验追踪、质量自检清单与 A/B 测试建议，为游戏质量提供可量化、可追踪、可改进的评估体系。

## 1. 核心理念

没有评估就没有改进。互动小说游戏的质量不能仅凭主观感受判断——需要一套可量化的指标体系，持续追踪游戏表现，发现薄弱环节，指导改进方向。本文档定义的 6 维评估框架，覆盖了互动小说游戏的核心体验维度。

评估分为三个层次：
- **实时自检**：每轮叙事后快速自检，防患于未然。
- **阶段评估**：每 10-20 轮深度评估，发现系统性问题。
- **终局评估**：游戏结束时全面评估，指导后续改进。

## 2. 6 维评估框架

| # | 维度 | 核心问题 | 权重 | 目标分 |
|---|------|----------|------|--------|
| 1 | 沉浸感 | 玩家是否"在场"？ | 20% | ≥ 85 |
| 2 | 自主权 | 选择是否真实有意义？ | 20% | ≥ 85 |
| 3 | NPC 智能 | NPC 是否像真人？ | 20% | ≥ 85 |
| 4 | 一致性 | 世界是否自洽？ | 15% | ≥ 90 |
| 5 | 节奏 | 张弛有度？ | 15% | ≥ 80 |
| 6 | 重玩性 | 不同选择不同体验？ | 10% | ≥ 75 |

综合评分 = 各维度得分 × 权重之和。目标综合分 ≥ 83。

## 3. 各维度指标与测量方式

### 3.1 沉浸感（Immersion）

衡量玩家是否感觉"身处游戏世界"而非"在读文本"。

| 指标 | 测量方式 | 目标值 | 数据源 |
|------|----------|--------|--------|
| 第二人称叙事使用率 | "你"字使用频率 / 总叙事句数 | > 90% | 叙事文本 |
| 感官描写密度 | 每场景涉及感官种类数 | ≥ 2 种 | 叙事文本 |
| 打破第四面墙次数 | 提及 AI/模型/生成的次数 | 0 次 | 叙事文本 + 审计日志 |
| 套话/机械应答次数 | 套话模式检出次数 | 0 次 | 审计日志 |
| 场景钩子覆盖率 | 有钩子的场景 / 总场景 | > 80% | 叙事文本 |
| 玩家沉默时间 | 玩家两次输入间的平均间隔 | < 60 秒 | 会话日志 |

**沉浸感评分公式**：
```
沉浸感分 = (第二人称率×0.2 + 感官密度×0.15 + (1-破墙次数)×0.2 
         + (1-套话率)×0.2 + 钩子率×0.15 + 沉默系数×0.1) × 100
```

### 3.2 自主权（Player Agency）

衡量玩家的选择是否真正影响游戏。

| 指标 | 测量方式 | 目标值 | 数据源 |
|------|----------|--------|--------|
| 假选择比例 | 结果相同的选项组 / 总选项组 | < 5% | choices-log.md |
| 选择后果体现率 | 已体现后果的选择 / 总选择 | > 90% | choices-log.md |
| 玩家自定义行动比例 | 自定义行动 / 总行动 | > 30% | 会话日志 |
| 选择影响 NPC 比例 | 影响 NPC 关系的选择 / 总选择 | > 40% | choices-log.md |
| 选择影响世界比例 | 影响世界状态的选择 / 总选择 | > 25% | choices-log.md |
| railroad 检出次数 | 强制走向被检出次数 | 0 次 | 审计日志 |

**自主权评分公式**：
```
自主权分 = ((1-假选择率)×0.25 + 后果体现率×0.25 + 自定义率×0.15 
         + NPC影响率×0.1 + 世界影响率×0.1 + (1-railroad率)×0.15) × 100
```

### 3.3 NPC 智能（NPC Intelligence）

衡量 NPC 是否表现得像真人而非木偶。

| 指标 | 测量方式 | 目标值 | 数据源 |
|------|----------|--------|--------|
| 弱智模式检出次数 | 10 种弱智模式的检出总数 | < 2 次/10轮 | 审计日志 |
| NPC 记忆一致性 | NPC 引用过去交互的准确性 | > 95% | 审计日志 |
| 情感变化合理性 | 有据可循的情感变化比例 | > 95% | relationships.json |
| NPC 个性化程度 | 不同 NPC 回复差异度 | > 80% | 叙事文本 |
| NPC 主动行动次数 | NPC 主动推进事件的次数 | ≥ 1 次/10轮 | 事件日志 |
| 万能回应检出次数 | 套话式回复检出 | 0 次 | 审计日志 |

**NPC 智能评分公式**：
```
NPC智能分 = ((1-弱智率)×0.3 + 记忆一致率×0.2 + 情感合理率×0.2 
          + 个性化度×0.1 + 主动率×0.1 + (1-万能率)×0.1) × 100
```

### 3.4 一致性（Coherence）

衡量游戏世界是否自洽无矛盾。

| 指标 | 测量方式 | 目标值 | 数据源 |
|------|----------|--------|--------|
| 矛盾检出次数 | 7 项检查的违规总数 | < 1 次/10轮 | audit-log.md |
| 时间线错误次数 | 时间线断裂/倒流检出 | 0 次 | audit-log.md |
| 知识边界违规次数 | NPC 知不该知的信息 | 0 次 | audit-log.md |
| OOC 检出次数 | 角色行为不符合设定 | < 1 次/10轮 | audit-log.md |
| 状态-叙事一致率 | 状态文件与叙事匹配度 | 100% | 状态文件 + 叙事 |
| 因果链完整率 | 可追溯因果的事件比例 | > 95% | event-log.md |

**一致性评分公式**：
```
一致性分 = ((1-矛盾率)×0.25 + (1-时间线错率)×0.2 + (1-知识违规率)×0.2 
         + (1-OOC率)×0.15 + 状态一致率×0.1 + 因果完整率×0.1) × 100
```

### 3.5 节奏（Pacing）

衡量游戏张弛有度，不拖沓不仓促。

| 指标 | 测量方式 | 目标值 | 数据源 |
|------|----------|--------|--------|
| 高潮/铺垫比例 | 高潮场景 / 铺垫场景 | 1:2 ~ 1:3 | 场景日志 |
| 场景长度分布 | 各场景叙事长度的方差 | 适中（不过长/过短） | 叙事文本 |
| 战斗节奏 | 战斗回合数 | 3-8 回合 | 战斗日志 |
| 对话节奏 | 单次对话轮数 | 3-10 轮 | 会话日志 |
| 玩家流失点 | 玩家沉默超过 2 分钟的场景 | < 10% | 会话日志 |
| 悬念设置率 | 有悬念收尾的会话比例 | > 80% | 会话日志 |

**节奏评分公式**：
```
节奏分 = (高潮比例合理度×0.25 + 场景长度合理度×0.2 + 战斗节奏度×0.15 
       + 对话节奏度×0.15 + (1-流失率)×0.15 + 悬念率×0.1) × 100
```

### 3.6 重玩性（Replayability）

衡量不同选择是否带来不同体验。

| 指标 | 测量方式 | 目标值 | 数据源 |
|------|----------|--------|--------|
| 结局多样性 | 不同结局数量 | ≥ 3 个 | 结局记录 |
| 分支覆盖率 | 玩家走过的分支 / 总分支 | 需多次游玩覆盖 | 分支图 |
| 隐藏内容发现率 | 发现的隐藏内容 / 总隐藏内容 | < 50%（鼓励重玩） | 隐藏内容记录 |
| NPC 路径多样性 | 同一 NPC 的不同交互路径 | ≥ 2 条 | NPC 档案 |
| 选择分歧点数量 | 真分支决策点数量 | ≥ 5 个/局 | choices-log.md |

**重玩性评分公式**：
```
重玩性分 = (结局多样性×0.3 + 分支覆盖潜力×0.2 + 隐藏内容率×0.15 
         + NPC路径多样×0.15 + 分歧点数×0.2) × 100
```

## 4. 玩家体验指标追踪

### 4.1 自动追踪指标

以下指标在游戏过程中自动追踪，记录到 `.game-state/metrics.json`：

```json
{
  "rounds_played": 35,
  "total_play_time_minutes": 120,
  "avg_response_time_seconds": 45,
  "player_silence_count": 3,
  "choices_made": 18,
  "custom_actions": 7,
  "battles_fought": 5,
  "npcs_interacted": 8,
  "consistency_violations": 1,
  "dumb_ai_patterns_detected": 2,
  "immersion_breakers": 0
}
```

### 4.2 玩家反馈收集

| 反馈类型 | 收集方式 | 时机 |
|----------|----------|------|
| 即时反馈 | 玩家用 `/feedback` 提交 | 任何时候 |
| 场景评分 | 场景结束后可选评分（1-5星） | 每个场景结束 |
| 会话评价 | 会话结束时的简短问卷 | 每次会话结束 |
| 终局评价 | 游戏结束时的完整评价 | 游戏结束 |

### 4.3 负面信号检测

以下信号暗示体验下降，检测到时主动预警：

| 信号 | 阈值 | 预警动作 |
|------|------|----------|
| 玩家连续沉默 > 2 分钟 | 1 次 | 主动推进事件或提供选项 |
| 玩家连续输入极短（"嗯"、"继续"） | 3 次 | 检查是否叙事过于平淡 |
| 玩家使用 `/skip` | 1 次 | 检查当前场景是否拖沓 |
| 玩家 `/feedback` 投诉 | 1 次 | 记录并分析原因 |
| 上下文使用率 > 90% | 持续 | 建议保存并新开会话 |

## 5. 游戏质量自检清单

### 5.1 每轮自检（快速）

每轮叙事后快速自检（< 30 秒）：

```
[ ] 本轮叙事是否使用第二人称？
[ ] 本轮是否有感官描写？
[ ] 本轮是否给了玩家可行动的空间？
[ ] 本轮是否有信息量（推动剧情/揭示角色/制造张力）？
[ ] 本轮是否有套话或机械应答？
[ ] 本轮是否打破第四面墙？
[ ] 状态文件是否与叙事一致？
```

### 5.2 每 10 轮自检（深度）

每 10 轮执行深度自检，对照 6 维评估框架：

```
[沉浸感]
[ ] 近 10 轮第二人称使用率 > 90%？
[ ] 每场景感官描写 ≥ 2 种？
[ ] 零打破第四面墙？
[ ] 零套话？
[ ] 场景钩子覆盖率 > 80%？

[自主权]
[ ] 假选择比例 < 5%？
[ ] 选择后果体现率 > 90%？
[ ] 玩家自定义行动 > 30%？
[ ] 零 railroad？

[NPC 智能]
[ ] 弱智模式检出 < 2 次？
[ ] NPC 记忆一致性 > 95%？
[ ] 情感变化合理性 > 95%？
[ ] NPC 有主动行动？

[一致性]
[ ] 矛盾检出 < 1 次？
[ ] 零时间线错误？
[ ] 零知识边界违规？
[ ] 状态-叙事 100% 一致？

[节奏]
[ ] 高潮/铺垫比例合理（1:2~1:3）？
[ ] 场景长度适中？
[ ] 玩家流失点 < 10%？

[重玩性]
[ ] 已出现 ≥ 1 个真分支？
[ ] 有隐藏内容线索？
```

### 5.3 终局自检

游戏结束时全面评估：

- 6 维评分各多少？综合分多少？
- 哪些维度达标？哪些未达标？
- 玩家反馈中最常见的问题是什么？
- 弱智模式检出率趋势如何？
- 一致性违规趋势如何？
- 结局是否由累积选择决定（而非最后一步）？

## 6. A/B 测试建议

### 6.1 测试框架

对关键设计决策进行 A/B 测试，用数据指导改进：

| 测试维度 | A 版本 | B 版本 | 测量指标 |
|----------|--------|--------|----------|
| 叙事详略 | 详细描写（每场景 300+ 字） | 精简描写（每场景 150 字） | 沉浸感、玩家沉默时间 |
| 选项数量 | 3 个选项 | 4-5 个选项 | 自主权、选择后果体现率 |
| NPC 主动性 | NPC 被动响应 | NPC 主动推进 | NPC 智能、玩家参与度 |
| 难度曲线 | 线性递增 | 波浪式起伏 | 节奏、玩家流失率 |
| 信息释放 | 集中倾倒 | 分层释放 | 沉浸感、信息记忆率 |

### 6.2 测试方法

1. **同一玩家顺序测试**：玩家先体验 A 版本一段，再体验 B 版本一段，比较感受。
2. **不同玩家平行测试**：不同玩家分别体验 A/B 版本，收集反馈对比。
3. **同一场景双版本**：对同一场景生成 A/B 两个版本，让玩家选择更喜欢的。

### 6.3 测试注意事项

- **每次只变一个变量**：不要同时改叙事详略和选项数量。
- **样本量**：至少 3 次体验同一版本才能初步判断。
- **主观+客观**：既看玩家主观反馈，也看自动追踪指标。
- **记录测试结果**：测试结论写入 `.game-state/evaluation-log.md`，指导后续改进。

## 7. 评估报告生成

### 7.1 阶段报告（每 20 轮）

```markdown
## 游戏质量阶段报告（第 1-20 轮）

### 6 维评分
| 维度 | 得分 | 目标 | 状态 |
|------|------|------|------|
| 沉浸感 | 88 | ≥85 | 达标 |
| 自主权 | 82 | ≥85 | 待改进 |
| NPC 智能 | 90 | ≥85 | 达标 |
| 一致性 | 95 | ≥90 | 达标 |
| 节奏 | 78 | ≥80 | 待改进 |
| 重玩性 | 80 | ≥75 | 达标 |
| **综合** | **85.2** | ≥83 | **达标** |

### 薄弱环节
1. 自主权（82分）：假选择比例偏高（8%），建议增加选择的实质差异。
2. 节奏（78分）：第 12-15 轮场景过长，建议精简铺垫。

### 改进建议
- 在后续选项设计中，确保每个选择有实质不同的后果。
- 控制探索场景长度在 200-300 字以内。
```

### 7.2 终局报告

游戏结束时生成完整评估报告，包含：
- 6 维评分及趋势图。
- 全程弱智模式检出统计。
- 一致性违规统计。
- 玩家反馈汇总。
- 最成功和最失败的场景分析。
- 后续改进方向建议。

报告存入 `.game-state/final-evaluation.md`，用于技能演进（见 `evolution-policy.md`）。
