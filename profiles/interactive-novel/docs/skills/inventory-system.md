# 物品系统（Inventory System）

> 本文档定义互动小说游戏引擎中物品的分类、稀有度、效果、交易、装备机制，以及物品描述叙事化的规范，让物品不仅是数据，更是叙事的一部分。

## 1. 设计目标

物品系统是玩家与游戏世界物质交互的核心。它要做到：

1. **物质真实感**：物品有重量、价值、来源、磨损，不是凭空出现的数值。
2. **叙事融合**：物品描述服务于氛围和剧情，不只是"铁剑 +5 攻击"。
3. **策略深度**：背包容量、装备搭配、资源消耗让玩家做有意义的取舍。

## 2. 物品分类

### 2.1 五大类别

| 类别 | 说明 | 示例 | 堆叠 |
|------|------|------|------|
| 武器 | 用于战斗，影响伤害和攻击方式 | 铁剑、长弓、法杖、匕首 | 否 |
| 防具 | 提供防护，分头盔/胸甲/护腿/靴子/饰品 | 锁子甲、皮靴、护身符 | 否 |
| 消耗品 | 使用后消失，产生即时效果 | 治疗药水、解毒剂、火把、食物 | 是 |
| 关键道具 | 剧情推进必需，不可丢弃不可出售 | 古老钥匙、信物、徽章、证据 | 否 |
| 杂物 | 装饰或低价值物品，可出售或合成材料 | 破旧铜币、兽骨、草药、空瓶 | 是 |

### 2.2 类别行为差异

不同类别的物品在游戏中有不同的处理规则：

```text
分类行为规则：

武器/防具：
  - 可装备/卸下
  - 有耐久度，战斗中磨损
  - 不可堆叠，占用独立槽位

消耗品：
  - 使用即消失（数量-1）
  - 可堆叠，同类合并到一个槽位
  - 战斗中可使用（消耗行动点）

关键道具：
  - 标记 unique = true，不可丢弃
  - 不可出售，商人会拒绝（"这东西我不收"）
  - 在背包中单独分区显示

杂物：
  - 可出售、可拆解为材料
  - 部分可作为任务提交物
```

## 3. 稀有度等级

### 3.1 五级稀有度

| 等级 | 颜色标识 | 出现频率 | 附加效果 | 示例 |
|------|----------|----------|----------|------|
| 普通 | 灰色 | 常见 | 无附加 | 铁剑、布衣、面包 |
| 精良 | 绿色 | 较常见 | +1 项基础属性 | 锋利铁剑、加固皮甲 |
| 稀有 | 蓝色 | 不常见 | +2 项属性或 1 项特殊 | 烈焰附魔剑、精灵皮甲 |
| 史诗 | 紫色 | 稀有 | +3 项属性或独特技能 | 龙骨巨剑、星辉法袍 |
| 传说 | 橙色 | 极稀有 | 强大独特效果，有背景故事 | 屠龙者之剑、永生者之冠 |

### 3.2 稀有度与来源

稀有度不仅影响数值，还影响获取方式和叙事权重：

- **普通/精良**：商店购买、普通敌人掉落、随处可见。
- **稀有**：精英敌人掉落、宝箱发现、特殊商人出售、任务奖励。
- **史诗**：Boss 掉落、隐藏地点探索、复杂任务链奖励。
- **传说**：需要满足特定条件组合（如击败隐藏 Boss + 持有特定道具 + 特定时间）。

### 3.3 传说物品的叙事要求

传说级物品必须有背景故事，不能只是数值堆叠：

```text
传说物品档案示例：

名称：屠龙者之剑
稀有度：传说
背景：三百年前，骑士 Vael 单枪匹马斩杀黑龙 Nythrax。
      此剑浸透了龙血，剑刃上永不消退的暗红色纹路即是证明。
特殊效果：对龙类敌人伤害 +200%；持有时龙类敌人对你的恐惧值 +30。
诅咒：持有者会逐渐梦见黑龙的低语，长期持有可能导致精神异常。
```

## 4. 物品效果系统

### 4.1 效果类型

| 效果类型 | 触发方式 | 持续 | 示例 |
|----------|----------|------|------|
| 即时效果 | 使用时 | 一次性 | 治疗药水回 50HP |
| 装备效果 | 装备时 | 装备期间 | 力量+5、火焰抗性 |
| 被动效果 | 持有即生效 | 持有期间 | 持有时幸运+10 |
| 使用效果 | 主动使用 | 即时或持续 | 火把照明 30 分钟 |
| 触发效果 | 满足条件 | 触发时 | HP低于30%自动护盾 |
| 诅咒效果 | 装备/持有 | 不可轻易解除 | 每日扣除 5HP |

### 4.2 效果数据结构

物品效果以结构化数据存储，便于引擎解析执行：

```json
{
  "item_id": "potion_greater_healing",
  "name": "高级治疗药水",
  "category": "consumable",
  "rarity": "rare",
  "effects": [
    {
      "type": "instant",
      "trigger": "use",
      "action": "heal",
      "target": "self",
      "value": 80,
      "duration": 0
    }
  ],
  "weight": 0.5,
  "value": 120,
  "stackable": true,
  "max_stack": 10
}
```

### 4.3 效果叠加规则

- 同名效果不叠加，取最高值（如两个 +5 力量装备只取 +5）。
- 不同名同类效果叠加（+5 力量 + +3 力量 = +8 力量）。
- 诅咒效果与增益效果独立计算，不互相抵消（除非有解除诅咒机制）。
- 消耗品效果有冷却时间，防止滥用。

## 5. 交易系统

### 5.1 三种交易方式

| 方式 | 说明 | 适用 |
|------|------|------|
| 购买 | 用货币从 NPC/商店购买物品 | 标准交易 |
| 出售 | 将物品卖给 NPC/商店换取货币 | 清理背包 |
| 以物易物 | 用物品直接交换其他物品 | 无货币世界、特殊商人 |

### 5.2 定价规则

物品价格由基础价值、稀有度系数、商人关系、区域供需共同决定：

```text
最终价格 = 基础价值 × 稀有度系数 × 关系修正 × 区域供需修正

稀有度系数：
  普通  ×1.0    精良 ×1.5    稀有 ×3.0
  史诗  ×8.0    传说 ×25.0

关系修正（玩家与商人好感度）：
  好感 ≥ 50：买入 ×0.9，卖出 ×1.2（老主顾优惠）
  好感 0~49：买入 ×1.0，卖出 ×1.0（标准价）
  好感 < 0：买入 ×1.3，卖出 ×0.7（不愿做生意）

区域供需修正：
  产地附近：买入 ×0.8（货源充足）
  偏远地区：买入 ×1.5（运输成本）
  禁运区域：买入 ×2.0 或无货（黑市才有）
```

### 5.3 交易限制

- 关键道具不可交易（系统标记 `unique = true`）。
- 部分物品有购买数量限制（如稀有药水每人每天限购 1 瓶）。
- 商人资金有限——不能把所有东西都卖给他换巨额货币。
- 黑市交易有风险——可能买到赃物、假货，或被便衣盯上。
- 以物易物需要双方都认为交换物价值相当，否则拒绝。

### 5.4 讨价还价

玩家可尝试讨价还价，判定基于魅力和口才：

| 判定结果 | 价格调整 | 副作用 |
|----------|----------|--------|
| 大成功 | 买入 ×0.8 / 卖出 ×1.3 | 商人好感+5 |
| 成功 | 买入 ×0.9 / 卖出 ×1.1 | 无 |
| 失败 | 维持原价 | 商人略显不耐烦 |
| 大失败 | 买入 ×1.2 / 卖出 ×0.8 | 商人好感-10，可能拒绝交易 |

## 6. 装备与卸下

### 6.1 装备槽位

| 槽位 | 数量 | 说明 |
|------|------|------|
| 主手 | 1 | 武器 |
| 副手 | 1 | 盾牌/副武器 |
| 头部 | 1 | 头盔 |
| 躯干 | 1 | 胸甲 |
| 腿部 | 1 | 护腿 |
| 足部 | 1 | 靴子 |
| 饰品 | 2 | 戒指/项链/护身符 |

### 6.2 装备规则

- 装备需满足属性要求（如"力量 ≥ 15 才能装备巨剑"）。
- 装备职业限制（法师不能装备重甲，除非有特殊技能）。
- 装备诅咒物品后无法轻易卸下，需找到解除诅咒的方法。
- 装备切换在非战斗中即时，在战斗中消耗行动点。

### 6.3 卸下与替换

- 卸下的装备回到背包，若背包满则无法卸下（提示先清理背包）。
- 替换装备时，旧装备自动回到背包。
- 装备耐久度归零时自动卸下并标记为损坏，需修复才能再使用。

```text
装备状态文件示例（.game-state/player.json 的 equipment 段）：

{
  "equipment": {
    "main_hand": { "id": "iron_sword", "durability": 85, "max_durability": 100 },
    "off_hand": null,
    "head": { "id": "leather_cap", "durability": 60, "max_durability": 80 },
    "body": { "id": "chainmail", "durability": 92, "max_durability": 120 },
    "legs": null,
    "feet": { "id": "leather_boots", "durability": 70, "max_durability": 80 },
    "accessory": [
      { "id": "ring_strength", "durability": null },
      null
    ]
  }
}
```

## 7. 物品描述叙事化

### 7.1 叙事化原则

物品描述不只是数值列表，而是融入世界观和氛围的文字：

| 错误做法 | 正确做法 |
|----------|----------|
| "铁剑 +5 攻击" | "一柄磨损的铁剑，剑柄上刻着模糊的铁匠印记。挥动时略显沉重，但足以致命。" |
| "治疗药水 回50HP" | "一瓶泛着微光的红色液体，瓶塞处渗出淡淡的草药气味。据说北方修道院的僧侣们以此救人无数。" |
| "古老钥匙 关键道具" | "一把锈迹斑斑的铜钥匙，钥匙头铸成蝙蝠展翅的形状。握在手中，你能感到一丝不属于这世界的凉意。" |

### 7.2 描述结构

每个物品的描述应包含以下要素（视重要程度取舍）：

1. **外观**：材质、颜色、磨损程度、特殊标记。
2. **来历**（稀有及以上）：制造者、历史事件、前主人。
3. **功能**：使用方式和效果（叙事化，非纯数值）。
4. **氛围**：触感、气味、握持感受、旁人反应。
5. **数值**：附在描述后，用简洁格式呈现。

### 7.3 获取与失去的叙事

物品的获取和失去也应当叙事化，而非简单的"+1 铁剑"：

```text
获取示例：
你弯腰拾起那柄剑。剑鞘已经朽烂，露出泛着冷光的剑刃。
剑柄缠皮的触感告诉你，这曾是一把被频繁使用的武器。
[获得：精良级 骑士长剑 | 攻击+8 | 耐久 75/100]

失去示例：
最后一瓶治疗药水滑过你的喉咙，辛辣中带着一丝甜味。
伤口的疼痛开始消退，但你知道，庇护已经所剩无几。
[消耗：治疗药水 ×1 | 剩余 0]
```

## 8. 背包管理

### 8.1 容量系统

- 背包有重量上限和槽位上限双重限制。
- 重量上限影响移动速度和体力消耗（超重减速、无法奔跑）。
- 槽位上限防止囤积杂物，鼓励取舍。
- 可通过购买更大背包或储物戒指扩展容量。

### 8.2 整理与分类

- 玩家可用 `/inventory` 查看物品栏，默认按类别分组。
- 关键道具单独分区，防止误操作丢弃。
- 玩家可标记"收藏"物品，批量出售时自动跳过。

## 9. 与其他系统的联动

| 联动系统 | 联动方式 |
|----------|----------|
| 战斗系统 | 武器/防具影响伤害和防护；消耗品战斗中使用 |
| 对话系统 | 出示/赠送物品影响 NPC 反应；贿赂需要物品 |
| 任务系统 | 关键道具推进任务；提交物完成任务 |
| 状态机 | 物品变动实时写入 `inventory.json` |
| 难度引擎 | 困难难度下资源稀缺，补给品掉落减少 |
