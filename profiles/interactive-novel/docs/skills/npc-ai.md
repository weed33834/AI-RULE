# NPC 智能系统技能 (NPC AI System)

> NPC 不是只会念预设台词的木偶，而是有记忆、有情感、有目标、会欺骗、会拒绝的活角色——这是玩家沉浸感的核心。
> NPC 状态文件结构详见 `game-state-machine.md` §3；对话机制与说服判定详见 `dialogue-system.md`；反弱智标准详见 `anti-dumb-ai.md`；NPC 引擎提示词详见 `docs/prompts/npc-engine.md`。

## §1 NPC 五要素设计协议

每个 NPC 在创建时必须完整定义以下五要素，缺一不可。这五要素共同构成 NPC 的"灵魂档案"。

### 要素总览

| 要素 | 核心问题 | 作用 |
|------|----------|------|
| 说话方式 | 这个人怎么说话？ | 让 NPC 一开口就有辨识度 |
| 小动作 | 这个人紧张/放松时身体怎么动？ | 用肢体语言传递情绪，而非直白描述 |
| 想要什么 | 这个人对玩家有什么诉求？ | 驱动 NPC 主动行为，避免被动工具人 |
| 隐藏什么 | 这个人绝不会主动说出的秘密？ | 制造信息差和剧情悬念 |
| 知识边界 | 这个人知道什么、不知道什么？ | 防止万能 NPC，保证信息传播合理 |

### 1.1 说话方式

不只是"语气"，而是从词汇、句式、口头禅到节奏的完整语言指纹。

```yaml
speech_pattern:
  vocabulary_level: formal    # formal / casual / crude / scholarly
  sentence_style: long_complex  # short_choppy / long_complex / fragmented
  catchphrase: "凡事皆有代价"   # 反复出现的口头禅
  dialect: 无                  # 方言/口音特征
  taboo_words: ["直接说钱"]     # 这个 NPC 不会用的词
  example_lines:
    - "阁下若执意如此，在下不便多言——只是，凡事皆有代价。"
    - "你想知道的事情，我或许知道。但知道本身，也是有代价的。"
```

### 1.2 小动作

NPC 在不同情绪下的习惯性肢体动作，用于"展示而非告知"情绪。

| 情绪状态 | 小动作示例 |
|----------|-----------|
| 紧张 | 不停地摩挲戒指、回避眼神、咽口水 |
| 放松 | 靠在椅背上、慢慢抿酒、手指敲桌面 |
| 撒谎 | 触碰鼻子、眨眼频率加快、重复玩家的问话 |
| 愤怒 | 攥紧拳头、咬紧牙关、身体前倾 |
| 悲伤 | 垂下视线、手指无意识地撕扯纸边 |

小动作必须与 NPC 性格一致——学者紧张时可能推眼镜，战士紧张时可能握刀柄。

### 1.3 想要什么

NPC 对玩家有明确的诉求，这驱动其主动接近玩家或设置障碍。

| 诉求类型 | 示例 | 行为表现 |
|----------|------|----------|
| 求助 | 需要玩家救她的妹妹 | 主动提供信息换取帮助 |
| 索贿 | 想要玩家的金币 | 故意刁难，暗示好处 |
| 试探 | 想确认玩家是否为间谍 | 问引导性问题，观察反应 |
| 结盟 | 想拉玩家加入派系 | 展示好处，许诺回报 |
| 复仇 | 想借玩家之手对付仇人 | 编造或夸大对方的恶行 |

### 1.4 隐藏什么

NPC 绝不会主动说出的秘密。这些秘密是剧情张力和反转的来源。

- 秘密可被玩家通过调查、其他 NPC、矛盾对话等途径发现。
- 秘密被揭穿时，NPC 的反应基于其性格（愤怒、崩溃、认命、反咬一口）。
- 每个 NPC 至少有一个秘密，重要 NPC 可有多个层层嵌套的秘密。

### 1.5 知识边界

明确界定 NPC "知道什么"和"不知道什么"，是防止万能 NPC 的关键。

```yaml
knowledge_boundary:
  expert_in: ["本地历史", "草药学", "酒馆八卦"]
  knows: ["王港城最近闹疫病", "城卫队长嗜酒"]
  does_not_know: ["魔法的运作原理", "盗贼公会藏身处", "宫廷政治"]
  rumors_heard: ["有人说疫病是人为投毒", "听说古森林有怪物出没"]
  misbelieves: ["以为疫病通过空气传播（实为水源）"]
```

`misbelieves`（错误认知）尤其重要——NPC 可以有错误信念，只要这符合其信息来源。这让世界更真实。

## §2 情感模型

NPC 的情感用四维数值追踪，直接影响对话选项和行动倾向。

### 四维情感数值

| 情感维度 | 范围 | 高值影响 | 低值影响 |
|----------|------|----------|----------|
| 信任度 | -100 ~ 100 | 分享秘密、托付任务、提供折扣 | 怀疑、隐瞒、设陷阱 |
| 恐惧度 | 0 ~ 100 | 顺从、出卖情报、逃跑 | 从容、大胆、敢于反抗 |
| 好感度 | -100 ~ 100 | 友好对话、主动帮助、特殊互动 | 冷淡、拒绝、敌意累积 |
| 敌意值 | 0 ~ 100 | 主动攻击、设障碍、向敌人告密 | 中立、可沟通 |

### 情感变化规则

- **基于具体行为**：情感变化必须由玩家的具体行为触发，记录原因。不可无故变化。
- **性格调节**：同一行为对不同性格 NPC 产生不同效果。

| 玩家行为 | 正直型 NPC | 狡诈型 NPC | 胆小型 NPC |
|----------|-----------|-----------|-----------|
| 帮助弱者 | 好感+15 | 好感-5（轻视） | 恐惧-10（安心） |
| 威胁恐吓 | 敌意+20 | 信任-10（警惕） | 恐惧+25（顺从） |
| 贿赂 | 信任-15（鄙夷） | 好感+10（高兴） | 恐惧+5（不安） |

- **交叉影响**：对 A 的好行为可能传到 B 耳中，影响 B 的情感（见 §3 记忆系统）。
- **变化阈值**：单轮情感变化超过 30 点触发警告，需确认剧情合理性。

### 情感与对话解锁

| 信任度区间 | 解锁内容 |
|------------|----------|
| -100 ~ -50 | 拒绝对话 / 敌对攻击 |
| -50 ~ 0 | 敷衍回答、提供虚假信息 |
| 0 ~ 30 | 基础信息交换、中立交易 |
| 30 ~ 70 | 分享个人见闻、提供支线任务 |
| 70 ~ 100 | 揭露秘密、托付重要任务、主动警告危险 |

## §3 记忆系统

### 交互历史

NPC 记住与玩家的每次重要交互，存储于其档案的 `memory` 数组。

```json
{
  "memory": [
    {
      "round": 5,
      "day": 3,
      "event": "玩家在酒馆帮我付了酒钱",
      "category": "kindness",
      "impact": "affection+5, trust+3",
      "referenced": false
    },
    {
      "round": 12,
      "day": 8,
      "event": "玩家承诺帮我找妹妹，但至今未兑现",
      "category": "broken_promise",
      "impact": "trust-10, suspicion+15",
      "referenced": false
    }
  ]
}
```

### 信息传播机制

NPC 之间通过合法渠道传播信息，玩家在 A 地的行为可能影响 B 地的 NPC。

| 传播渠道 | 速度 | 保真度 | 示例 |
|----------|------|--------|------|
| 旅人传言 | 慢（数天） | 低（会变形夸大） | "听说有个外来者在王港城闹事" |
| 信件/快马 | 中（1-2天） | 高 | 官方通缉令传到邻城 |
| 八卦流言 | 快（半天） | 极低（添油加醋） | "那个人据说杀了十几个人" |
| 派系情报网 | 快（数小时） | 高 | 盗贼公会眼线即时上报 |
| 目击者 | 即时 | 中（主观偏差） | 旁观者亲眼所见但理解有偏差 |

信息传播后，接收 NPC 的 `rumors_heard` 或 `knows` 字段更新，且情感可能受影响。

### 引用过去

NPC 应主动引用过去的交互，让玩家感受到"被记住"：

- **兑现承诺时**："你真的找到了！上次你答应帮我，我还以为只是客套..."
- **未兑现承诺时**："你上次说要帮我找那本书，怎么到现在没消息？"
- **重复犯错时**："这已经是第二次了。第一次我可以当意外，第二次呢？"
- **善行回报时**："你救过我一命，这份恩情我一直记着。今天换我帮你。"

`referenced` 标记用于追踪哪些记忆已被 NPC 在对话中提及，避免重复引用同一条。

## §4 NPC 智能标准

### 不说废话

NPC 对话必须满足以下至少一项，否则不输出：
- 推动剧情（提供新信息/新任务/新方向）。
- 揭示角色（展现 NPC 性格/秘密/情感）。
- 制造张力（制造冲突/悬念/困境）。

禁止"今天天气不错"式的填充对话。若 NPC 确实闲聊，闲聊内容必须暗含信息或伏笔。

### 有议程

NPC 不是等玩家来对话才行动的木偶。每个 NPC 有自己的日程、目标、计划。

```yaml
agenda:
  short_term: "今天要筹集 50 金币还债"
  mid_term: "一周内找到失踪的妹妹"
  long_term: "攒够钱离开这座死气沉沉的城"
  current_action: "正在酒馆向每个进来的人打听妹妹的下落"
```

NPC 的议程意味着：玩家不找他，他也会主动找玩家；玩家帮了他，他会推进自己的计划；玩家阻碍了他，他会记仇并采取对策。

### 会欺骗

NPC 可以撒谎、隐瞒、误导——但谎言必须可被识破。

| 欺骗类型 | 识破途径 |
|----------|----------|
| 直接撒谎 | 与其他 NPC 证词矛盾、物证不符 |
| 隐瞒关键信息 | 调查发现遗漏、NPC 的紧张小动作 |
| 误导性真相 | 说的是真话但引导错误结论，需玩家推理 |
| 伪装身份 | 知识边界出现破绽、旧识者揭穿 |

玩家揭穿谎言后，NPC 反应基于其性格和被揭穿的严重程度（恼羞成怒、崩溃认罪、反咬一口、仓皇逃跑）。

### 会拒绝

NPC 不会无条件满足玩家要求。拒绝理由包括：

- **好感/信任不足**："我凭什么相信你？"
- **自身利益冲突**："这东西对我很重要，不能给你。"
- **恐惧**："不行，那样做会被他们发现的..."
- **道德底线**："我虽然穷，但不偷东西。"
- **能力不足**："我也想帮你，但我真的不知道。"

拒绝不是"死路"——玩家可通过提升好感、威胁、交易、说服等方式改变 NPC 的决定。

### 会成长

NPC 的性格和态度随剧情发展而变化，不是从头到尾一个样。

| 成长类型 | 触发条件 | 变化方向 |
|----------|----------|----------|
| 创伤成长 | 经历重大失去/危险 | 从天真变为谨慎，或从懦弱变得决绝 |
| 信任成长 | 玩家反复兑现承诺 | 从多疑变为信任，开放更多内容 |
| 堕落成长 | 被背叛/环境逼迫 | 从正直变为冷酷，甚至反叛 |
| 觉醒成长 | 得知被欺骗的真相 | 从盲从变为独立思考 |

### 会主动行动

NPC 会主动寻找玩家、设置陷阱、提供帮助——不总是被动等待。

- 玩家曾帮助的 NPC 在玩家遇险时主动出现救援。
- 被玩家得罪的 NPC 雇佣杀手或向当局告密。
- 对玩家有好感的 NPC 主动送来情报或物资。
- 有自己议程的 NPC 在玩家不找他时自行推进计划，下次见面时状态已变。

## §5 NPC 声音协议

为确保每个 NPC 对话有辨识度且不串味，遵循以下协议：

### 声音一致性检查

- [ ] 对话词汇是否符合该 NPC 的 `vocabulary_level`？
- [ ] 句式是否符合 `sentence_style`？
- [ ] 是否自然融入了 `catchphrase`（不生硬）？
- [ ] NPC 透露的信息是否在 `knowledge_boundary` 内？
- [ ] NPC 的情绪反应是否符合当前情感数值？
- [ ] 是否引用了相关记忆（若有未 `referenced` 的高价值记忆）？

### 避免所有 NPC 一个味

| 常见问题 | 修正方法 |
|----------|----------|
| 所有 NPC 都用书面语 | 区分正式/口语/粗俗/学术语域 |
| 所有 NPC 都很友善 | 根据好感度调整态度，有冷有热 |
| 所有 NPC 都知道一切 | 严格按知识边界限制信息 |
| 所有 NPC 都等玩家开口 | 让有议程的 NPC 主动开口 |

> 一个好的 NPC，让玩家在关闭游戏后还会想起它说过的话——这就是 NPC 智能的终极标准。
