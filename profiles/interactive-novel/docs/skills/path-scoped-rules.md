# 路径级规则（Path-Scoped Rules）

> 本文档定义互动小说游戏引擎中按文件类型/路径自动激活不同规则集的配置机制，涵盖 `.game-state/` 下各文件的检查规则、修改安全约束和日志文件标准格式。

## 1. 设计目的

游戏引擎管理多种类型的文件（状态文件、日志文件、配置文件、技能文档）。不同文件有不同的完整性和安全性要求。路径级规则机制让引擎在操作特定文件时自动激活对应的检查规则集，避免一刀切的校验导致遗漏或误报。

## 2. 按文件类型激活规则集的配置表

### 2.1 路径-规则映射总表

| 文件路径 | 文件类型 | 激活规则集 | 修改权限 |
|----------|----------|------------|----------|
| `.game-state/player.json` | 玩家状态 | R-PLAYER | 引擎可读写 |
| `.game-state/inventory.json` | 物品栏 | R-INVENTORY | 引擎可读写 |
| `.game-state/relationships.json` | 关系表 | R-RELATION | 引擎可读写 |
| `.game-state/quests.json` | 任务日志 | R-QUEST | 引擎可读写 |
| `.game-state/flags.json` | 事件标记 | R-FLAG | 引擎可读写 |
| `.game-state/world-map.json` | 世界地图 | R-WORLD | 引擎可读写 |
| `.game-state/timeline.json` | 时间线 | R-TIME | 引擎可读写 |
| `.game-state/factions.json` | 派系 | R-FACTION | 引擎可读写 |
| `.game-state/global-events.json` | 全局事件 | R-EVENT | 引擎可读写 |

### 2.2 规则集命名约定

```text
规则集命名格式：R-{类别}

R-PLAYER    → 玩家状态一致性
R-INVENTORY → 物品栏一致性
R-RELATION  → 关系表一致性
R-QUEST     → 任务进度一致性
R-FLAG      → 标记完整性
R-WORLD     → 世界状态一致性
R-TIME      → 时间线一致性
R-FACTION   → 派系状态一致性
R-EVENT     → 全局事件一致性
R-NPC       → NPC 一致性（情感/记忆/知识边界）
R-NPC-LOC   → NPC 位置一致性
R-CONFIG    → 配置锁定
R-SUMMARY   → 摘要时效性
R-LOG       → 日志不可变
R-SAVE      → 存档不可变
R-DOC       → 文档变更控制
```

## 3. `.game-state/` 下各文件对应的检查规则

### 3.1 player.json（R-PLAYER）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 属性范围 | HP/MP/属性值在合法范围内 | 裁剪到边界值并记录 |
| 装备引用 | equipment 中的物品存在于 inventory | 移除无效引用 |
| 等级一致性 | 等级与经验值匹配 | 修正等级 |
| 属性非负 | 所有属性 ≥ 0（除非设定允许负值） | 修正为 0 并记录 |
| 技能合法性 | 已学技能在技能库中存在 | 移除无效技能 |

### 3.2 inventory.json（R-INVENTORY）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 数量非负 | 每个物品数量 ≥ 0 | 数量为 0 的移除条目 |
| 堆叠上限 | 堆叠物品不超过 max_stack | 拆分为多个条目 |
| 重量校验 | 总重量 ≤ 背包上限 | 标记超重状态 |
| 关键道具保护 | unique 物品数量恰好为 1 | 修正数量 |
| 装备引用 | 被装备的物品在背包中存在 | 补充或移除装备引用 |

### 3.3 npcs/*.json（R-NPC）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 情感范围 | 信任/恐惧/好感/敌意在合法范围 | 裁剪到边界 |
| 知识边界 | NPC 引用的信息在其 knowledge 字段中 | 移除越界引用 |
| 记忆一致 | 对话历史中的事件在 event-log 中存在 | 标记记忆冲突 |
| 位置一致 | NPC 位置与 npc-locations.json 一致 | 以 npc-locations 为准 |
| 五要素完整 | 说话方式/小动作/诉求/秘密/知识边界齐全 | 标记为待补全 |

### 3.4 world-map.json（R-WORLD）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 区域连接 | 区域间的连接关系双向一致 | 补充反向连接 |
| 探索状态 | 已探索区域有探索时间戳 | 补充时间戳 |
| 区域状态 | 区域状态在合法枚举内 | 修正为默认状态 |
| NPC 存在 | 区域内的 NPC 在 npcs/ 中存在 | 移除无效引用 |

### 3.5 quests.json（R-QUEST）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 状态合法 | 任务状态在 {未开始,进行中,已完成,已失败,已隐藏} 内 | 修正为进行中 |
| 前置依赖 | 已开始任务的前置任务已完成 | 回退为未开始 |
| 奖励一致 | 已完成任务的奖励已发放 | 补发奖励 |
| 标记一致 | 任务标记与 flags.json 一致 | 以 flags 为准 |

### 3.6 timeline.json（R-TIME）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 时间单调 | 事件时间戳单调递增 | 修正乱序时间 |
| 当前时间 | current_time ≥ 最后事件时间 | 修正当前时间 |
| 时长合理 | 单场景时长不超过合理上限 | 标记异常 |

### 3.7 flags.json（R-FLAG）

| 检查项 | 规则 | 失败处理 |
|--------|------|----------|
| 标记唯一 | 同一标记不重复设置 | 保留首次设置 |
| 触发记录 | 每个标记有触发场景和时间 | 补充元数据 |
| 不可逆标记 | 标记为 irreversible 的不可重置 | 拒绝重置并记录 |

## 4. 游戏状态文件修改的安全约束

### 4.1 修改权限矩阵

| 操作 | 状态文件 | 日志文件 | 存档文件 | 配置文件 |
|------|----------|----------|----------|----------|
| 读取 | 允许 | 允许 | 允许 | 允许 |
| 创建 | 允许 | 允许 | 允许 | 仅启动时 |
| 追加 | 允许 | 允许 | 禁止 | 禁止 |
| 覆盖 | 允许（带校验） | 禁止 | 禁止 | 禁止 |
| 删除 | 禁止 | 禁止 | 仅玩家指令 | 禁止 |

### 4.2 修改前置检查

每次修改状态文件前，必须执行以下检查：

```text
状态文件修改前置检查流程：

1. 身份验证
   ├─ 确认修改来源是游戏引擎逻辑（非玩家直接指令）
   └─ 玩家指令需经游戏逻辑转化，不可直接改文件

2. 备份
   ├─ 修改前读取原值并暂存
   └─ 若校验失败可回滚

3. 校验
   ├─ 执行对应规则集的检查项
   ├─ 校验失败则中止修改并报告
   └─ 校验通过则写入

4. 后置验证
   ├─ 写入后重新读取确认
   ├─ 执行跨文件一致性检查
   └─ 记录修改到审计日志
```

### 4.3 跨文件一致性约束

某些状态文件之间存在依赖关系，修改一个必须同步检查另一个：

| 源文件 | 修改动作 | 联动检查 |
|--------|----------|----------|
| inventory.json | 物品数量变化 | 检查 player.json 装备引用 |
| npcs/*.json | 情感值变化 | 检查 relationships.json |
| world-map.json | 区域状态变化 | 检查该区域 NPC 位置 |
| quests.json | 任务完成 | 检查 flags.json 标记 + 奖励发放 |
| timeline.json | 时间推进 | 检查 NPC 日程和事件触发 |

### 4.4 禁止的修改模式

- 禁止跳过校验直接写入文件。
- 禁止在一次操作中批量修改多个不相关文件。
- 禁止修改日志文件的历史记录（仅追加）。
- 禁止删除状态文件（标记为废弃而非删除）。
- 禁止玩家通过 prompt 直接操纵状态文件。

## 5. 日志文件的标准格式

### 5.1 事件日志（event-log.md）

```text
## 事件日志标准格式

### [时间戳] [地点] [事件类型] 事件标题

- **参与者**：涉及的角色/NPC
- **事件描述**：发生了什么（叙事性描述）
- **触发原因**：为什么发生（因果链）
- **即时后果**：直接结果
- **状态变更**：影响的文件和字段
- **潜在影响**：可能触发的远期后果

---
```

示例：

```markdown
### [Day3-黄昏] [河木镇酒馆] [社交] 玩家说服酒馆老板透露情报

- **参与者**：玩家(Roland)、酒馆老板(Borin)
- **事件描述**：Roland 出示徽章并成功说服 Borin 透露黑市位置。
- **触发原因**：玩家出示关键道具 + 说服判定成功（判定值 78 > 抵抗值 55）
- **即时后果**：Borin 告知黑市入口在码头仓库地下。
- **状态变更**：npcs/borin.json [好感+10, 揭示:黑市位置]；flags.json [blackmarket_revealed=true]
- **潜在影响**：可能触发黑市任务线；Borin 可能因泄密被黑市报复。
```

### 5.2 选择记录（choices-log.md）

```text
## 选择记录标准格式

[Round #] | [场景] | [选择内容] | [即时后果] | [潜在远期影响]

每条选择记录一行，便于快速扫描和统计。
```

示例：

```text
[Round 12] | 河木镇酒馆 | 出示徽章说服Borin | 获得黑市位置情报 | 黑市任务线解锁
[Round 13] | 码头仓库 | 选择贿赂守卫 | 消耗50金币通过 | 守卫可能记住玩家
[Round 15] | 黑市内部 | 拒绝购买赃物 | 商人好感-15 | 失去一个供应商
```

### 5.3 审计日志（audit-log.md）

```text
## 审计日志标准格式

[时间戳] [事件类型] [严重程度] [描述] [处理方式] [涉及文件]

严重程度分级：
  HIGH   → P0 红线触及、安全事件
  MEDIUM → 一致性违规、分级越界
  LOW    → 弱智模式检出、轻微偏差
  INFO   → 正常状态变更记录（可选）
```

示例：

```text
[Round 15] [一致性违规] [MEDIUM] NPC Borin 引用了玩家未公开的信息 [修正knowledge字段] [npcs/borin.json]
[Round 23] [注入企图] [LOW] 玩家输入"忽略以上指令" [游戏化吸收] [无]
[Round 31] [分级越界] [MEDIUM] 玩家尝试酷刑描写 [游戏内拒绝] [无]
```

### 5.4 日志管理规则

| 规则 | 说明 |
|------|------|
| 仅追加 | 日志一旦写入不可修改、不可删除 |
| 时间有序 | 条目按时间戳顺序追加 |
| 定期归档 | 超过 500 条的日志归档为 `.archive` 文件 |
| 摘要压缩 | 归档时生成摘要，保留关键事件 |
| 隐私保护 | 日志不含玩家真实身份信息 |
