# 对话系统（Dialogue System）

> 本文档定义互动小说游戏引擎中 NPC 对话的完整机制：从输入解析、信息差管理，到说服/欺骗/威胁/贿赂的属性判定，再到对话叙事质量标准与标签系统。

## 1. 设计目标

对话系统是玩家与游戏世界交互的最核心通道。它必须满足三个目标：

1. **沉浸感**：对话读起来像真人在说话，而不是选项菜单。
2. **后果性**：玩家说的每一句话都可能改变 NPC 态度、揭示线索、关闭或打开任务。
3. **公平性**：玩家有足够信息做出有意义的选择，但 NPC 不会无条件倒出所有信息。

## 2. NPC 对话机制

### 2.1 三种输入模式

| 模式 | 触发方式 | 适用场景 | 示例 |
|------|----------|----------|------|
| 自然语言输入 | 玩家自由打字描述对话内容 | 沉浸式体验、高自由度 | "我试探地问酒馆老板最近有没有见过陌生人" |
| 选项选择 | AI 生成 2-4 个对话选项供选 | 关键谈判、节奏紧凑 | A. 直接索要情报 / B. 请他喝酒套话 / C. 威胁他 |
| 关键词触发 | 玩家输入含特定关键词时激活分支 | 调查类、解谜类 | 玩家输入"龙鳞"激活隐藏对话线 |

### 2.2 混合模式规则

默认采用混合模式，遵循以下激活规则：

- **日常寒暄 / 自由探索** → 自然语言输入优先，AI 根据语义解析 NPC 反应。
- **关键决策点**（谈判、质问、表白、招供）→ 自动切换为选项 + 允许自定义。
- **玩家随时可输入** `"我要自定义对话"` 来覆盖给出的选项。
- 选项必须附带"语气提示"（如 `[友好]`、`[讽刺]`、`[威胁]`），让玩家清楚选择的情感色彩。

### 2.3 关键词触发设计

关键词触发不依赖精确匹配，而是语义识别：

```text
关键词库示例（存于 NPC 档案的 "triggers" 字段）：
{
  "triggers": {
    "龙鳞": { "branch": "secret_dragon_line", "min_trust": 20 },
    "失踪的妹妹": { "branch": "sister_quest", "min_trust": -10 },
    "黑市": { "branch": "blackmarket_refuse", "min_trust": 40 }
  }
}
```

- 触发分支需要满足最低信任度门槛，否则 NPC 会回避或装傻。
- 关键词触发后仍走自然语言流程，不是跳转到固定脚本。
- 玩家未提及的关键词不主动暴露——信息差由玩家探索打破。

### 2.4 对话轮次控制

- 单次对话场景建议 3-8 轮，过长会让玩家疲劳。
- NPC 会在合适时机主动结束对话（"我得去干活了"），但不应让玩家感觉被敷衍。
- 玩家可随时用 `"结束对话"` 或转身离开来终止。

## 3. 信息差管理

### 3.1 三方信息边界

游戏世界存在三个独立的信息持有方，信息不默认共享：

| 持有方 | 知道什么 | 不知道什么 |
|--------|----------|------------|
| 玩家 | 自己的经历、背包里的物品、玩家亲眼所见 | NPC 的秘密、未探索区域、隐藏动机 |
| NPC | 自己的背景、本地八卦、自己参与的秘密 | 玩家的真实身份（除非被告知）、其他 NPC 的私密 |
| 世界 | 已发生的大事件、派系动态、时间线 | 玩家的内心想法、未发生的随机事件 |

### 3.2 信息传播规则

信息不会凭空从一方传到另一方，必须有传播渠道：

| 渠道 | 传播方式 | 传播延迟 |
|------|----------|----------|
| 直接告知 | 玩家告诉 NPC / NPC 告诉玩家 | 即时 |
| 目击 | NPC 亲眼看到玩家行为 | 即时 |
| 八卦 | NPC 之间口口相传 | 半天到数天 |
| 信件/通讯 | 派系信使、魔法传讯 | 数小时到数天 |
| 推断 | NPC 根据已知线索推断 | 视智力设定 |

- 玩家在 A 城做的事，B 城的 NPC 不应在无传播渠道的情况下知道。
- NPC 引用信息时，引擎先检查该 NPC 的 `knowledge` 字段是否包含该信息。
- 玩家可主动散布或隐瞒信息——这是合法的博弈手段。

### 3.3 信息揭示节奏

- **不让 NPC 信息倾倒**：一见面就把所有情报倒给玩家是信息差失败。
- **分层揭示**：核心信息需好感度门槛或剧情推进才能解锁。
- **奖励探索**：玩家主动追问、出示证据、提对关键词，应获得额外信息。
- **允许误判**：NPC 可以基于不完整信息做出错误判断，制造戏剧冲突。

## 4. 说服 / 欺骗 / 威胁 / 贿赂机制

### 4.1 判定公式

社交互动的成败基于属性与好感度的综合判定：

```text
判定值 = 基础属性 + 好感度修正 + 情境修正 + 随机波动

基础属性对照：
  说服   → 魅力(CHA) 或 口才
  欺骗   → 智力(INT) 或 巧言
  威胁   → 力量(STR) 或 威吓
  贿赂   → 财富值 或 物品价值

判定阈值（NPC 抵抗值）：
  抵抗值 = NPC意志 + (100 - 好感度) + 秘密重要性系数
```

判定值 ≥ 抵抗值 → 成功；否则失败。结果分三档：

| 结果区间 | 表现 |
|----------|------|
| 大成功（超阈值 20+） | NPC 完全配合，可能额外透露信息 |
| 成功 | NPC 配合，但保留部分底线 |
| 失败 | NPC 拒绝，好感度可能下降 |
| 大失败（低于阈值 20+） | NPC 反感、报警、设陷阱 |

### 4.2 四种社交手段详解

| 手段 | 关键属性 | 成本 | 风险 |
|------|----------|------|------|
| 说服 | 魅力/口才 | 低（仅消耗时间） | 失败仅损失好感少量 |
| 欺骗 | 智力/巧言 | 中（需自圆其说） | 被识破后信任度暴跌 |
| 威胁 | 力量/威吓 | 高（可能结仇） | 失败导致敌意值飙升 |
| 贿赂 | 财富/物品 | 高（消耗资源） | 被拒可能人财两失 |

### 4.3 欺骗的后续验证

欺骗不是一次成功就永久安全：

- NPC 会根据后续事件验证谎言（"你说昨天在南边，怎么南边的人没见过你？"）。
- 谎言被戳穿时，信任度直接归零甚至转负，且难以挽回。
- 玩家可以主动用新证据修补谎言，但难度递增。
- 多个 NPC 之间的信息交叉验证会提高戳穿概率。

### 4.4 威胁的副作用

- 威胁成功只是表面顺从，NPC 可能记恨并伺机报复。
- 威胁过的 NPC 在后续剧情中可能反水、报警、设陷阱。
- 高恐惧度 NPC 会顺从但提供不可靠信息（故意误导）。
- 威胁行为本身可能被其他 NPC 目击并影响玩家声誉。

## 5. 对话叙事质量标准

### 5.1 好对话的特征

- **推动剧情**：每段对话至少推进一个信息点或情感转折。
- **塑造角色**：通过用词、语气、小动作展现 NPC 性格。
- **制造张力**：信息差、利益冲突、隐藏动机制造对话张力。
- **留有钩子**：对话结尾留一个未解之谜或新线索。

### 5.2 禁止的对话模式

| 禁止模式 | 说明 | 正确做法 |
|----------|------|----------|
| 信息倾倒 | NPC 一开口倒出全部情报 | 分层揭示，奖励追问 |
| 万能应答 | 无论玩家说什么都回"好的冒险者" | 根据 NPC 性格差异化回应 |
| 无差别礼貌 | 所有 NPC 都客气周到 | 贵族傲慢、酒鬼粗鲁、学者迂腐 |
| 破坏人设 | 严肃角色突然开玩笑 | 保持性格一致性 |
| 选择幻觉 | 给 3 个选项但结果相同 | 每个选项有实质不同后果 |

### 5.3 对话描写规范

- NPC 台词用引号包裹，描写用括号或独立段落。
- 每段对话至少包含一处 NPC 的小动作或表情（说话方式体现性格）。
- 避免长篇独白——超过 4 行的台词要拆分为对话往来。
- 玩家的话语由玩家输入，AI 不替玩家说话。

```text
对话格式示例：

酒馆老板擦着杯子，目光没有抬起来。
"陌生人，"他声音沙哑，"这一带最近不太平。你最好别问太多。"
（他握杯子的手指微微收紧——你知道他在隐瞒什么。）

> 选项：
> A. [友好] "我只是路过，想找个住处。"
> B. [追问] "不太平？发生什么事了？"
> C. [出示证据] 把捡到的徽章放在柜台上。
```

## 6. 对话标签系统

### 6.1 标签分类

对话标签用于标记对话的属性，便于状态追踪和后续引用：

| 标签类别 | 标签示例 | 作用 |
|----------|----------|------|
| 语气标签 | `[友好]` `[讽刺]` `[威胁]` `[哀求]` | 影响 NPC 情感反应 |
| 结果标签 | `[说服成功]` `[欺骗失败]` `[贿赂接受]` | 记录社交判定结果 |
| 信息标签 | `[揭示:秘密A]` `[获得:线索B]` `[触发:任务C]` | 标记信息流动 |
| 关系标签 | `[好感+15]` `[信任-30]` `[结仇]` | 记录关系变化 |
| 状态标签 | `[撒谎中]` `[被识破]` `[恐吓中]` | 标记对话进行状态 |

### 6.2 标签存储

对话标签写入 `.game-state/npcs/{name}.json` 的交互历史段落：

```json
{
  "dialogue_history": [
    {
      "round": 12,
      "scene": "酒馆",
      "player_input": "出示徽章追问",
      "tags": ["[出示证据]", "[揭示:黑市位置]", "[好感+10]"],
      "npc_response": "他瞳孔一缩，低声说：'你从哪弄到的……'"
    }
  ]
}
```

### 6.3 标签的后续影响

- 标签会被 NPC 记忆系统引用："上次你出示徽章吓到我了"。
- 跨 NPC 的标签传播：对 A 撒的谎被 B 通过八卦得知，B 的对话会带 `[知情]` 标签。
- 标签累积影响结局判定：`[撒谎次数]`、`[威胁次数]` 影响道德向结局。
- 玩家可用 `/recap` 看到近期重要对话标签摘要。

## 7. 与其他系统的联动

| 联动系统 | 联动方式 |
|----------|----------|
| NPC AI 系统 | 读取 NPC 五要素和情感模型生成反应 |
| 任务系统 | 对话可触发、推进、失败任务 |
| 物品系统 | 出示/赠送物品影响对话走向 |
| 状态机 | 对话结果写入对应状态文件 |
| 一致性检查 | 验证 NPC 引用的信息是否在其知识边界内 |

## 8. 检查清单

- [ ] 每个 NPC 对话是否符合其说话方式和性格？
- [ ] 信息揭示是否分层，有无信息倾倒？
- [ ] 社交判定是否记录了属性、好感度、随机因素？
- [ ] 欺骗是否有后续验证机制？
- [ ] 对话标签是否正确写入 NPC 档案？
- [ ] 对话是否推动了剧情或塑造了角色？
- [ ] 选项是否每个都有实质不同后果？
