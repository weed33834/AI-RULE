# 世界模拟技能 (World Simulation)

> 游戏世界不是静态背景，而是一个活的、会变化的世界：NPC 有日程、派系有计划、事件在发生——无论玩家是否在场。
> 世界状态文件结构详见 `game-state-machine.md` §2；NPC 日程与位置详见 `npc-ai.md`；因果链一致性检查详见 `narrative-coherence.md`。

## §1 世界独立运行原则

世界存在的根本前提：它不围着玩家转。玩家是世界的参与者，不是世界的中心。

### 独立运行的三大支柱

| 支柱 | 说明 | 表现 |
|------|------|------|
| NPC 日程 | 每个 NPC 有自己的每日作息和长期计划 | 玩家找铁匠时他可能在午休；过几天再找他，他已出门进货 |
| 派系计划 | 各派系有自己的目标和行动时间表 | 玩家不管盗贼公会，他们也会自行扩张势力、发动袭击 |
| 事件发生 | 世界事件按自身逻辑发生，不依赖玩家触发 | 无论玩家是否在场，疫病会扩散、战争会爆发、节庆会举行 |

### 玩家不在场时发生了什么

玩家离开 A 地前往 B 地期间，A 地并非"暂停"。模拟引擎推演 A 地的变化：

```json
{
  "offscreen_simulation": {
    "region": "kingsport",
    "player_left_day": 10,
    "player_returns_day": 15,
    "events_during_absence": [
      { "day": 11, "event": "疫病扩散至东区，封锁区扩大" },
      { "day": 13, "event": "盗贼公会趁乱洗劫了粮仓" },
      { "day": 14, "event": "治愈师 Elena 因过度劳累病倒" }
    ],
    "state_changes": {
      "region_state": "plague_worsening",
      "food_supply": "critical",
      "elena_health": "ill"
    }
  }
}
```

玩家返回时，通过环境描写和 NPC 对话体现这些变化——让玩家感到"我离开的这段时间，世界没闲着"。

## §2 因果关系链

事件 A 导致事件 B。世界的每个变化都应有可追溯的因果链。

### 因果链结构

```
玩家烧毁了古森林的桥
  ├→ 直接后果：桥无法通行，必须绕路
  ├→ 二级后果：依赖桥梁贸易的村庄经济受损
  ├→ 三级后果：村庄怨恨"纵火者"，对玩家敌意上升
  └→ 远期后果：森林东侧的盗贼因桥梁断绝而无法被围剿，势力壮大
```

### 因果链设计原则

| 原则 | 说明 |
|------|------|
| 可追溯 | 每个世界事件能追溯到其触发原因（玩家行为或世界自身演进） |
| 合理性 | 后果必须从原因逻辑推出，不可"为了剧情需要"强行联系 |
| 衰减性 | 因果影响随距离（时间/空间/关联度）衰减，不无限蔓延 |
| 多因一果 | 同一结果可由多个原因促成，增加世界真实感 |
| 不可逆性 | 某些后果不可撤销（烧毁的桥不会自己重建），增强选择权重 |

### 因果矛盾检测

- 若叙事中出现"无因之果"（事件无触发原因），标记为因果矛盾。
- 若叙事中出现"因后果矛盾"（原因与结果逻辑不符），暂停并修正。
- 因果链记录在 `.game-state/causality-log.json`，供一致性检查调用。

## §3 时间推进系统

### 昼夜系统

| 时段 | 游戏内小时 | 环境特征 | 影响 |
|------|-----------|----------|------|
| 黎明 | 5-7 | 光线微弱、雾气、露水 | 某些 NPC 起床、夜间生物退散 |
| 白昼 | 7-17 | 明亮、热闹 | 商店营业、NPC 按日程活动 |
| 黄昏 | 17-19 | 夕阳、人潮归家 | 酒馆开始热闹、赶路者寻找住宿 |
| 夜晚 | 19-5 | 黑暗、安静、危险 | 宵禁区域巡逻加强、夜间生物出没、犯罪率上升 |

### 季节系统

| 季节 | 持续天数 | 环境影响 | 游戏机制影响 |
|------|----------|----------|-------------|
| 春 | 30 | 万物复苏、融雪、道路泥泞 | 旅行速度降低、草药丰富 |
| 夏 | 30 | 炎热、暴雨、作物生长 | 体力消耗加快、食物充足 |
| 秋 | 30 | 凉爽、丰收、落叶 | 交易繁荣、疫病高发 |
| 冬 | 30 | 严寒、大雪、食物稀缺 | 旅行危险、物价上涨、需御寒装备 |

### NPC 日程推进

每个 NPC 有每日日程，随时间推进而切换活动：

```json
{
  "schedule": {
    "morning": { "time": "6-12", "activity": "treat_patients", "location": "chapel" },
    "afternoon": { "time": "12-18", "activity": "gather_herbs", "location": "forest_edge" },
    "evening": { "time": "18-22", "activity": "rest", "location": "chapel_quarters" },
    "night": { "time": "22-6", "activity": "sleep", "location": "chapel_quarters" }
  },
  "exceptions": [
    { "condition": "疫病封锁", "override": "全天在教堂治疗患者" },
    { "condition": "满月之夜", "override": "夜间前往古森林采集月光草" }
  ]
}
```

玩家在不同时段找同一 NPC，会遇到不同的场景——这增强了世界的真实感。

## §4 信息传播机制

玩家在 A 地的行为，会通过合法渠道传播到 B 地的 NPC。

### 传播渠道详表

| 渠道 | 传播速度 | 信息保真度 | 影响范围 | 示例 |
|------|----------|-----------|----------|------|
| 旅人传言 | 1-3 天 | 低（夸大变形） | 沿交通线城镇 | "听说有个外地人在王港城大闹一场" |
| 信件/快马 | 1-2 天 | 高（原文传达） | 有通信网络处 | 官方通缉令、贵族私信 |
| 八卦流言 | 半天-1天 | 极低（添油加醋） | 同城/邻城 | "那个人据说一人杀了十条龙" |
| 派系情报网 | 数小时 | 高（精准情报） | 派系势力范围 | 盗贼公会眼线即时上报 |
| 目击者 | 即时 | 中（主观偏差） | 目击者社交圈 | "我亲眼看见他动的手" |
| 商队传播 | 3-7 天 | 中低 | 商路沿线城镇 | 随货物一同传播的见闻 |

### 信息传播的状态更新

```json
{
  "information_spread": {
    "event": "玩家在王港城击败盗贼头目",
    "origin": "kingsport",
    "origin_day": 10,
    "spread_log": [
      { "region": "old_forest", "channel": "旅人传言", "arrived_day": 12, "version": "一个外来者打跑了盗贼团" },
      { "region": "harbor_district", "channel": "八卦流言", "arrived_day": 11, "version": "有人单挑了整个盗贼公会" },
      { "region": "thieves_guild_hideout", "channel": "派系情报网", "arrived_day": 10, "version": "精确的战斗细节和玩家外貌" }
    ]
  }
}
```

同一事件经不同渠道传播，到达不同 NPC 时版本不同——这制造了信息差和误解的戏剧空间。

## §5 蝴蝶效应设计

看似微小的选择可能在后续产生重大影响。

### 蝴蝶效应设计原则

| 原则 | 说明 |
|------|------|
| 微小起点 | 触发行为看似无关紧要（帮路人指路、随手关上门） |
| 链式放大 | 通过多级因果链逐步放大影响 |
| 合理推演 | 每一级放大都有逻辑支撑，不可"硬跳" |
| 延迟爆发 | 后果在数个场景甚至章节后才显现 |
| 不可预测 | 玩家无法在做出选择时预见远期后果 |

### 蝴蝶效应链示例

```
第3轮：玩家在酒馆随手帮一个醉汉付了酒钱（好感+2，看似无关紧要）
  ↓
第8轮：醉汉其实是退役的情报贩子，感激玩家，将一条关键情报"免费"透露
  ↓
第15轮：这条情报让玩家提前得知了暗杀计划，救了一位重要 NPC
  ↓
第25轮：被救的 NPC 后来成为关键盟友，在终局战中提供决定性援助
```

### 蝴蝶效应追踪

```json
{
  "butterfly_chains": [
    {
      "seed_action": "帮醉汉付酒钱",
      "seed_round": 3,
      "chain": [
        { "round": 8, "link": "醉汉透露暗杀情报", "status": "realized" },
        { "round": 15, "link": "提前获知并阻止暗杀", "status": "realized" },
        { "round": 25, "link": "被救者成为终局盟友", "status": "pending" }
      ]
    }
  ]
}
```

`status: pending` 的链环在满足条件后激活。若中间链环未实现（玩家走了不同路线），后续链环取消。

## §6 派系系统

### 派系四要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 目标 | 派系想要达成什么 | 皇家卫队：维持秩序；盗贼公会：扩张势力 |
| 资源 | 派系拥有的筹码 | 兵力/情报/金钱/人脉/领地 |
| 关系 | 与其他派系的关系 | 敌对/中立/联盟/附属 |
| 动态变化 | 派系状态随时间和玩家行为演变 | 实力增减、关系转变、目标调整 |

### 派系动态变化

```json
{
  "faction_dynamics": {
    "thieves_guild": {
      "power_trajectory": [
        { "day": 1, "power": 30, "note": "游戏开始，势力较弱" },
        { "day": 10, "power": 45, "note": "趁疫病扩张，洗劫粮仓" },
        { "day": 15, "power": 25, "note": "玩家揭发了其藏身处，遭围剿" }
      ],
      "current_goal": "报复告密者，重建势力",
      "attitude_to_player": -60,
      "relation_changes": [
        { "day": 12, "target": "royal_guard", "from": "hostile", "to": "open_war" }
      ]
    }
  }
}
```

### 玩家与派系的互动

- 帮助一个派系可能得罪另一个（零和博弈）。
- 派系实力对比因玩家行为而变化（帮盗贼公会抢劫→卫队实力下降）。
- 派系间的战争、联盟、背叛是动态的，非预设脚本。
- 玩家可成为派系成员、甚至改变派系走向（推翻首领、促成和谈）。

## §7 区域状态系统

### 区域状态类型

| 状态 | 特征 | 可遇 NPC | 可获物品 | 可触发事件 |
|------|------|----------|----------|-----------|
| 和平 | 秩序井然、商业繁荣 | 商人、卫兵、平民 | 正常商品 | 节庆、集市、日常纠纷 |
| 战乱 | 战斗频发、难民逃亡 | 士兵、难民、雇佣兵 | 武器、战利品 | 战斗、劫掠、求救 |
| 瘟疫 | 封锁、恐慌、药价飞涨 | 治愈师、守卫、病患 | 药材（高价） | 隔离、哄抢、人道主义危机 |
| 繁荣 | 大兴土木、机会涌现 | 工匠、贵族、投机者 | 奢侈品、稀有材料 | 投资、攀附、阴谋 |
| 废墟 | 荒凉、危险、遗物散落 | 流浪者、拾荒者、怪物 | 古代遗物（有风险） | 探索、遭遇、发现秘密 |

### 区域状态转换

区域状态可因玩家行为或世界事件而转换：

| 触发条件 | 状态转换 | 示例 |
|----------|----------|------|
| 玩家灭火/平息动乱 | 战乱 → 和平 | 玩家击败盘踞的盗贼团 |
| 玩家引发暴乱 | 和平 → 战乱 | 玩家煽动民众反抗 |
| 疫病自然扩散 | 和平 → 瘟疫 | 时间推进，疫病蔓延 |
| 玩家找到解药 | 瘟疫 → 和平 | 玩家分发解药，控制疫情 |
| 战争摧毁 | 繁荣 → 废墟 | 派系战争波及该区域 |

区域状态变化后，场景描述必须体现新状态的特征——同一个地点在不同状态下呈现完全不同的面貌。

> 世界模拟的目标，是让玩家在每一次回头时，都发现世界已经不再是离开时的样子——这才是一个活着的世界。
