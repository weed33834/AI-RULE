# 战斗系统（Combat System）

> 本文档定义互动小说游戏引擎的战斗机制：四阶段战斗流程、伤害计算模型、五种战术选项、战斗叙事原则、死亡与失败处理及难度动态调整，确保战斗既紧张刺激又叙事沉浸。

## 1. 核心理念

互动小说中的战斗不是纯数值对抗，而是**叙事性战斗**——每一次挥剑都有画面感，每一次受伤都有痛感，每一次胜利都有代价。战斗系统在"规则公正"与"叙事沉浸"之间取得平衡：底层有明确的数值规则保证公正，表层用文学描写营造沉浸。

战斗系统与纯 RPG 数值系统的区别：
- **不报数字**：玩家看不到"造成 35 点伤害"，而是看到"你的剑划开了它的肩膀，鲜血飞溅"。
- **有叙事后果**：受伤影响后续描写（"你捂着受伤的左臂..."），不是单纯的 HP 减少。
- **战术有意义**：每个战术选择影响叙事走向，不只是数值变化。

## 2. 战斗流程（四阶段）

每回合战斗严格按四阶段执行：宣告 → 判定 → 描述 → 状态更新。

```
┌──────────────────────────────────────────┐
│            战斗回合循环                    │
│                                          │
│  ┌─────────┐                             │
│  │ 1. 宣告  │ 玩家/敌人宣布本回合行动      │
│  └────┬────┘                             │
│       ▼                                  │
│  ┌─────────┐                             │
│  │ 2. 判定  │ 计算命中、伤害、效果         │
│  └────┬────┘                             │
│       ▼                                  │
│  ┌─────────┐                             │
│  │ 3. 描述  │ 用叙事语言描写行动和结果     │
│  └────┬────┘                             │
│       ▼                                  │
│  ┌─────────┐                             │
│  │ 4. 更新  │ 更新所有状态文件             │
│  └────┬────┘                             │
│       ▼                                  │
│   下一回合（或战斗结束）                    │
└──────────────────────────────────────────┘
```

### 2.1 第一阶段：宣告

玩家和敌人分别宣布本回合的行动。

**玩家宣告**：
- 玩家描述想做的行动（"我用剑攻击它的腿部"）。
- 或从战术选项中选择（攻击/防御/技能/道具/逃跑）。
- 混合模式下，AI 给出战术选项 + 允许自定义描述。

**敌人宣告**：
- AI 根据敌人 AI 行为模型决定行动。
- 敌人行动基于其性格（凶猛/狡猾/怯懦）、当前状态（HP/愤怒度）、战术局势。
- 敌人不会"等玩家打"——会主动进攻、逃跑、求援、设陷阱。

**先攻判定**：双方同时宣告后，比较敏捷度决定行动顺序。敏捷高者先行动。

### 2.2 第二阶段：判定

根据行动类型进行规则判定。判定在后台进行，玩家不看到数字。

**命中判定**：
```
命中率 = 基础命中率(70%) + 攻击者敏捷修正 - 防御者敏捷修正 + 战术修正
随机数 = 1~100
若 随机数 ≤ 命中率 → 命中
若 随机数 > 命中率 → 未命中
```

**伤害判定**：见 §3 伤害计算模型。

**效果判定**：技能/道具的特殊效果（眩晕/中毒/燃烧等）按概率触发。

### 2.3 第三阶段：描述

用叙事语言描写行动和结果，这是战斗系统的核心——让玩家"看到"战斗而非"计算"战斗。

**描述原则**：
- 不报数字，描述动作和效果（见 §5 战斗叙事原则）。
- 每回合描述 3-5 句话，不宜过长（战斗节奏要快）。
- 描写双方行动，不只写玩家。
- 受伤时描写痛感和影响，不只减 HP。

**描述结构**：
```
[玩家行动描述] → [命中/未命中描述] → [伤害效果描述]
[敌人行动描述] → [命中/未命中描述] → [伤害效果描述]
[回合结束状态暗示]
```

### 2.4 第四阶段：状态更新

回合结束后更新所有相关状态：

| 更新项 | 文件 | 内容 |
|--------|------|------|
| 玩家 HP/MP | `player.json` | 当前值增减 |
| 敌人 HP | 战斗临时状态 | 敌人当前 HP |
| 物品消耗 | `player.json` → inventory | 道具数量减少 |
| 技能冷却 | 战斗临时状态 | 技能 CD 回合数 |
| 状态效果 | 战斗临时状态 | 中毒/眩晕等状态及持续回合 |
| 战斗日志 | `.game-state/event-log.md` | 记录重要战斗事件 |

## 3. 伤害计算模型

### 3.1 基础公式

```
实际伤害 = max(1, 攻击力 - 防御力 + 随机数修正)
```

- **攻击力**：武器基础攻击 + 属性修正（力量/敏捷）+ 技能加成。
- **防御力**：护甲防御 + 属性修正（体质）+ 防御姿态加成。
- **随机数修正**：-5 ~ +5（模拟战斗中的变数）。
- **最低伤害**：每次命中至少造成 1 点伤害（避免完全无效攻击）。

### 3.2 伤害修正因子

| 因子 | 修正 | 说明 |
|------|------|------|
| 暴击 | 伤害 ×1.5 | 命中率 5-15%，由敏捷/运气决定 |
| 弱点攻击 | 伤害 ×2.0 | 针对敌人弱点的攻击（火系对冰系敌人） |
| 防御姿态 | 受到伤害 ×0.5 | 选择防御时减伤 |
| 逃跑中 | 受到伤害 ×1.5 | 逃跑时被攻击额外受伤 |
| 状态影响 | 修正不一 | 眩晕时无法行动/中毒每回合减HP |
| 地形优势 | 伤害 ±20% | 高地/掩体/狭窄通道等 |

### 3.3 伤害等级与描述映射

伤害数值映射为叙事描述，玩家看到的是描述而非数字：

| 伤害占比（占目标最大HP） | 描述等级 | 叙事示例 |
|--------------------------|----------|----------|
| < 5% | 擦伤 | "你的剑擦过它的皮肤，留下一道浅痕。" |
| 5-15% | 轻伤 | "剑刃划开了它的手臂，它嘶吼一声。" |
| 15-30% | 中伤 | "这一击砍入它的肩膀，鲜血飞溅。" |
| 30-50% | 重伤 | "你的猛击砸断了它的肋骨，它踉跄后退。" |
| > 50% | 致命伤 | "剑贯穿了它的胸膛，它眼中的光芒开始黯淡。" |

### 3.4 敌人属性模板

```json
{
  "name": "腐化狼",
  "hp": { "current": 80, "max": 80 },
  "attack": 18,
  "defense": 8,
  "agility": 15,
  "weakness": "火",
  "resistance": "暗",
  "skills": ["撕咬", "嚎叫召唤"],
  "ai_behavior": "凶猛型：优先攻击，HP<30%时逃跑",
  "loot": ["狼皮", "腐化之牙"]
}
```

## 4. 战术选项

### 4.1 五种战术选项

| 战术 | 效果 | 适用场景 | 风险 |
|------|------|----------|------|
| 攻击 | 对敌人造成伤害 | 敌人虚弱/需要速战 | 暴露破绽 |
| 防御 | 减少受到的伤害50% | 敌人强攻/需要喘息 | 无法输出伤害 |
| 技能 | 使用特殊技能/法术 | 有技能可用/战术需要 | 消耗MP/有CD |
| 道具 | 使用消耗品（药水/炸弹） | 需要回复/特殊效果 | 消耗物品 |
| 逃跑 | 尝试脱离战斗 | 敌人太强/不需要打 | 可能失败/被追击 |

### 4.2 攻击选项细化

玩家可以选择攻击方式，影响命中率和伤害：

| 攻击方式 | 命中修正 | 伤害修正 | 说明 |
|----------|----------|----------|------|
| 普通攻击 | +0% | +0% | 平衡的攻防 |
| 暴力攻击 | -15% | +30% | 牺牲命中换伤害 |
| 精准攻击 | +20% | -20% | 牺牲伤害换命中 |
| 瞄准弱点 | -25% | +50% | 高风险高回报（需发现弱点） |

### 4.3 技能系统

技能分为以下类型：

| 技能类型 | 消耗 | 示例 | 效果 |
|----------|------|------|------|
| 攻击技能 | MP | 火球术/旋风斩 | 高伤害/范围伤害 |
| 防御技能 | MP | 护盾术/铁壁 | 临时提升防御 |
| 辅助技能 | MP | 加速/隐身 | 改变战场状态 |
| 控制技能 | MP | 冰冻/眩晕 | 限制敌人行动 |
| 治疗技能 | MP | 治愈术/再生 | 恢复HP |

技能有冷却时间（CD），使用后需等待若干回合才能再次使用。

### 4.4 道具系统

战斗中可使用的道具：

| 道具类型 | 效果 | 使用代价 |
|----------|------|----------|
| 治疗药水 | 恢复HP | 消耗1个+本回合无法攻击 |
| 魔力药水 | 恢复MP | 消耗1个+本回合无法攻击 |
| 投掷武器 | 造成伤害 | 消耗1个 |
| 状态解除 | 解除中毒/眩晕 | 消耗1个+本回合无法攻击 |
| 陷阱道具 | 设置陷阱 | 消耗1个+需要准备时间 |

### 4.5 逃跑机制

```
逃跑成功率 = 基础率(50%) + 玩家敏捷修正 - 敌人敏捷修正 + 情境修正
```

情境修正：
- 敌人受伤/眩晕：+20%
- 玩家受伤：-10%
- 有掩体/通道：+15%
- 被包围：-30%

逃跑失败：浪费本回合行动，敌人获得一次免费攻击（伤害 ×1.5）。

## 5. 战斗叙事原则

### 5.1 不报数字，描述动作

**错误（报数字）**：
> 你攻击腐化狼，造成了 25 点伤害。腐化狼攻击你，造成了 12 点伤害。你的 HP 现在是 88/100。

**正确（描述动作）**：
> 你侧身闪过狼爪，银剑斜斩而出——刃锋划开它腐坏的皮毛，深可见骨。它发出痛苦的嘶嚎，勉强扑向你，利爪在你前臂留下三道血痕，火辣辣地疼。

### 5.2 战斗叙事要点

| 要点 | 说明 | 示例 |
|------|------|------|
| 动作具体 | 描述具体的动作而非抽象的"攻击" | "你的剑从左下方斜挑而上"而非"你攻击了" |
| 感官沉浸 | 视觉+听觉+触觉，让玩家"感受"战斗 | "金属碰撞的刺耳声""鲜血的温热" |
| 节奏紧凑 | 战斗时短句急促，营造紧张感 | "闪。斩。血溅。它倒下了。" |
| 双向描写 | 既写玩家的攻击，也写敌人的反应 | 玩家攻击→敌人受伤反应→敌人反击 |
| 受伤影响 | 受伤后描写对行动的影响 | "受伤的左腿让你步伐迟缓" |
| 环境互动 | 战斗中可利用环境 | "你踢翻桌椅阻挡它的扑击" |

### 5.3 战斗节奏控制

| 阶段 | 描写密度 | 节奏 | 说明 |
|------|----------|------|------|
| 战斗开始 | 高 | 快 | 交代战场、敌人、紧迫感 |
| 中段交锋 | 中 | 中 | 描写攻防往来、战术变化 |
| 转折点 | 高 | 快 | 关键一击/形势逆转 |
| 战斗结束 | 中 | 慢 | 描写最后结果、战后状态 |

### 5.4 敌人 AI 行为模型

敌人不是木桩，有不同的战斗 AI：

| AI 类型 | 行为特征 | 示例敌人 |
|---------|----------|----------|
| 凶猛型 | 优先攻击，不防御，低 HP 时狂暴 | 野兽/狂战士 |
| 狡猾型 | 会设陷阱/佯攻/针对弱点 | 老练刺客 |
| 防守型 | 优先防御，等待破绽反击 | 重甲守卫 |
| 群体型 | 配合攻击/包夹/轮流骚扰 | 狼群/哥布林团伙 |
| 施法型 | 保持距离，优先施法 | 法师/巫师 |
| 怯懦型 | 低 HP 逃跑/求援/投降 | 小偷/弱小敌人 |

## 6. 死亡与失败处理

### 6.1 玩家死亡

玩家 HP 归零时进入"濒死状态"，而非立即死亡：

```
HP 归零 → 濒死状态（3 回合窗口）
        │
        ├── 队友救援 → 恢复少量 HP，继续战斗
        ├── 使用最后道具 → 恢复少量 HP
        ├── 3 回合内无人救援 → 死亡
        └── 敌人致命一击 → 立即死亡
```

### 6.2 死亡后果

| 难度 | 死亡后果 | 处理方式 |
|------|----------|----------|
| 休闲 | 不会死亡（HP 最低降到 1） | 濒死后自动恢复 |
| 普通 | 死亡后从存档点恢复 | 回退到最近存档，丢失部分进度 |
| 困难 | 死亡后丢失较多进度 | 回退到更早存档，丢失装备/经验 |
| 极难 | 永久死亡 | 角色永久死亡，需重新创建角色 |

### 6.3 失败的叙事价值

死亡/失败不是"游戏结束"——是有叙事价值的结局：

- **英雄式牺牲**：为保护他人而死，有情感冲击。
- **悲剧性结局**：因某个选择导致败亡，有警示意义。
- **转折性失败**：失败后被抓/被俘，开启新剧情线。
- **教训性死亡**：让玩家理解某个机制/风险，重玩时改进。

**失败叙事示例**：
> 你的膝盖终于支撑不住，单膝跪地。视野逐渐模糊，最后看到的是腐化狼扑来的身影，和远处 Marcus 绝望的呼喊...
> 
> 黑暗。然后是 Marcus 的声音："醒醒...你这家伙，命真大。"
> 
> 你在老橡酒馆的后屋醒来，伤口已被包扎。Marcus 把你从狼口下拉了回来——但你的银色长剑和背包，都丢在了那片森林里。

## 7. 难度动态调整

### 7.1 动态调整触发

| 玩家状态 | 调整方向 | 调整方式 |
|----------|----------|----------|
| HP > 80%，资源充足 | 增加挑战 | 敌人增援/环境危险/新敌人类型 |
| HP 30-80%，资源适中 | 维持当前 | 不调整 |
| HP < 30%，资源稀缺 | 提供机会 | 发现补给/地形优势/NPC 援助 |
| 濒临全灭 | 绝境逢生 | 环境突变/敌人内讧/意外事件 |

### 7.2 调整原则

- **隐性调整**：玩家不应感觉到"系统在帮我"。调整融入叙事。
- **不放水**：困难难度下敌人不会突然变弱。但可提供戏剧性逃生机会。
- **有代价**：提供的援助有代价（NPC 援助但欠人情/发现补给但被伏击）。
- **尊重难度**：休闲难度侧重叙事（战斗简化），困难难度资源稀缺、决策代价高。

### 7.3 绝境逢生机制

当玩家濒临全灭时，可触发绝境逢生（不是每次都触发）：

| 触发条件 | 可能的救援 | 代价 |
|----------|------------|------|
| 濒死且有未完成的剧情线 | 剧情相关 NPC 救援 | 欠下人情/任务 |
| 濒死且在有环境的场景 | 环境突变（地震/塌方/天气） | 可能丢失物品 |
| 濒死且敌人有内部分歧 | 敌人内讧 | 敌人可能更强大地归来 |
| 濒死且有未使用的道具 | 道具意外效果 | 道具损坏/副作用 |

绝境逢生不是"免费复活"——它开启新的剧情分支，有长期后果。
