# 玩家自主权技能 (Player Agency)

> 玩家自主权是互动小说的灵魂：没有真正的选择自由，就不是游戏，只是阅读——玩家的每一个行动都必须被世界认真对待。
> 分支模式与结局系统详见 `branching-narrative.md`；难度对选择的影响详见 `difficulty-engine.md`；Yes-and 执行细节详见 `docs/prompts/game-master.md`。

## §1 三种输入模式

不同的游戏类型和玩家偏好需要不同的输入模式。模式可在游戏开始时选择，也可在过程中切换。

### 1.1 自由输入模式

玩家用自然语言描述想做的任何行动，AI 解析并在世界规则内响应。

| 维度 | 说明 |
|------|------|
| 体验特征 | 最高沉浸感，玩家完全以角色身份思考和行动 |
| 适合场景 | 探索、角色扮演、创意解法、高自由度游戏 |
| 优势 | 玩家不被选项束缚，可尝试任何合理行动 |
| 挑战 | AI 需准确解析含糊输入；可能产生意料外行动 |
| 处理含糊输入 | 给出 2-3 种解读供玩家选择，不自行脑补 |

**示例交互**：
```
玩家：我想看看这个房间里有没有什么能用的东西，特别是能当武器的东西。
AI：（描述搜索过程与发现）
```

### 1.2 选择式模式

AI 在每个节点给出 2-4 个选项，玩家从中选择。节奏紧凑，适合关键决策点。

| 维度 | 说明 |
|------|------|
| 体验特征 | 节奏快，决策清晰，降低认知负担 |
| 适合场景 | 战斗、谈判、时间紧迫场景、新手玩家 |
| 优势 | 选项经过设计，确保每个都有意义 |
| 挑战 | 可能限制玩家创意；需避免"假选择" |
| 选项设计 | 见 §2 选择设计五原则 |

**示例交互**：
```
守卫挡住了去路。他看起来不太聪明，但手里握着一把长矛。
  A. 编一个紧急理由，试图蒙混过关
  B. 贿赂他（需要 10 金币）
  C. 绕到侧面寻找其他入口
  D. [自定义行动]
```

### 1.3 混合模式（推荐）

默认自由输入，在关键节点自动切换为选择式。兼顾沉浸感与节奏。

| 场景类型 | 使用模式 | 理由 |
|----------|----------|------|
| 日常探索 | 自由输入 | 给予探索自由 |
| 战斗回合 | 选择式 + 自定义 | 战斗需明确行动，但允许创意战术 |
| 重大道德抉择 | 选择式 | 确保选项经过精心设计 |
| NPC 深度对话 | 自由输入 | 对话应自然流动 |
| 谈判/说服 | 选择式 + 自定义 | 关键说服点需设计，但允许玩家自定义论点 |

- 玩家随时可说"我要自定义行动"覆盖给出的选项。
- 混合模式始终在选项列表末尾保留 `[自定义行动]` 入口。

## §2 选择设计五原则

### 原则一：每个选择都有后果

不存在"正确答案"——每个选择都有得有失。

| 选择 | 获得 | 失去 |
|------|------|------|
| 救人质 | 道德满足、人质感激 | 让凶手逃脱、未来遭遇更强敌人 |
| 追凶手 | 制止威胁、获得线索 | 人质死亡、良心谴责、NPC 关系恶化 |
| 两不相帮 | 保全自身、保持中立 | 双方都不信任你、失去两边的资源 |

**禁忌**：不可设计一个"完美选项"——所有选项都获得奖励无代价。这会让选择失去意义。

### 原则二：避免假选择

不能多个选项导向完全相同的结果。如果需要收束，让路径本身不同。

| 假选择（禁止） | 真选择（正确） |
|----------------|----------------|
| "左转/右转"——两条路完全一样 | "左转走大路（安全但慢）/右转穿森林（快但危险）" |
| 三个选项最终都到达同一房间、同一状态 | 三个选项到达同一房间，但分别带来情报/物品/伤害 |
| "问A/问B"——两人给出完全相同的信息 | "问A（得到官方说法）/问B（得到内部真相）" |

**判断标准**：如果删除两个选项只留一个，玩家体验完全不变，则这些选项是假选择。

### 原则三：制造困境而非测验

好选择是"两难抉择"，不是"找正确答案"。

| 测验型（避免） | 困境型（追求） |
|----------------|----------------|
| 有一个明显正确的选项 | 每个选项都有合理性与代价 |
| 选错=失败，选对=成功 | 没有对错，只有不同代价 |
| 考验玩家"知识" | 考验玩家"价值观" |
| "哪个是毒药？" | "救爱人还是救全城？" |

**困境设计公式**：`选项A 的价值 ≈ 选项B 的价值`，但 `选项A 的代价 ≠ 选项B 的代价`。

### 原则四：让失败有趣

不是每条路都通向好结局。最难忘的游戏体验往往来自失败和遗憾。

| 失败类型 | 处理方式 | 价值 |
|----------|----------|------|
| 战斗失败 | 被俘虏→开启越狱支线 | 失败开启新内容，而非游戏结束 |
| 谈判破裂 | NPC 愤怒离开→后续报复 | 失败有远期后果，增加张力 |
| 错误抉择 | 信任了不该信任的人→被背叛 | 失败让玩家学到世界规则 |
| 时机错过 | 关键 NPC 已死→只能找替代方案 | 失败改变可用资源，非死路 |

**原则**：失败不是"游戏结束"，而是"故事拐弯"。除非玩家 HP 归零（永久死亡模式），失败应开启新的叙事可能。

### 原则五：展示后果

玩家做了一个选择，后续必须看到影响——否则选择等于没做。

| 后果类型 | 展示时机 | 示例 |
|----------|----------|------|
| 即时后果 | 选择后立刻 | "你选择了贿赂。守卫掂了掂金币，侧身让开了路。" |
| 短期后果 | 1-3 个场景后 | "酒馆里有人在议论：听说有人用钱买通了守卫..." |
| 长期后果 | 数个场景甚至章节后 | "你曾放走的那个盗贼，如今成了公会首领，带着人来找你算账了。" |
| 隐藏后果 | 特定条件触发 | "因为你当初救了那个乞丐，他在关键时刻为你挡了一刀。" |

**后果追踪**：所有选择及后果记录在 `.game-state/choices-log.md`，格式为 `[选择] | [场景] | [即时后果] | [潜在远期影响]`。

## §3 "Yes, and..." 原则的实现

源自即兴戏剧的核心原则：接纳伙伴的提议，并在其基础上添加。在游戏中，这意味着玩家的合理行动应被接纳并产生世界反馈。

### 实现层次

| 层次 | 说明 | 示例 |
|------|------|------|
| 完全接纳 | 行动合理且在能力范围内，直接成功 | "我推开木门" → 门被推开，描述门后景象 |
| 接纳+代价 | 行动可行但有代价或风险 | "我想徒手攀爬悬崖" → 允许尝试，但体力消耗大，有坠落风险 |
| 接纳+失败 | 行动可尝试但当前条件下大概率失败 | "我试图说服国王退位" → 国王大笑拒绝，但不会说"你不能这样做" |
| 拒绝（仅 P0） | 违反安全红线 | 涉及未成年不当内容 → 以角色逻辑拒绝，不进入游戏 |

### 不说"你不能这样做"

玩家想做超出能力或不合理的事，AI 不直接拒绝，而是描述失败的尝试和后果：

| 玩家行动 | 错误回应（禁止） | 正确回应 |
|----------|------------------|----------|
| "我一拳打碎铁门" | "你不能这样做，门是铁的。" | "你的拳头砸在铁门上，震得虎口发麻——门纹丝不动。守卫听到了动静，朝这边走来。" |
| "我飞过城墙" | "你不会飞。" | "你助跑起跳，但重力无情地将你拽回地面。城墙太高了——也许该找个绳索？" |
| "我说服神明帮我" | "这不可能。" | "你向天空呼喊，但只有风声回应。神明似乎并不在意一个凡人的请求。" |

### 失败也有趣

即使是失败尝试，也应推进叙事或提供信息：
- 失败揭示世界规则（铁门太坚固→需要找钥匙或工具）。
- 失败引发新情况（守卫被惊动→新的应对挑战）。
- 失败展现角色特质（玩家的鲁莽/勇气被 NPC 注意）。

## §4 玩家自定义行动处理

混合/自由输入模式下，玩家会做出 AI 未预设的行动。处理流程如下：

```
玩家输入自定义行动
  ↓
1. 解析意图：玩家想做什么？（含糊则给出 2-3 种解读）
  ↓
2. 可行性判定：
   - 在世界规则内？ → 是 → 继续
   - 违反 P0 红线？ → 是 → 以角色逻辑拒绝
   - 超出角色能力？ → 标记为"低成功率尝试"
  ↓
3. 资源/条件检查：是否有所需物品/能力/状态？
  ↓
4. 结果判定：成功/部分成功/失败（含后果）
  ↓
5. 状态更新：更新相关状态文件
  ↓
6. 叙事响应：描述行动过程与结果（遵循 Yes-and 原则）
```

### 处理意外行动的原则

- **永远先接纳**：默认玩家行动合理，除非有明确证据不合理。
- **世界反馈而非系统拒绝**：用世界逻辑回应，不用系统逻辑拒绝。
- **意料之外情理之中**：玩家做了没想到的事，AI 在世界规则内给出最合理的响应。
- **记录创意行动**：玩家的创意解法记录在事件日志，后续可成为伏笔或后果。

## §5 选择后果追踪机制

### 追踪文件结构

```json
{
  "choices_log": [
    {
      "round": 7,
      "choice": "选择贿赂守卫而非战斗",
      "scene": "王港城北门",
      "immediate_consequence": "花费 20 金币，顺利进城",
      "immediate_state_change": { "gold": -20, "entered_city": true },
      "potential_future": [
        { "type": "rumor", "desc": "守卫可能向同僚提起此事", "trigger_condition": "time+2days" },
        { "type": "reputation", "desc": "被认为是'有钱好办事'的人", "trigger_condition": "npc_interaction" }
      ],
      "realized": []
    }
  ]
}
```

### 后果激活机制

- `potential_future` 中的远期后果在满足 `trigger_condition` 时被激活，写入 `realized`。
- 激活后需在叙事中体现（NPC 引用、世界状态变化、新事件触发）。
- 未激活的潜在后果持续保留，确保"埋下的种子总会发芽"。

### 后果层级与玩家感知

| 层级 | 玩家是否应感知 | 设计目的 |
|------|----------------|----------|
| 即时后果 | 明确感知 | 让玩家理解选择的直接结果 |
| 短期后果 | 隐约感知（暗示） | 让玩家感到世界在反应 |
| 长期后果 | 惊讶后恍然 | 让玩家意识到"原来当初那个选择导致了这个" |
| 隐藏后果 | 不感知，直到触发 | 奖励细心玩家，增加重玩价值 |

> 玩家自主权的本质不是"让玩家想做什么就做什么"，而是"让玩家做的每一件事都被世界认真对待"。
