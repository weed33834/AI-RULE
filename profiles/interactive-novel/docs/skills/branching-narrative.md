# 分支叙事技能 (Branching Narrative)

> 互动小说的核心是有意义的选择路径：不是无限分支，而是让每个选择都改变玩家经历的旅程与抵达的终点。
> 玩家输入模式与选择设计详见 `player-agency.md`；叙事一致性检查详见 `narrative-coherence.md`；游戏主持人分支管理详见 `docs/prompts/game-master.md`。

## §1 四种分支模式

不同的剧情节点需要不同的分支结构。选择正确的模式，是叙事节奏与开发可控性的关键。

### 1.1 漏斗式（Funnel）

多条选择收束到同一个剧情点。选择影响"过程"但不改变"大方向"。

```
        ┌─ 选择A ─┐
起点 ───┼─ 选择B ─┼──→ 收敛点（共同剧情）
        └─ 选择C ─┘
```

| 维度 | 说明 |
|------|------|
| 结构特征 | 多入一出，路径不同终点相同 |
| 适用场景 | 序章收束、必经主线节点、教程段 |
| 价值 | 让玩家感觉有选择，同时控制叙事成本 |
| 风险 | 若收敛后完全无差异，会产生"假选择"感 |

**设计要点**：虽然终点相同，但不同路径应带来不同的信息量、资源或 NPC 印象。例如三条路抵达同一个村庄：A 路获得旅人的情报，B 路捡到一把旧剑，C 路被劫匪打伤（HP 损耗）。终点相同，但状态不同。

### 1.2 分支合并（Branch & Merge）

不同路径平行发展一段时间，在关键节点合流。这是大多数主线剧情的首选模式。

```
        ┌─ 选择A → 路径A1 → 路径A2 ─┐
起点 ───┤                              ├──→ 合流点
        └─ 选择B → 路径B1 → 路径B2 ─┘
```

| 维度 | 说明 |
|------|------|
| 结构特征 | 多入多出再合一，平行段有实质差异 |
| 适用场景 | 主线中段、可选解决途径、阵营选择 |
| 价值 | 提供真实的选择差异，同时防止分支爆炸 |
| 合流条件 | 需要一个逻辑自洽的"为什么两条路最终相遇" |

**合流设计示例**：玩家选择潜入或强攻盗贼据点。潜入路线遭遇巡逻队潜行通过，强攻路线正面战斗。两条路最终都到达据点核心——但潜入者保留了突袭优势，强攻者已惊动敌人，核心房间的敌人配置因此不同。

### 1.3 真分支（True Branch）

选择永久改变故事走向，路径不再合流，通向不同结局。

```
        ┌─ 选择A → ... → 结局A
起点 ───┼─ 选择B → ... → 结局B
        └─ 选择C → ... → 结局C
```

| 维度 | 说明 |
|------|------|
| 结构特征 | 一入多出，永不交汇 |
| 适用场景 | 关键道德抉择、结局分歧点、阵营永久站队 |
| 价值 | 给予选择最大权重，提升重玩价值 |
| 风险 | 开发成本高，必须为每条线准备完整内容 |
| 使用频率 | 全游戏仅 1-3 处，过多则无法管理 |

**使用原则**：真分支应出现在故事中段偏后（约 60%-70% 进度），此时玩家已建立情感投入，分歧带来的冲击最大。过早的真分支会让玩家觉得"还没玩够就要选边"。

### 1.4 枢纽式（Hub）

以一个中心地点为基地，向外探索多个分支，分支完成后返回枢纽。

```
              ┌─→ 支线A ─┐
枢纽（基地） ──┼─→ 支线B ─┼──→ 枢纽（状态已变）
              └─→ 支线C ─┘
```

| 维度 | 说明 |
|------|------|
| 结构特征 | 中心辐射，支线可按任意顺序完成 |
| 适用场景 | 探索类、调查类、开放区域游戏 |
| 价值 | 给予玩家探索自由度，支线互相影响 |
| 枢纽演化 | 每完成一条支线，枢纽状态应发生变化 |

**枢纽演化示例**：玩家以酒馆为枢纽调查连环命案。完成支线 A（找到第一具尸体线索）后，酒馆老板态度变化；完成支线 B（救下目击者）后，酒馆出现新的 NPC。枢纽不是静态的，而是随探索进度演变的。

## §1.5 行业标准分支结构分类（Sam Kabo Ashwell 分类法）

> 以下 8 种结构是互动叙事领域的经典分类，被 Choice of Games、Inkle、Fallen London 等业界标杆广泛采用。

| # | 结构 | 机制 | 代表作 | 写作成本 | 合流频率 |
|---|------|------|--------|----------|----------|
| 1 | **Time Cave（时间洞穴）** | 每个选择都分叉，几乎不合流，结局极多 | 早期 CYOA | 极高 | 几乎不 |
| 2 | **Gauntlet（关卡式）** | 一条主线，分支多为死路/失败/快速回流 | Zork | 中 | 高 |
| 3 | **Branch & Bottleneck（分支与瓶颈）** | 分叉后在关键场景合流，重度状态追踪 | Choice of Games、80 Days | 低-中 | 中高 |
| 4 | **Quest（任务式）** | 模块化分支簇，多为地理驱动，合流频繁 | Fighting Fantasy、Lone Wolf | 中 | 高 |
| 5 | **Open Map（开放地图）** | 节点间可逆移动，静态地理，强状态追踪 | Fallen London | 极高 | 无 |
| 6 | **Sorting Hat（分院帽）** | 前期重分叉重合流决定主线，后半线性 | Katawa Shoujo | 中 | 前高后无 |
| 7 | **Floating Modules（浮动模块）** | 无主干，模块化遭遇按状态/随机触发 | StoryNexus | 高 | 无 |
| 8 | **Loop and Grow（循环生长）** | 中心线程循环，状态解锁新选项 | Bee、Trapped in Time | 中 | 循环式 |

### 选型决策树

```
你的游戏需要多少结局？
├─ 1-3 个 → 后半线性结构（Sorting Hat / Gauntlet）
├─ 4-7 个 → Branch & Bottleneck（推荐默认）
├─ 8+ 个 → Quest / Floating Modules
└─ "无限" → Open Map / Loop and Grow

你的写作预算如何？
├─ 低 → Branch & Bottleneck（性价比最高）
├─ 中 → Quest / Gauntlet
└─ 高 → Time Cave / Open Map

玩家自由度需求？
├─ 线性体验+选择感 → Sorting Hat
├─ 探索自由 → Open Map / Hub
├─ 重玩价值最大化 → Branch & Bottleneck + 真分支
└─ 涌现式叙事 → Floating Modules / Loop and Grow
```

**行业共识**：**Branch & Bottleneck 是默认推荐结构**——前期选择累积状态，到关键节点才产生分化，既给玩家选择感又控制写作量。Choice of Games 的 Dan Fabulich 称之为"延迟分支"。

## §2 收敛点设计方法

为防止分支无限膨胀（叙事成本爆炸），必须设计收敛点。

### 收敛原则

| 原则 | 说明 |
|------|------|
| 间隔控制 | 每隔 3-5 个选择点设计一个收敛点 |
| 差异保留 | 收敛的是"剧情节点"，不是"状态差异"——路径带来的资源/关系/信息差异应保留 |
| 逻辑自洽 | 收敛需要一个合理的叙事理由，不能生硬地"突然到了同一个地方" |
| 后果延迟 | 收敛点的选择后果可在数个场景后才爆发 |

### 深分支预算机制

> 防止指数爆炸：3选项×4层深 = 81 结局，没人写得完。

| 分支类型 | 定义 | 每章上限 | 每局上限 |
|----------|------|----------|----------|
| 浅分支（风味分支） | 只改变对白/短期张力，2拍内合流 | 3-5 个 | 不限 |
| 中分支（状态分支） | 改变资源/关系/信息，3-5拍后合流 | 1-2 个 | 8-12 个 |
| 深分支（真分支） | 永久改变故事走向，不合流 | 0-1 个 | 1-3 个 |

**执行规则**：
- 每章设深分支预算上限，评审中强制执行
- 浅分支快合流：只改风味/短期张力，两拍内合流
- 深分支留稀缺：影响未来状态（阵营/任务可用性）的选择才持久分叉
- 假选择检测：两个选项通向同一场景 → 标记为"假选择"，必须修改

### 收敛的叙事理由库

- **外部事件强制**：一场暴风雪/敌军围城/地震，迫使所有路径的角色聚集到同一避难所。
- **目标驱动**：无论走哪条路，最终目的地都是同一个（如夺回圣物），只是抵达方式不同。
- **NPC 引导**：一个关键 NPC 在各条路径上都以不同方式将玩家引向同一地点。
- **时间窗口**：某个事件有固定发生时间（如月蚀仪式），所有路径必须在此时限前抵达。

## §3 动态分支生成策略

AI 游戏引擎不依赖预设脚本树，而是根据当前状态动态生成分支选项。

### 生成依据

```
当前游戏状态
├─ 玩家状态：能力/物品/关系 → 决定"能做什么"
├─ 世界状态：区域/时间/派系 → 决定"环境中有什么可互动"
├─ NPC 状态：在场/情绪/知识 → 决定"能和谁互动、对方如何反应"
└─ 叙事上下文：当前场景张力/未解线索 → 决定"哪些选择有戏剧价值"
```

### 选项生成规则

| 规则 | 说明 |
|------|------|
| 至少一个激进选项 | 推动剧情前进的行动，不是纯观望 |
| 至少一个保守选项 | 允许玩家谨慎行事，收集更多信息 |
| 至少一个创意选项 | 基于玩家特定能力/物品的非显然选项 |
| 选项间信息差 | 每个选项基于不同信息，玩家需判断哪个更优 |
| 允许自定义 | 始终保留"自定义行动"入口 |

### 动态生成的禁忌

- 不可生成"明显错误"的选项来衬托"正确"选项——每个选项都应有合理性。
- 不可生成依赖玩家不具备的能力的选项（无钥匙却给"开锁"选项）。
- 不可生成与当前场景情绪完全脱节的选项（战斗中给"坐下喝茶"）。

## §4 结局系统设计

### 结局数量与类型

每局游戏设计 3-7 个结局，分布如下：

| 结局类型 | 数量 | 特征 |
|----------|------|------|
| 好结局 | 1-2 | 玩家达成核心目标，付出可接受代价 |
| 坏结局 | 1-2 | 目标失败或付出惨痛代价，但有叙事价值 |
| 中立结局 | 1 | 妥协方案，部分达成、部分牺牲 |
| 隐藏结局 | 1-2 | 需满足特定条件组合，奖励深度探索 |

### 累积选择决定机制

结局不是由最后一个选择决定，而是由全程累积的选择倾向计算。

```json
{
  "ending_tracker": {
    "morality_score": -15,
    "morality_history": [
      { "choice": "救下落水儿童", "delta": +20, "round": 3 },
      { "choice": "为情报拷问俘虏", "delta": -25, "round": 12 },
      { "choice": "拒绝参与屠杀平民", "delta": +15, "round": 20 }
    ],
    "loyalty": { "royal_guard": 30, "thieves_guild": -10 },
    "key_flags": ["spared_bandit_leader", "revealed_conspiracy"]
  }
}
```

- `morality_score` 持续累积，最终区间映射到结局倾向。
- `loyalty` 决定阵营线结局归属。
- `key_flags` 触发隐藏结局的特殊条件。

### 隐藏结局条件设计

隐藏结局需要"非显然"的条件组合，奖励深度探索与连贯角色扮演：

| 条件类型 | 示例 |
|----------|------|
| 行为一致性 | "全程不杀任何人类" + "帮助所有求助者" → 和平使者结局 |
| 信息收集 | "找到全部 7 块石碑碎片" + "在月蚀夜合成" → 真相结局 |
| 关系极限 | "与隐藏 NPC 好感度达 100" + "拒绝主线最终任务" → 隐居结局 |
| 反直觉 | "在第一关就丢掉圣剑" + "全程用拳头战斗" → 武僧结局 |

隐藏结局不可在选项中暗示，只能通过世界线索（NPC 传闻、古籍记载）隐晦提示。

## §5 伏笔与回收机制

### 伏笔类型

| 类型 | 说明 | 回收时机 |
|------|------|----------|
| 物品伏笔 | 早期获得的看似无用之物 | 后期关键场景发挥作用 |
| 对话伏笔 | NPC 一句不经意的话 | 数个场景后真相揭晓 |
| 环境伏笔 | 场景中的异常细节 | 重返该场景或相关场景时呼应 |
| 角色伏笔 | NPC 的可疑行为 | 该 NPC 真实身份/动机揭露时 |

### 回收原则

- **必埋必收**：埋下的伏笔必须在后续得到呼应——无论玩家走哪条分支。若某分支无法回收，则不应埋下该伏笔。
- **延迟满足**：伏笔与回收之间至少间隔 3-5 个场景，让玩家"忘记"后再被唤起，冲击更强。
- **多义性**：好的伏笔在回收前可以有多种解读，回收时才揭示真意。
- **记录追踪**：所有伏笔记录在 `.game-state/foreshadowing.json`，标记已埋/已收状态。

```json
{
  "foreshadowing": [
    {
      "id": "fs_rusty_key",
      "planted_round": 5,
      "planted_context": "地牢守卫身上搜到生锈钥匙",
      "intended_payoff": "开启终章密室",
      "status": "planted",
      "payoff_round": null
    }
  ]
}
```

## §6 分支管理工具

| 工具 | 用途 |
|------|------|
| `choices-log.md` | 记录每个重要选择及其即时/远期后果 |
| `foreshadowing.json` | 追踪伏笔埋设与回收状态 |
| `ending-tracker.json` | 累积计算结局倾向分数 |
| `branch-state.json` | 记录当前所处分支路径及可达结局集合 |

> 分支叙事的艺术，不在于让玩家"走到哪里"，而在于让他们觉得"每一步都算数"。
