# 会话管理技能 (Session Management)

> 游戏可能持续数小时甚至跨多次会话：存档、读档、前情提要是必需功能，确保玩家离开后回来能无缝续接。

## §1 存档系统

### 存档指令

| 指令 | 功能 | 说明 |
|------|------|------|
| `/save [名称]` | 保存当前游戏状态到 `.game-saves/{名称}.json` | 手动命名存档 |
| `/load [名称]` | 从存档恢复游戏状态 | 读取指定存档 |
| `/autosave` | 手动触发自动存档 | 系统每 10 轮也会自动存档 |
| `/saves` | 列出所有存档 | 显示存档名称、时间、轮次 |

### 自动存档机制

- **触发频率**：每 10 轮玩家行动后自动存档一次。
- **存档命名**：`auto_{day}_{hour}_{round}`，如 `auto_day14_21_round30`。
- **保留数量**：保留最近 5 个自动存档，更早的自动清理。
- **关键节点强制存档**：重大事件发生时（角色死亡、结局分歧、章节切换）立即自动存档。

### 存档内容（JSON 格式）

```json
{
  "save_metadata": {
    "save_name": "进入密室前",
    "created_at": "2026-07-13T21:30:00",
    "game_seed": { "genre": "奇幻", "difficulty": "普通", "rating": "PG-13" },
    "round": 28,
    "playtime_hours": 4.5
  },
  "player_state": {
    "file": ".game-state/player.json",
    "snapshot": "（完整玩家状态 JSON 内联）"
  },
  "world_state": {
    "files": [".game-state/world-map.json", ".game-state/timeline.json", ".game-state/factions.json"],
    "snapshot": "（完整世界状态 JSON 内联）"
  },
  "npc_states": {
    "files": [".game-state/npcs/*.json"],
    "snapshot": "（所有 NPC 状态 JSON 内联）"
  },
  "story_summary": "（当前 story-summary.md 内容内联）",
  "current_scene": {
    "description": "你站在密室门前，手中握着那把生锈的钥匙。门上的符文微微发光...",
    "available_choices": ["插入钥匙", "仔细检查符文", "退后观察", "自定义行动"],
    "pending_events": []
  },
  "choices_log": "（choices-log.md 内容内联）",
  "foreshadowing": "（foreshadowing.json 内容内联）"
}
```

### 存档完整性

- 存档必须包含完整可恢复的全部状态，不可依赖外部文件（防止文件被修改后存档失效）。
- 存档后立即校验：能否从该存档完整恢复游戏（读取所有字段、验证一致性）。
- 存档失败时通知玩家并保留上一个有效存档。

## §2 读档恢复流程

```
玩家输入 /load [名称]
  ↓
1. 读取存档文件 → 校验 JSON 完整性
  ↓
2. 校验存档版本与当前游戏种子是否兼容
   - 兼容 → 继续
   - 不兼容 → 警告玩家并询问是否强制加载
  ↓
3. 恢复状态文件
   - 覆写 player.json / world-map.json / npcs/*.json 等
   - 恢复 choices-log.md / foreshadowing.json
  ↓
4. 重建上下文
   - 读取 story-summary.md（存档内快照）
   - 读取 current_scene.description
   - 注入最近 5 轮对话原文（若存档包含）
  ↓
5. 生成"前情提要"并展示给玩家（见 §3）
  ↓
6. 从 current_scene 恢复叙事，给出可用选择
```

### 读档校验项

| 校验项 | 说明 | 失败处理 |
|--------|------|----------|
| JSON 格式 | 存档文件是否为合法 JSON | 报错，存档损坏 |
| 字段完整 | 必填字段是否齐全 | 报错，列出缺失字段 |
| 数值合法 | HP/好感度等是否在合法区间 | 尝试修正并警告 |
| 引用一致 | inventory_ref 等引用路径是否存在 | 重建引用或警告 |
| 种子兼容 | 游戏类型/难度是否与当前一致 | 警告并询问是否继续 |

## §3 前情提要生成

### 提要结构（2-3 段）

每次新会话开始或玩家使用 `/recap` 时，生成结构化前情提要：

```markdown
## 前情提要

### 当前目标
你正在寻找治愈王港城疫病的方法。治愈师 Elena 告诉你，解药的
关键成分"月光草"只在古森林的满月之夜生长。距离下一个满月
还有三天。

### 重要事件
- 第 3 天：你抵达王港城，发现城市已被疫病封锁。
- 第 8 天：你帮助治愈师 Elena 治疗患者，获得了她的信任。
- 第 12 天：你在酒馆打听到盗贼公会趁乱囤积药材，企图哄抬药价。
- 第 14 天：你潜入盗贼据点，夺回了一批被囤积的药材，但暴露了身份。

### 未解线索
- 古森林中传闻有"守护者"出没，可能与月光草有关。
- 盗贼公会头目似乎与城内某位贵族有秘密往来。
- 你在据点发现一封密信，提及"第七日仪式"，含义不明。
```

### 提要生成规则

| 段落 | 内容来源 | 生成规则 |
|------|----------|----------|
| 当前目标 | quests.json 的活跃主线 | 取最高优先级任务的当前阶段 |
| 重要事件 | event-log.md | 选取最近 5-8 个标记为"重要"的事件 |
| 未解线索 | foreshadowing.json 中 status=planted 的条目 | 选取 2-3 个与当前目标相关的未回收伏笔 |

### 提要更新机制

- `story-summary.md` 每 10 轮自动更新一次（与自动存档同步）。
- 重大事件发生后立即更新（不等 10 轮周期）。
- 摘要中保留：关键决策、玩家选择、NPC 态度变化、获得/失去的物品。
- 摘要中丢弃：普通探索描写、重复的战斗过程、已解决的次要对话。

## §4 会话节奏控制

### 开场回顾

每次新会话以简短前情提要开始：

```
（前情提要，2-3 段）
---
你站在古森林的边缘。月光透过稀疏的树冠洒下斑驳的光影。
远处隐约传来流水声。Elena 给你的地图上标记着月光草的
大致位置——在森林深处的一处溪流旁。

你身边带着：铁制长剑、3 瓶治疗药水、Elena 画的地图。
你的状态：HP 85/120，精神尚佳，但月光草只在今夜可采。

你要怎么做？
```

- 回顾后立即接入当前场景，不停留在纯信息罗列。
- 回顾中顺带提醒玩家当前状态（HP/关键物品），但以叙事方式融入，不列数据表。

### 场景边界

| 边界类型 | 说明 | 处理方式 |
|----------|------|----------|
| 场景开始 | 玩家进入新场景 | 明确的场景描写（地点、氛围、在场角色） |
| 场景结束 | 当前场景的目标达成/失败/中断 | 简短收束，过渡到下一场景或给予选择 |
| 场景中断 | 会话结束于场景中间 | 标记中断点，下次从此处精确恢复 |

- 不在场景中间突然停止叙事（除非玩家主动结束会话）。
- 每个场景有明确的"入场"和"出场"，避免模糊过渡。

### 悬念收尾

会话结束时留在悬念点，激发玩家下次回来的欲望：

| 悬念类型 | 示例 |
|----------|------|
| 未知揭晓前夕 | "你转动钥匙，锁芯咔哒一声弹开。门后是——"（停止） |
| 危险逼近 | "远处传来马蹄声，越来越近。你隐约看到旗帜上的纹章——那是敌对家族的标志。" |
| 真相碎片 | "密信的最后一行写着：'第七日，月落之时，一切将终结。'你抬头看了一眼月亮..." |
| 选择悬停 | "三个选项摆在你面前，每一个都可能改变一切。你伸出手——"（停止） |

### 会话长度建议

| 篇幅类型 | 单局时长 | 轮次 | 适用场景 |
|----------|----------|------|----------|
| 短篇 | 30-45 分钟 | 10-20 轮 | 快速体验、碎片时间 |
| 中篇 | 1-2 小时 | 20-50 轮 | 标准游戏会话 |
| 长篇连载 | 2 小时+ | 50+ 轮 | 深度沉浸、周末长玩 |

- 建议 30-60 分钟一轮会话，在自然停止点收尾。
- 玩家明确表示要结束时，立即触发存档 + 悬念收尾。
- 不要为了"凑时长"而拖沓——宁可短而紧凑，不要长而注水。

## §5 跨会话上下文重建

长篇游戏跨多次会话时，需高效重建上下文，避免"失忆"。

### 重建流程

```
新会话开始
  ↓
1. 读取最新自动存档 → 恢复全部状态文件
  ↓
2. 读取 story-summary.md → 获取压缩的剧情进度
  ↓
3. 读取最近 5 轮对话原文 → 获取即时上下文（语气、未完成对话）
  ↓
4. 读取当前场景描述 → 确定叙事起点
  ↓
5. 重注入游戏核心设定 → 类型/难度/角色/当前目标（防止长篇漂移）
  ↓
6. 生成前情提要 → 展示给玩家
  ↓
7. 从中断点恢复叙事
```

### 上下文分层策略

| 层级 | 内容 | 刷新频率 | 作用 |
|------|------|----------|------|
| 核心层 | 游戏种子、角色身份、当前主线目标 | 每 5 轮重注入 | 防止长篇漂移，保持方向感 |
| 摘要层 | story-summary.md（压缩剧情） | 每 10 轮更新 | 提供中期记忆，知道"走到哪了" |
| 近期层 | 最近 5-10 轮对话原文 | 持续滚动 | 提供即时上下文，保持语气连贯 |
| 即时层 | 当前场景描述、可用选择 | 每轮更新 | 确定当前行动空间 |

### 大型游戏处理（50 轮+）

- 超过 50 轮的游戏：先用 story-summary.md 定位当前阶段，再精读最近 10 轮。
- 超过 100 轮的游戏：story-summary.md 分章节，只加载当前章节摘要 + 全局关键事件。
- NPC 交互前：先读取该 NPC 档案和与玩家的交互历史，确保反应一致。

### 防止长篇漂移

长篇游戏中，AI 容易逐渐偏离初始设定。以下机制防止漂移：

| 漂移类型 | 表现 | 防止机制 |
|----------|------|----------|
| 角色漂移 | 玩家性格被 AI 慢慢改写 | 每 5 轮重注入角色身份和背景 |
| 难度漂移 | 困难逐渐变简单（AI 下意识放水） | 每 5 轮检查难度设定，对照困难标准 |
| 语气漂移 | 黑暗基调逐渐变轻松 | 每 5 轮重注入游戏基调和氛围要求 |
| 设定漂移 | 魔法规则/世界设定被违反 | 每 5 轮重注入核心世界规则 |
| 关系漂移 | NPC 态度无理由软化/硬化 | 读取 NPC 档案的情感数值，对照叙事 |

> 会话管理的本质是"记忆"——让游戏记住玩家做过的每一件事，无论中间隔了多少天、多少次离开与回来。
