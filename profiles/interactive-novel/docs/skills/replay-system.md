# 重玩系统 (Replay System)

> 本文档定义互动小说游戏的重玩机制：周目继承、路径发现、元叙事设计、重玩激励。
> 好的互动小说不是"玩完就结束"，而是"每次玩都不一样"。
> 与 `ending-system.md` 互补——结局系统定义"怎么收束"，重玩系统定义"怎么重新开始"。
> 与 `branching-narrative.md` 互补——分支定义"一次游玩的路径"，重玩定义"多次游玩的探索"。

## §1 设计原则

- **每次都不一样**：重玩不是为了重复，而是为了发现新的可能性。
- **尊重时间**：不要让玩家重新经历完全相同的内容——跳过已体验的部分或提供变体。
- **累积有意义**：前几次游玩的成果在新周目中有体现，但不是纯粹数值碾压。
- **惊喜优先**：重玩的最大动力是"上次没见过的东西"。
- **自然引导**：不强制重玩，但让玩家"想知道如果选另一条路会怎样"。

## §2 重玩激励设计

### 2.1 重玩动机矩阵

| 动机类型 | 玩家心理 | 设计策略 | 示例 |
|----------|----------|----------|------|
| 路径好奇 | "另一条路会怎样？" | 结局后提示其他路径存在 | "如果你当时选择了信任..." |
| 隐藏内容 | "我错过了什么？" | 结局统计显示未发现的内容 | "你发现了 7/15 个隐藏场景" |
| 角色探索 | "用不同的人设玩会怎样？" | 允许不同的角色创建 | 同一故事但扮演不同性格的主角 |
| 完美主义 | "能达成更好的结局吗？" | 设计需要多次尝试才能达成的完美结局 | 需要特定选择组合的隐藏结局 |
| 元叙事 | "这个游戏还有更深的故事？" | 跨周目的元叙事线索 | NPC 似乎"记得"上周目的事 |

### 2.2 结局后激励

结局展示后，提供重玩激励（可选，不强制）：

```
═════════════════════════════════
  结局: 「灰烬中的黎明」
  游玩轮次: 47
  关键选择: 12
  发现场景: 23/35
  NPC 互动: 8/12
  隐藏线索: 3/8
═════════════════════════════════

这个世界上还有其他故事在发生。
Marcus 的过去、Elara 的秘密、北境的战争...
有些真相，需要不同的选择才能触及。

输入 /restart 开始新的旅程
输入 /epilogue 查看尾声
输入 /achievements 查看成就
```

## §3 周目继承机制

### 3.1 继承层次

```
第一周目 (完全新鲜)
  ↓ 继承
第二周目 (轻度继承: 记忆碎片、结局加成)
  ↓ 继承
第三周目+ (深度继承: 隐藏路线、元叙事、NPC 变化)
```

### 3.2 继承类型详表

| 继承类型 | 说明 | 实现方式 | 玩家感知 |
|----------|------|----------|----------|
| 记忆碎片 | 上周目的细节以模糊方式出现 | NPC 说出上周目的台词但表示"不知为何想起" | "这个游戏记得我" |
| 物品继承 | 上周目获得的某件物品出现在新周目 | 特定物品在场景中出现，附带模糊提示 | "这不是上次那个..." |
| 信息优势 | 玩家知道的信息可以角色不知道的方式利用 | 出现额外对话选项"我好像知道..." | "我的知识有用武之地" |
| 结局加成 | 好结局给予新周目小优势 | 起始状态微调（额外物品/起始好感度） | "上次没白玩" |
| NPC 变化 | NPC 的反应因周目不同而变化 | 某些 NPC "感觉到"玩家似曾相识 | "它知道我回来过" |
| 隐藏选项 | 多周目后解锁新的选择 | 第三周目出现的全新对话/行动选项 | "原来还可以这样" |

### 3.3 继承数据管理

```json
{

    "total_playthroughs": 2,
```

## §4 路径发现系统

### 4.1 路径地图

帮助玩家理解"还有多少没探索的"：

```
故事路径地图 (已探索 2/7 条主要路径):

主线分叉点:
  第一章选择:
    ├── 信任 Marcus ✓ (周目1 → 结局A)
    ├── 怀疑 Marcus ✓ (周目2 → 结局D)
    ├── 无视 Marcus ✗ (未探索)
    └── [隐藏路径] ✗ (条件未满足)

  第三章选择:
    ├── 进入森林 ✓
    ├── 绕过森林 ✗
    └── [隐藏路径] ✗

结局: 2/7 已达成
  A ✓ | B ✗ | C ✗ | D ✓ | E ✗ | F ✗ | [隐藏] ✗
```

### 4.2 差异化体验

重玩时，即使走相同路径，也应有差异化体验：

| 差异化策略 | 说明 | 示例 |
|------------|------|------|
| NPC 反应变化 | NPC 对"回来的人"有微妙不同 | "你...我们是不是见过？" |
| 环境变化 | 同一场景有细微不同 | 上次锁着的门这次虚掩着 |
| 新信息层 | 已知信息可以更深入探索 | 上次只能问名字，这次可以问过去 |
| 伏笔前置 | 上周目后期的伏笔在新周目更早出现 | 上周目第三章才发现的线索，这次第一章就出现暗示 |
| 平行视角 | 同一事件从不同角度展示 | 上次在城内经历围城，这次在城外 |

## §5 元叙事设计

### 5.1 什么是元叙事

元叙事是"关于叙事的叙事"——游戏知道自己在被游玩，玩家知道自己在重玩，这种共识成为故事的一部分。

### 5.2 元叙事层级

| 层级 | 深度 | 示例 | 适用 |
|------|------|------|------|
| L0 无元叙事 | 完全沉浸，不暗示重玩 | 传统单结局故事 | 纯叙事向 |
| L1 轻度暗示 | 结局后提示其他可能 | "这个故事还有其他讲法..." | 大多数游戏 |
| L2 NPC 感知 | NPC 似乎感觉到"循环" | "你又来了...不，我什么都没说" | 悬疑/恐怖 |
| L3 世界感知 | 世界本身在"循环" | 同一天的报纸，不同的标题 | 时间循环类 |
| L4 玩家对话 | 游戏直接与"玩家"对话 | "你为什么要回来？你想要什么？" | 哲学向 |

### 5.3 元叙事使用原则

- **不要过度**：元叙事是调味料，不是主菜。过度使用会破坏沉浸感。
- **服务于主题**：元叙事应与故事主题呼应，不是为元而元。
- **给玩家选择**：允许玩家"忽略"元叙事，按正常方式游玩。
- **有 payoff**：元叙事的铺设必须在最终有回报，不能只是噱头。

## §6 跳过与加速机制

尊重玩家时间，重玩时不强制重复已体验的内容：

| 机制 | 说明 | 触发条件 |
|------|------|----------|
| 摘要跳过 | 已经历的场景用摘要带过 | 玩家选择"跳过已体验内容" |
| 快速对话 | NPC 对话加速，关键信息高亮 | 玩家已与该 NPC 交互过 |
| 路径标记 | 在分叉点标注"上次走了A" | 第二周目起 |
| 选择预览 | 允许玩家查看"如果选另一个会怎样"的简短预览 | 第三周目起（可选） |
| 起点选择 | 允许从特定章节开始新周目 | 完成至少一次全流程后 |

## §7 重玩质量评估

| 指标 | 测量方式 | 目标值 |
|------|----------|--------|
| 重玩率 | 完成一次后开始第二次的玩家比例 | > 40% |
| 路径探索率 | 平均每个玩家探索的路径比例 | > 30% |
| 结局发现率 | 平均每个玩家达成的结局数 | > 2 |
| 内容新鲜度 | 重玩时遇到新内容的比例 | > 40% |
| 重玩满意度 | 重玩后的满意度评分 | > 4/5 |

## §8 与其他文档的关系

- **`ending-system.md`**: 结局系统定义"怎么收束"，重玩系统定义"怎么重新开始"。结局后的激励是重玩的入口。
- **`branching-narrative.md`**: 分支定义单次路径，重玩定义多次探索。分支数量决定重玩价值。
- **`game-state-machine.md`**: 周目继承需要跨游戏状态管理。
- **`session-management.md`**: 多周目的存档管理——每周目独立存档 + 继承数据。
- **`memory-system.md`**: 跨周目的"记忆"需要特殊的记忆管理。
- **`narrative-coherence.md`**: 元叙事不能破坏单周目内的一致性。
- **`player-agency.md`**: 重玩是玩家自主权的延伸——"我想用不同方式体验这个故事"。
