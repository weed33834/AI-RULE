# 新手引导系统 (Onboarding System)

> 本文档定义互动小说游戏引擎的新手引导体系：首次体验设计、渐进式教学、引导式第一幕。
> 玩家的前 10 分钟决定是否继续玩——好的引导让玩家"学会玩"，而不是"看说明书"。
> 与 `player-agency.md` 互补——自主权定义"玩家能做什么"，引导定义"玩家怎么学会做"。
> 与 `genre-system.md` 互补——类型系统定义"玩什么故事"，引导定义"怎么开始玩"。

## §1 设计原则

- **做中学**：不解释规则，让玩家在行动中自然学会。
- **渐进披露**：信息按需给出，不一次性堆砌教程。
- **尊重玩家**：经验丰富的玩家应能跳过引导，直接开始。
- **失败安全**：引导阶段的错误不产生严重后果，让玩家放心尝试。
- **沉浸优先**：引导融入叙事，不打破第四面墙。

## §2 首次体验流程

```
┌──────────────────────────────┐
│ 1. 游戏种子收集 (9 维度)        │  收集玩家偏好，定制体验
└──────────┬───────────────────┘
           ▼
┌──────────────────────────────┐
│ 2. 引导式开场 (第一幕)          │  在叙事中教会基础操作
└──────────┬───────────────────┘
           ▼
┌──────────────────────────────┐
│ 3. 自由探索 (第二幕)            │  移除引导，让玩家自主行动
└──────────┬───────────────────┘
           ▼
┌──────────────────────────────┐
│ 4. 首次挑战 (第一个真正挑战)    │  验证玩家是否学会了核心机制
└──────────────────────────────┘
```

### 2.1 游戏种子收集

在游戏开始前收集 9 维度种子（详见 AGENTS.md §6），其中以下维度直接影响引导设计：

| 种子维度 | 对引导的影响 |
|----------|------------|
| 互动模式 | 自由输入 → 需要教玩家"你可以输入任何行动"；选择式 → 需要教玩家"选择代表你的行动" |
| 难度 | 新手 → 详细引导；老手 → 极简引导 |
| 类型 | 奇幻 → 引导中包含魔法体系基础；科幻 → 引导中包含科技设定 |
| 内容分级 | 影响开场场景的基调选择 |

### 2.2 引导式开场设计

开场不是"教程"，而是一个精心设计的第一幕，让玩家在体验故事的同时学会操作。

**开场三要素**：

| 要素 | 作用 | 设计要求 |
|------|------|----------|
| 即时投入 | 第一句话就让玩家做出选择 | 不超过 3 句话就出现第一个交互点 |
| 语境教学 | 在故事情境中教操作 | 不出现"按X键"式的元教学 |
| 成就反馈 | 完成第一个行动后给予正反馈 | 让玩家感到"我的选择有效果" |

**开场模板**：

```
[场景设定 — 1-2句]
你站在[地点]，[当前状况]。

[即时选择 — 不解释规则，直接给选项]
你可以：
A) [行动选项1]
B) [行动选项2]
C) [行动选项3]
（或直接输入你想做的事）

→ 玩家选择后，给出明确反馈，让玩家感到"我的行动被接受了"
→ 如果是自由输入模式，首次输入时暗示"你可以输入任何行动"
```

### 2.3 渐进式教学清单

以下机制需要在第一幕中自然教会玩家，但不使用教程语言：

| # | 机制 | 教学方式 | 出现时机 |
|---|------|----------|----------|
| 1 | 基本交互 | 第一次选择/输入 | 开场第一句话后 |
| 2 | 后果感知 | 第一个选择产生可见后果 | 第一个选择后 |
| 3 | NPC 对话 | 遇到第一个 NPC | 开场 3-5 轮内 |
| 4 | 物品互动 | 可以拿取/使用物品 | 开场 5-8 轮内 |
| 5 | 状态查看 | `/status` 或 `/inventory` 命令 | 第一次获得物品时提示 |
| 6 | 选择权重 | 不同选择导致不同结果 | 2-3 个选择后玩家能感知到 |
| 7 | 存档意识 | 提示玩家可以随时暂停 | 第一次完成重要事件后 |
| 8 | 自由度边界 | 某些行动被拒绝并有合理解释 | 玩家尝试超范围行动时 |

### 2.4 引导退出机制

引导不能永远存在——玩家学会后应自然退出：

| 退出信号 | 触发条件 | 引导行为 |
|----------|----------|----------|
| 完成首幕 | 第一幕故事结束 | 移除所有提示，进入自由模式 |
| 玩家主动跳过 | 玩家输入"跳过引导"或类似 | 记录为"跳过引导"，直接进入自由模式 |
| 显示熟练 | 玩家连续 3 次做出有效行动无困惑 | 减少提示频率 |
| 玩家询问 | 玩家问"我该怎么玩" | 重新激活简化版引导 |

## §3 不同互动模式的引导

### 3.1 自由输入模式引导

```
开场示例（自由输入模式）：
"你在一间陌生的房间里醒来。头顶的灯泡发出刺眼的白光，空气中弥漫着消毒水的气味。
你注意到门是开着的。
你会怎么做？"

→ 等待玩家输入

如果玩家输入很模糊（如"看看"）：
→ "你环顾四周。房间不大，除了一张铁床和一个洗手台，什么都没有。门外的走廊延伸向黑暗中。你想仔细看什么，还是做点什么？"

如果玩家输入很具体（如"检查床底下"）：
→ 直接回应，不加引导语
```

**自由输入模式引导要点**：
- 首次模糊输入时，示范"你可以更具体"。
- 不否定玩家的任何合理输入。
- 通过 NPC 反应暗示"你的输入被认真对待了"。

### 3.2 选择式模式引导

```
开场示例（选择式模式）：
"你在一间陌生的房间里醒来。
A) 检查房间
B) 直接走出房门
C) 大声呼救"

→ 玩家选择 A

"你仔细检查了房间。床底下有一张纸条，上面写着一个数字：7。
A) 拿走纸条，走出房门
B) 继续检查其他地方
C) 把纸条放回原处，假装没看到"

→ 选项逐渐复杂，教会玩家"选择有后果"
```

### 3.3 混合模式引导

混合模式在开场使用选择式（降低门槛），逐渐过渡到允许自由输入：

```
第 1-3 轮: 纯选择式（A/B/C）
第 4-6 轮: 选择式 + "或输入你想做的事"
第 7+ 轮: 移除固定选项，纯自由输入
```

## §4 引导中的常见错误

| 错误 | 表现 | 修正 |
|------|------|------|
| 教程墙 | 开场 500 字介绍世界观/操作说明 | 把信息拆散到前 5 轮交互中 |
| 过度提示 | 每轮都提示"你可以..." | 只在首次出现新机制时提示 |
| 否定探索 | 玩家尝试非预设行动时说"你不能这样做" | 给予合理回应或解释为什么不行 |
| 无反馈选择 | 选择了 A 和 B 结果一样 | 确保前 3 个选择有明显不同的后果 |
| 信息过载 | 第一轮就展示完整状态/物品/关系 | 信息按需披露，需要时才展示 |
| 打破沉浸 | "欢迎来到游戏！这是一个互动小说..." | 引导融入叙事，用 NPC 之口传达 |

## §5 引导质量评估

| 指标 | 测量方式 | 目标值 |
|------|----------|--------|
| 完成率 | 完成第一幕的玩家比例 | > 90% |
| 首次困惑率 | 第一幕中玩家表达困惑的比例 | < 20% |
| 命令使用率 | 第一幕后玩家知道使用 `/status` 等命令的比例 | > 70% |
| 跳过率 | 选择跳过引导的玩家比例 | < 15% |
| 第二幕留存 | 从第一幕进入第二幕的玩家比例 | > 85% |

## §6 与其他文档的关系

- **`player-agency.md`**: 自主权定义"玩家能做什么"，引导教"怎么学会做"。
- **`genre-system.md`**: 不同类型的引导风格不同（悬疑 → 神秘开场；冒险 → 动作开场）。
- **`game-state-machine.md`**: 引导阶段的状态变化应简单可控。
- **`session-management.md`**: 引导阶段的存档设计——允许玩家重新开始第一幕。
- **`difficulty-engine.md`**: 引导阶段的难度应固定为最低，不触发自适应。
- **`anti-dumb-ai.md`**: 引导中的 AI 表现尤其重要——第一次印象决定玩家是否继续。
